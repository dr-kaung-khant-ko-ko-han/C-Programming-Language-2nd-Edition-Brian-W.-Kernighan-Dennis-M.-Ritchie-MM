### ၁.၃ The for statement (For ကွင်းဆက်)

လုပ်ငန်းဆောင်တာတစ်ခုအတွက် ပရိုဂရမ်ရေးသားရာတွင် နည်းလမ်းပေါင်းစုံရှိပါသည်။ အပူချိန်ပြောင်းလဲပေးသော converter ပရိုဂရမ်ကို ပုံစံတစ်မျိုးပြောင်းပြီး ရေးကြည့်ကြပါစို့။

```
#include <stdio.h>

/* print Fahrenheit-Celsius table */

main()
{
    int fahr;

    for (fahr = 0; fahr <= 300; fahr = fahr + 20)
        printf("%3d %6.1f\n", fahr, (5.0/9.0)*(fahr-32));
}
```

> **မှတ်ချက်:** မူရင်းဖိုင်တွင် `#include` ပြီးနောက် library အမည်မပါသော်လည်း စံသတ်မှတ်ချက်အရ `<stdio.h>` ဖြစ်လေ့ရှိသည်။

ဤပရိုဂရမ်သည် အဖြေတူတူပင် ထွက်ရှိသော်လည်း ပုံစံအားဖြင့် ကွဲပြားခြားနားပါသည်။ အဓိကပြောင်းလဲသွားသောအချက်မှာ variable (ကိန်းရှင်) အများစုကို ဖယ်ရှားလိုက်ခြင်းဖြစ်ပြီး `fahr` တစ်ခုတည်းသာ ကျန်ရှိကာ `int` အဖြစ် သတ်မှတ်ထားသည်ကို တွေ့ရပါမည်။ အနိမ့်ဆုံးသတ်မှတ်ချက် (lower limit)၊ အမြင့်ဆုံးသတ်မှတ်ချက် (upper limit) နှင့် အတိုးနှုန်း (step size) တို့သည် `for` statement ထဲတွင် ကိန်းသေ (constants) များအဖြစ်သာ ပါဝင်လာပါသည်။ ဆဲလ်စီးယပ်စ် (Celsius) အပူချိန်တွက်ချက်သည့် ပုံသေနည်းသည် သီးခြား variable သတ်မှတ်ခြင်းမပြုတော့ဘဲ `printf` ၏ တတိယမြောက် argument နေရာတွင် ဝင်ရောက်လာပါသည်။

ဤနောက်ဆုံးအပြောင်းအလဲသည် ယေဘုယျစည်းမျဉ်းတစ်ခုကို ပြသနေခြင်းဖြစ်သည် - တန်ဖိုးတစ်ခု (value) ကို အသုံးပြုခွင့်ရှိသော မည်သည့်နေရာတွင်မဆို၊ ထိုအမျိုးအစားနှင့် သက်ဆိုင်သည့် ပိုမိုရှုပ်ထွေးသော တွက်ချက်မှုပုံစံ (expression) များကို အစားထိုး အသုံးပြုနိုင်ပါသည်။ `printf` ၏ တတိယ argument သည် `%6.1f` နှင့် ကိုက်ညီရန် floating-point တန်ဖိုးဖြစ်ရမည်ဖြစ်သောကြောင့်၊ မည်သည့် floating-point expression ကိုမဆို ဤနေရာတွင် ထည့်သွင်းနိုင်ပါသည်။

`for` statement သည် ကွင်းဆက် (loop) တစ်ခုဖြစ်ပြီး `while` loop ကို ယေဘုယျပုံစံပြုထားခြင်း ဖြစ်သည်။ ၎င်းကို ယခင် `while` နှင့် နှိုင်းယှဉ်ကြည့်ပါက လုပ်ဆောင်ပုံကို ရှင်းလင်းစွာ သိနိုင်ပါသည်။ ကွင်းစကွင်းပိတ် `()` အတွင်း၌ အပိုင်း ၃ ပိုင်းရှိပြီး semicolon `;` များဖြင့် ခြားနားထားသည် -

1. **Initialization (ကနဦးသတ်မှတ်ခြင်း):** ပထမအပိုင်းဖြစ်သော `fahr = 0` သည် loop အတွင်းသို့ မဝင်ရောက်မီ တစ်ကြိမ်သာ လုပ်ဆောင်ပါသည်။
    
2. **Condition (စစ်ဆေးခြင်း):** ဒုတိယအပိုင်းမှာ loop ကို ထိန်းချုပ်မည့် `fahr <= 300` ဖြစ်သည်။ ဤအခြေအနေကို စစ်ဆေးပြီး မှန်ကန်ပါက (true ဖြစ်လျှင်) loop ၏ body (ဤနေရာတွင် `printf` တစ်ကြောင်းတည်း) ကို လုပ်ဆောင်ပါသည်။
    
3. **Increment (တိုးမြှင့်ခြင်း):** ထို့နောက် တတိယအပိုင်းဖြစ်သော `fahr = fahr + 20` ကို လုပ်ဆောင်ပြီး condition ကို ပြန်လည်စစ်ဆေးပါသည်။ အကယ်၍ condition သည် မှားယွင်းသွားပါက (false ဖြစ်လျှင်) loop ပြီးဆုံးပါသည်။
    

`while` နည်းတူ၊ loop ၏ body တွင် statement တစ်ကြောင်းတည်း (သို့မဟုတ်) တွန့်ကွင်း `{}` ဖြင့် အုပ်ထားသော statement အများအပြား ပါဝင်နိုင်ပါသည်။ Initialization, condition နှင့် increment နေရာများတွင် မည်သည့် expression ကိုမဆို ထည့်သွင်းနိုင်ပါသည်။

`while` နှင့် `for` ရွေးချယ်မှုသည် မိမိစိတ်ကြိုက်ဖြစ်ပြီး ဘယ်အရာက ပိုရှင်းလင်းသလဲဟူသည့်အပေါ် မူတည်ပါသည်။ Initialization နှင့် increment တို့သည် statement တစ်ခုစီသာရှိပြီး ယုတ္တိဗေဒအရ ဆက်စပ်နေသည့်အခါမျိုးတွင် `for` ကို အသုံးပြုခြင်းက ပိုသင့်လျော်လေ့ရှိသည်။ အဘယ်ကြောင့်ဆိုသော် `for` သည် `while` ထက် ပိုကျစ်လျစ်ပြီး loop ထိန်းချုပ်သည့် statement များကို တစ်နေရာတည်းတွင် စုစည်းထားသောကြောင့် ဖြစ်သည်။

**Exercise 1-5.** အပူချိန်ပြောင်းလဲပေးသော ပရိုဂရမ်ကို ပြင်ဆင်ရေးသားပါ။ အပူချိန် ၃၀၀ ဒီဂရီမှ ၀ ဒီဂရီအထိ ပြောင်းပြန် (reverse order) စီပြီး ထုတ်ပြရမည်။

### ၁.၄ Symbolic Constants (သင်္ကေတ ကိန်းသေများ)

အပူချိန်ပြောင်းလဲခြင်းကိစ္စကို အပြီးအပိုင် မထားခဲ့ခင် နောက်ဆုံးလေ့လာစရာတစ်ခု ကျန်ပါသေးသည်။ ပရိုဂရမ်တစ်ခုအတွင်း၌ `300` နှင့် `20` ကဲ့သို့သော ဂဏန်းများကို ဒီအတိုင်းမြှုပ်နှံထည့်သွင်းထားခြင်း (magic numbers ဟုခေါ်သည်) သည် မကောင်းသော အလေ့အကျင့်တစ်ခု ဖြစ်ပါသည်။ ၎င်းတို့သည် ပရိုဂရမ်ကို နောက်မှဖတ်ရှုသူများအတွက် သတင်းအချက်အလက် ပေးနိုင်စွမ်း နည်းပါးသလို၊ စနစ်တကျ ပြောင်းလဲပြင်ဆင်ရန်လည်း ခက်ခဲစေပါသည်။

Magic number များကို ကိုင်တွယ်ဖြေရှင်းရန် နည်းလမ်းတစ်ခုမှာ ၎င်းတို့ကို အဓိပ္ပာယ်ရှိသော အမည်များ ပေးခြင်းဖြစ်ပါသည်။ `#define` စာကြောင်းသည် သင်္ကေတအမည် (symbolic name) သို့မဟုတ် သင်္ကေတကိန်းသေ (symbolic constant) တစ်ခုကို သတ်မှတ်ထားသော စာလုံးတွဲတစ်ခုအဖြစ် အဓိပ္ပာယ်သတ်မှတ်ပေးသည်။

```
#define name replacement list
```

ထိုသို့သတ်မှတ်ပြီးပါက၊ ထိုအမည် (name) ပါဝင်သည့် နေရာတိုင်းတွင် (quote `""` အတွင်းမရှိသော၊ အခြားအမည်တစ်ခု၏ အစိတ်အပိုင်းမဟုတ်သော နေရာများ) သက်ဆိုင်ရာ အစားထိုးစာသား (replacement text) ဖြင့် အစားထိုးသွားမည် ဖြစ်ပါသည်။ အမည်ပေးရာတွင် variable အမည်ပေးသည့် ပုံစံအတိုင်း စာလုံးများနှင့် ဂဏန်းများ ပါဝင်နိုင်ပြီး စာလုံးဖြင့် စတင်ရပါမည်။ အစားထိုးစာသားသည် ဂဏန်းဖြစ်ရန်မလိုပါ၊ မည်သည့် စာလုံးတွဲမဆို ဖြစ်နိုင်ပါသည်။

```
#include <stdio.h>

#define LOWER 0 /* lower limit of table */
#define UPPER 300 /* upper limit */
#define STEP 20 /* step size */

/* print Fahrenheit-Celsius table */

main()
{
    int fahr;

    for (fahr = LOWER; fahr <= UPPER; fahr = fahr + STEP)
        printf("%3d %6.1f\n", fahr, (5.0/9.0)*(fahr-32));
}
```

`LOWER`၊ `UPPER` နှင့် `STEP` တို့သည် symbolic constant များသာဖြစ်ပြီး variable များ မဟုတ်ကြပါ။ ထို့ကြောင့် ၎င်းတို့ကို declaration (ကြေညာခြင်း) အပိုင်းတွင် ထည့်သွင်းရန် မလိုပါ။ Symbolic constant များကို စာလုံးအသေး (lower case) ဖြင့် ရေးလေ့ရှိသော variable များနှင့် ကွဲပြားစေရန် စာလုံးအကြီး (UPPER CASE) ဖြင့် ရေးသားလေ့ရှိကြပါသည်။ `#define` စာကြောင်းများ၏ အဆုံးတွင် semicolon `;` မပါဝင်သည်ကို သတိပြုပါ။
