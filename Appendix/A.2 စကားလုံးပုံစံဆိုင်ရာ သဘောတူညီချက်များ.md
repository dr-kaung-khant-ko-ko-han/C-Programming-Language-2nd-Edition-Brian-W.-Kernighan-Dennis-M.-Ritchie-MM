# A.2 စကားလုံးပုံစံဆိုင်ရာ သဘောတူညီချက်များ

ပရိုဂရမ်တစ်ခုသည် ဖိုင်များတွင် သိမ်းဆည်းထားသော ဘာသာပြန်ယူနစ် (translation units) တစ်ခု သို့မဟုတ် တစ်ခုထက်ပိုသောအရာများဖြင့် ဖွဲ့စည်းထားပါသည်။ ၎င်းကို အဆင့်များစွာဖြင့် ဘာသာပြန်ဆိုပြီး၊ Par.A.12 တွင် ဖော်ပြထားပါသည်။ ပထမအဆင့်များသည် အောက်ခြေအဆင့် စကားလုံးပုံစံ ပြောင်းလဲမှုများကို လုပ်ဆောင်ပြီး၊ # အက္ခရာဖြင့် စတင်သော လိုင်းများက မိတ်ဆက်ထားသော ညွှန်ကြားချက်များကို ဆောင်ရွက်ကာ၊ macro အဓိပ္ပါယ်ဖွင့်ဆိုခြင်းနှင့် တိုးချဲ့ခြင်းကို လုပ်ဆောင်ပါသည်။ Par.A.12 ၏ ကြိုတင်စီမံခြင်း ပြီးစီးသောအခါ၊ ပရိုဂရမ်သည် တိုကင်များ (tokens) ၏ အစဉ်လိုက်ဖြစ်လာပါသည်။

## A.2.1 တိုကင်များ

တိုကင် အမျိုးအစား ခြောက်မျိုးရှိပါသည် - အမည်သတ်မှတ်ချက်များ (identifiers)၊ သော့ချက်စကားလုံးများ (keywords)၊ ကိန်းသေများ (constants)၊ စာသား literals များ၊ အော်ပရေတာများ (operators)၊ နှင့် အခြားပိုင်းခြားသင်္ကေတများ (separators)။ အောက်တွင် ဖော်ပြထားသည့်အတိုင်း လစ်လပ်နေရာ (blanks)၊ အလျားလိုက်နှင့် ဒေါင်လိုက် tab များ၊ စာကြောင်းအသစ်များ (newlines)၊ formfeeds နှင့် မှတ်ချက်များ (စုပေါင်း၍ "white space" ဟုခေါ်သည်) ကို တိုကင်များကို ပိုင်းခြားခြင်းမှအပ လျစ်လျူရှုထားပါသည်။ တစ်ခါတစ်ရံ ကပ်လျက်ရှိသော အမည်သတ်မှတ်ချက်များ၊ သော့ချက်စကားလုံးများ၊ နှင့် ကိန်းသေများကို ပိုင်းခြားရန် white space လိုအပ်ပါသည်။

အကယ်၍ input stream ကို ပေးထားသော အက္ခရာတစ်ခုအထိ တိုကင်များအဖြစ် ပိုင်းခြားပြီးပါက၊ နောက်တိုကင်သည် တိုကင်တစ်ခု ဖွဲ့စည်းနိုင်သော အရှည်ဆုံး အက္ခရာစာကြောင်းဖြစ်ပါသည်။

## A.2.2 မှတ်ချက်များ

အက္ခရာ /* သည် မှတ်ချက်တစ်ခုကို မိတ်ဆက်ပြီး၊ အက္ခရာ */ ဖြင့် အဆုံးသတ်ပါသည်။ မှတ်ချက်များသည် အတွင်းတွင် ထပ်ဆင့်မထည့်နိုင်ပါ၊ ၎င်းတို့သည် string သို့မဟုတ် character literals များအတွင်း မပေါ်ပါ။

## A.2.3 အမည်သတ်မှတ်ချက်များ

အမည်သတ်မှတ်ချက်သည် စာလုံးများနှင့် ဂဏန်းများ၏ အစဉ်လိုက်ဖြစ်ပါသည်။ ပထမအက္ခရာသည် စာလုံးဖြစ်ရမည်၊ underscore _ ကို စာလုံးအဖြစ် သတ်မှတ်ပါသည်။ အကြီးနှင့် အသေး စာလုံးများသည် ကွဲပြားပါသည်။ အမည်သတ်မှတ်ချက်များသည် မည်သည့်အရှည်မဆို ရှိနိုင်ပြီး၊ အတွင်းပိုင်း အမည်သတ်မှတ်ချက်များအတွက်၊ အနည်းဆုံး ပထမ ၃၁ လုံးသည် အရေးပါပါသည်။

အချို့ implementation များသည် ပိုမိုသော အက္ခရာများကို အရေးပါစေနိုင်ပါသည်။ အတွင်းပိုင်း အမည်သတ်မှတ်ချက်များတွင် preprocessor macro အမည်များနှင့် ပြင်ပချိတ်ဆက်မှု (external linkage) မရှိသော အခြားအမည်အားလုံး (Par.A.11.2) ပါဝင်ပါသည်။ ပြင်ပချိတ်ဆက်မှုရှိသော အမည်သတ်မှတ်ချက်များသည် ပိုမိုကန့်သတ်ထားပါသည် - implementation များသည် ပထမ ခြောက်လုံးကိုသာ အရေးပါစေနိုင်ပြီး၊ အကြီးအသေး ကွာခြားမှုများကို လျစ်လျူရှုနိုင်ပါသည်။

## A.2.4 သော့ချက်စကားလုံးများ

အောက်ပါ အမည်သတ်မှတ်ချက်များသည် သော့ချက်စကားလုံးများအဖြစ် သုံးရန် သီးသန့်ထားရှိပြီး၊ အခြားနည်းဖြင့် အသုံးပြု၍မရပါ -

auto double int struct break else long switch case enum register typedef char extern return union const float short unsigned continue for signed void default goto sizeof volatile do if static while

အချို့ implementation များသည် fortran နှင့် asm စကားလုံးများကိုလည်း သီးသန့်ထားရှိပါသည်။

သော့ချက်စကားလုံးများဖြစ်သော const၊ signed၊ နှင့် volatile တို့သည် ANSI စံချိန်စံညွှန်းနှင့်အတူ အသစ်ဖြစ်ပါသည်၊ enum နှင့် void တို့သည် ပထမထုတ်ဝေမှုကတည်းက အသစ်ဖြစ်သော်လည်း အများအသုံးပြုကြပါသည်၊ entry သည် ယခင်က သီးသန့်ထားရှိသော်လည်း တစ်ခါမျှ မသုံးခဲ့သောကြောင့် ယခု သီးသန့်မထားရှိတော့ပါ။

## A.2.5 ကိန်းသေများ

ကိန်းသေများ အမျိုးအစားများစွာရှิပါသည်။ တစ်ခုစီတွင် ဒေတာအမျိုးအစား (data type) ရှိပါသည်၊ Par.A.4.2 သည် အခြေခံအမျိုးအစားများကို ဆွေးနွေးထားပါသည် -

constant:

integer-constant

character-constant

floating-constant

enumeration-constant

### A.2.5.1 ကိန်းပြည့်ကိန်းသေများ

ဂဏန်းများ၏ အစဉ်လိုက် ဖွဲ့စည်းထားသော ကိန်းပြည့်ကိန်းသေတစ်ခုသည် 0 (ဂဏန်းသုည) ဖြင့် စတင်လျှင် octal အဖြစ် သတ်မှတ်ပြီး၊ မဟုတ်လျှင် decimal အဖြစ် သတ်မှတ်ပါသည်။ Octal ကိန်းသေများတွင် ဂဏန်း 8 သို့မဟုတ် 9 မပါဝင်ပါ။ 0x သို့မဟုတ် 0X (ဂဏန်းသုည) ဖြင့် ရှေ့က ပြထားသော ဂဏန်းအစဉ်တစ်ခုသည် hexadecimal ကိန်းပြည့်အဖြစ် သတ်မှတ်ပါသည်။ Hexadecimal ဂဏန်းများတွင် a သို့မဟုတ် A မှ f သို့မဟုတ် F အထိ ပါဝင်ပြီး တန်ဖိုး 10 မှ 15 အထိ ကိုယ်စားပြုပါသည်။

ကိန်းပြည့်ကိန်းသေတစ်ခုသည် ၎င်းကို unsigned ဖြစ်ကြောင်း သတ်မှတ်ရန် u သို့မဟုတ် U စာလုံးဖြင့် ပူးတွဲထားနိုင်ပါသည်။ ၎င်းကို long ဖြစ်ကြောင်း သတ်မှတ်ရန် l သို့မဟုတ် L စာလုံးဖြင့်လည်း ပူးတွဲနိုင်ပါသည်။

ကိန်းပြည့်ကိန်းသေတစ်ခု၏ အမျိုးအစားသည် ၎င်း၏ပုံစံ၊ တန်ဖိုးနှင့် ပူးတွဲအက္ခရာပေါ် မူတည်ပါသည်။ (အမျိုးအစားများ ဆွေးနွေးချက်အတွက် Par.A.4 ကိုကြည့်ပါ)။ အကယ်၍ ပူးတွဲမထားပဲ decimal ဖြစ်လျှင်၊ ၎င်း၏ တန်ဖိုးကို ကိုယ်စားပြုနိုင်သော ဤအမျိုးအစားများထဲက ပထမတစ်ခုဖြစ်ပါသည် - int၊ long int၊ unsigned long int။ အကယ်၍ ပူးတွဲမထားပဲ octal သို့မဟုတ် hexadecimal ဖြစ်လျှင်၊ ဤအမျိုးအစားများထဲက ပထမဖြစ်နိုင်သောအရာ - int၊ unsigned int၊ long int၊ unsigned long int။ အကယ်၍ u သို့မဟုတ် U ဖြင့် ပူးတွဲထားလျှင်၊ unsigned int၊ unsigned long int။ အကယ်၍ l သို့မဟုတ် L ဖြင့် ပူးတွဲထားလျှင်၊ long int၊ unsigned long int။ အကယ်၍ ကိန်းပြည့်ကိန်းသေတစ်ခုကို UL ဖြင့် ပူးတွဲထားလျှင်၊ ၎င်းသည် unsigned long ဖြစ်ပါသည်။

ကိန်းပြည့်ကိန်းသေများ၏ အမျိုးအစားများကို အသေးစိတ်ဖော်ပြခြင်းသည် ပထမထုတ်ဝေမှုထက် များစွာကျော်လွန်သွားပါသည်၊ ပထမထုတ်ဝေမှုတွင် ကြီးမားသော ကိန်းပြည့်ကိန်းသေများကို long အဖြစ်သာ ပြောင်းလဲပေးခဲ့ပါသည်။ U ပူးတွဲအက္ခရာများသည် အသစ်ဖြစ်ပါသည်။

### A.2.5.2 အက္ခရာကိန်းသေများ

အက္ခရာကိန်းသေသည် 'x' ကဲ့သို့ single quotes ဖြင့် ပိတ်ထားသော အက္ခရာ တစ်ခု သို့မဟုတ် တစ်ခုထက်ပိုသော အစဉ်လိုက်ဖြစ်ပါသည်။ အက္ခရာတစ်ခုတည်းသာရှိသော အက္ခရာကိန်းသေ၏ တန်ဖိုးသည် စက်၏ အက္ခရာစနစ်တွင် ထိုအက္ခရာ၏ ကိန်းဂဏန်းတန်ဖိုးဖြစ်ပြီး အသုံးပြုချိန်တွင် ရရှိပါသည်။ အက္ခရာများစွာပါသော ကိန်းသေ၏ တန်ဖိုးသည် implementation-defined ဖြစ်ပါသည်။

အက္ခရာကိန်းသေများတွင် ' အက္ခရာ သို့မဟုတ် စာကြောင်းအသစ်များ မပါဝင်ပါ၊ ၎င်းတို့ကို ကိုယ်စားပြုရန်နှင့် အချို့အခြားအက္ခရာများအတွက်၊ အောက်ပါ escape sequences များကို သုံးနိုင်ပါသည် -

newline NL (LF) \n backslash \ \

horizontal tab HT \t question mark ? ?

vertical tab VT \v single quote ' '

backspace BS \b double quote " "

carriage return CR \r octal number ooo \ooo

Escape \ooo သည် backslash နောက် octal ဂဏန်း 1၊ 2 သို့မဟုတ် 3 လုံးဖြင့် ဖွဲ့စည်းထားပြီး၊ လိုချင်သော အက္ခရာ၏ တန်ဖိုးကို သတ်မှတ်ရန် အသုံးပြုပါသည်။ ဤတည်ဆောက်ပုံ၏ အဖြစ်များသော ဥပမာမှာ \0 (ဂဏန်းနောက် မပါသော) ဖြစ်ပြီး၊ NUL အက္ခရာကို သတ်မှတ်ပါသည်။ Escape \xhh သည် backslash နောက် x၊ နောက်ထပ် hexadecimal ဂဏန်းများဖြင့် ဖွဲ့စည်းထားပြီး၊ လိုချင်သော အက္ခရာ၏ တန်ဖိုးကို သတ်မှတ်ရန် အသုံးပြုပါသည်။ ဂဏန်းအရေအတွက်တွင် ကန့်သတ်ချက်မရှိသော်လည်း၊ ရလဒ်အက္ခရာတန်ဖိုးသည် အကြီးဆုံးအက္ခရာထက် ကျော်လွန်လျှင် အပြုအမူကို သတ်မှတ်မထားပါ။ Octal သို့မဟုတ် hexadecimal escape အက္ခရာများအတွက်၊ အကယ်၍ implementation သည် char အမျိုးအစားကို signed အဖြစ် သတ်မှတ်လျှင်၊ တန်ဖိုးကို char အမျိုးအစားသို့ cast လုပ်သကဲ့သို့ sign-extended လုပ်ပါသည်။ အကယ်၍ \ နောက်က အက္ခရာသည် သတ်မှတ်ထားသောအရာများထဲမှ မဟုတ်လျှင်၊ အပြုအမူကို သတ်မှတ်မထားပါ။

အချို့ implementation များတွင်၊ char အမျိုးအစားတွင် ကိုယ်စားပြု၍မရသော အက္ခရာအစု တိုးချဲ့ထားရှိပါသည်။ ဤတိုးချဲ့အစုထဲက ကိန်းသေတစ်ခုကို L ရှေ့တွင်ထား၍ ရေးသားပါသည်၊ ဥပမာ L'x'၊ နှင့် ၎င်းကို wide character constant ဟုခေါ်ပါသည်။ ထိုကဲ့သို့သော ကိန်းသေသည် wchar_t အမျိုးအစားရှိပြီး၊ စံချိန်စံညွှန်း header တွင် သတ်မှတ်ထားသော integral အမျိုးအစားဖြစ်ပါသည်။ သာမန်အက္ခရာကိန်းသေများကဲ့သို့ပင်၊ hexadecimal escapes များကို သုံးနိုင်ပါသည်၊ သတ်မှတ်ထားသော တန်ဖိုးသည် wchar_t ဖြင့် ကိုယ်စားပြုနိုင်သော တန်ဖိုးကို ကျော်လွန်လျှင် အကျိုးသက်ရောက်မှု သတ်မှတ်မထားပါ။

ဤ escape sequences များထဲမှ အချို့သည် အသစ်ဖြစ်ပြီး၊ အထူးသဖြင့် hexadecimal အက္ခရာကိုယ်စားပြုမှုဖြစ်ပါသည်။ တိုးချဲ့အက္ခရာများလည်း အသစ်ဖြစ်ပါသည်။ အမေရိကများနှင့် အနောက်ဥရောပတွင် အများအသုံးပြုသော အက္ខရာစနစ်များကို char အမျိုးအစားတွင် ကိုက်ညီစေရန် encode လုပ်နိုင်ပါသည်၊ wchar_t ကို ထည့်သွင်းခြင်း၏ အဓိကရည်ရွယ်ချက်မှာ အာရှဘာသာစကားများကို ထောက်ပံ့ရန်ဖြစ်ပါသည်။

### A.2.5.3 ဒသမကိန်းသေများ

ဒသမကိန်းသေတစ်ခုတွင် ကိန်းပြည့်အပိုင်း၊ ဒသမအပိုင်း၊ အပိုင်းကိန်း၊ e သို့မဟုတ် E၊ လက္ခဏာနှင့်တကွ သို့မဟုတ် မပါဘဲ ကိန်းပြည့် exponent နှင့် ရွေးချယ်နိုင်သော အမျိုးအစားပူးတွဲ - f၊ F၊ l သို့မဟုတ် L တို့ ပါဝင်ပါသည်။ ကိန်းပြည့်နှင့် အပိုင်းကိန်းအပိုင်းများ နှစ်ခုလုံးသည် ဂဏန်းအစဉ်လိုက်ဖြင့် ဖွဲ့စည်းထားပါသည်။ ကိန်းပြည့်အပိုင်း သို့မဟုတ် အပိုင်းကိန်းအပိုင်း (နှစ်ခုလုံးမဟုတ်) သည် ပျောက်နိုင်ပါသည်၊ ဒသမအမှတ် သို့မဟုတ် e နှင့် exponent (နှစ်ခုလုံးမဟုတ်) သည် ပျောက်နိုင်ပါသည်။ အမျိုးအစားကို ပူးတွဲအက္ခရာဖြင့် သတ်မှတ်ပါသည်၊ F သို့မဟုတ် f သည် float ဖြစ်စေပြီး၊ L သို့မဟုတ် l သည် long double ဖြစ်စေပါသည်၊ မဟုတ်လျှင် double ဖြစ်ပါသည်။

### A2.5.4 စာရင်းကောက်ယူမှု ကိန်းသေများ

Enumerators အဖြစ် ကြေညာထားသော အမည်သတ်မှတ်ချက်များ (Par.A.8.4 ကြည့်ပါ) သည် int အမျိုးအစား၏ ကိန်းသေများဖြစ်ပါသည်။

## A.2.6 String Literals

String literal (string constant ဟုလည်းခေါ်သည်) သည် "..." ကဲ့သို့ double quotes ဖြင့် ဝန်းရံထားသော အက္ခရာအစဉ်လိုက်ဖြစ်ပါသည်။ String တစ်ခုတွင် "array of characters" အမျိုးအစားနှင့် static storage class (အောက်ပါ Par.A.3 ကြည့်ပါ) ရှိပြီး ပေးထားသော အက္ခရာများဖြင့် initialize လုပ်ထားပါသည်။ တူညီသော string literals များသည် ကွဲပြားသည်ဟု သတ်မှတ်မှုမှာ implementation-defined ဖြစ်ပြီး၊ string literal ကို ပြောင်းလဲရန် ကြိုးစားသော ပရိုဂရမ်၏ အပြုအမူသည် သတ်မှတ်မထားပါ။

ကပ်လျက်ရှိသော string literals များကို string တစ်ခုတည်းအဖြစ် ပေါင်းစပ်ပါသည်။ မည်သည့် ပေါင်းစပ်မှုပြီးနောက်၊ null byte \0 ကို string သို့ ထည့်ပြီး string ကို စကင်ဖတ်သော ပရိုဂရမ်များက ၎င်း၏အဆုံးကို ရှာတွေ့နိုင်ပါသည်။ String literals များတွင် စာကြောင်းအသစ် သို့မဟုတ် double-quote အက္ခရာများ မပါဝင်ပါ၊ ၎င်းတို့ကို ကိုယ်စားပြုရန်၊ အက္ခရာကိန်းသေများအတွက် ရှိသည့် escape sequences များကိုပင် သုံးနိုင်ပါသည်။

အက္ခရာကိန်းသေများကဲ့သို့ပင်၊ တိုးချဲ့အက္ခရာစနစ်ရှိ string literals များကို L'..." ကဲ့သို့ L ရှေ့တွင်ထား၍ ရေးသားပါသည်။ Wide-character string literals များတွင် "array of wchar_t" အမျိုးအစားရှိပါသည်။ သာမန်နှင့် wide string literals များကို ပေါင်းစပ်ခြင်းမှာ သတ်မှတ်မထားပါ။

String literals များသည် ကွဲပြားမှု မလိုအပ်ကြောင်း သတ်မှတ်ချက်နှင့်၊ ၎င်းတို့ကို ပြင်ဆင်ခြင်းကို တားမြစ်ခြင်းတို့သည် ANSI စံချိန်စံညွှန်းတွင် အသစ်ဖြစ်ပြီး၊ ကပ်လျက်ရှိသော string literals များကို ပေါင်းစပ်ခြင်းလည်း အသစ်ဖြစ်ပါသည်။ Wide-character string literals များလည်း အသစ်ဖြစ်ပါသည်။
