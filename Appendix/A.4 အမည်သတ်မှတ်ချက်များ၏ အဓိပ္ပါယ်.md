# A.4 အမည်သတ်မှတ်ချက်များ၏ အဓိပ္ပါယ်

အမည်သတ်မှတ်ချက်များ သို့မဟုတ် အမည်များသည် အမျိုးမျိုးသောအရာများကို ရည်ညွှန်းပါသည် - လုပ်ငန်းဆောင်တာများ (functions)၊ structures၊ unions နှင့် enumerations များ၏ tags များ၊ structures သို့မဟုတ် unions များ၏ အဖွဲ့ဝင်များ (members)၊ enumeration ကိန်းသေများ၊ typedef အမည်များ၊ နှင့် objects များ။ Object တစ်ခု (တစ်ခါတစ်ရံ variable ဟုခေါ်သည်) သည် သိုလှောင်မှုတွင် တည်နေရာတစ်ခုဖြစ်ပြီး၊ ၎င်း၏ အဓိပ္ပါယ်ဖွင့်ဆိုချက်သည် အဓိက လက္ခဏာနှစ်ခုပေါ် မူတည်ပါသည် - ၎င်း၏ storage class နှင့် ၎င်း၏ အမျိုးအစား (type)။ Storage class သည် ခွဲခြားသတ်မှတ်ထားသော object နှင့် ဆက်စပ်သော သိုလှောင်မှု၏ သက်တမ်းကို သတ်မှတ်ပါသည်၊ အမျိုးအစားသည် ခွဲခြားသတ်မှတ်ထားသော object တွင် တွေ့ရှိသော တန်ဖိုးများ၏ အဓိပ္ပါယ်ကို သတ်မှတ်ပါသည်။ အမည်တစ်ခုတွင် scope လည်း ရှိပြီး၊ ၎င်းသည် ပရိုဂရမ်တွင် သိရှိထားသော နယ်ပယ်ဖြစ်ကာ၊ linkage လည်း ရှိပြီး၊ အခြား scope တွင်ရှိသော အမည်တူသည် object တူ သို့မဟုတ် function တူကို ရည်ညွှန်းခြင်း ရှိမရှိကို သတ်မှတ်ပါသည်။ Scope နှင့် linkage ကို Par.A.11 တွင် ဆွေးနွေးထားပါသည်။

## A.4.1 Storage Class

Storage class နှစ်မျိုးရှိပါသည် - automatic နှင့် static။ သော့ချက်စကားလုံးများစွာသည် object ၏ ကြေညာချက်၏ အကြောင်းအရာနှင့်အတူ ၎င်း၏ storage class ကို သတ်မှတ်ပါသည်။ Automatic objects များသည် block တစ်ခုအတွက် (Par.9.3) local ဖြစ်ပြီး၊ block မှ ထွက်သောအခါ စွန့်ပစ်ပါသည်။ Block တစ်ခုအတွင်းရှි ကြေညာချက်များသည် storage class သတ်မှတ်ချက် မဖော်ပြပါက သို့မဟုတ် auto specifier ကို အသုံးပြုပါက automatic objects များကို ဖန်တီးပါသည်။ register ဖြင့် ကြေညာထားသော objects များသည် automatic ဖြစ်ပြီး၊ (ဖြစ်နိုင်လျှင်) စက်၏ မြန်သော registers များတွင် သိမ်းဆည်းပါသည်။

Static objects များသည် block တစ်ခုအတွက် local ဖြစ်နိုင်သည် သို့မဟုတ် blocks အားလုံး၏ ပြင်ပတွင် ရှိနိုင်ပါသည်၊ သို့သော် မည်သည့်ကိစ္စတွင်မဆို functions နှင့် blocks များမှ ထွက်ခြင်းနှင့် ပြန်ဝင်ခြင်းများတစ်လျှောက် ၎င်းတို့၏ တန်ဖိုးများကို ထိန်းသိမ်းထားပါသည်။ Block တစ်ခုအတွင်း (function အတွက် code ပံ့ပိုးပေးသော block အပါအဝင်)၊ static objects များကို static သော့ချက်စကားလုံးဖြင့် ကြေညာပါသည်။ Blocks အားလုံး၏ပြင်ပတွင် function အဓိပ္ပါယ်ဖွင့်ဆိုချက်များနှင့် တူညီသောအဆင့်တွင် ကြေညာထားသော objects များသည် အမြဲ static ဖြစ်ပါသည်။ ၎င်းတို့ကို static သော့ချက်စကားလုံးဖြင့် သုံး၍ သီးခြား translation unit တစ်ခုအတွက် local ဖြစ်စေနိုင်ပါသည်၊ ဤနည်းက ၎င်းတို့အား internal linkage ပေးပါသည်။ တိကျသော storage class ကို ချန်လှပ်ခြင်း သို့မဟုတ် extern သော့ချက်စကားလုံးကို သုံးခြင်းဖြင့် ၎င်းတို့သည် ပရိုဂရမ်တစ်ခုလုံးအတွက် global ဖြစ်လာပါသည်၊ ဤနည်းက ၎င်းတို့အား external linkage ပေးပါသည်။

## A.4.2 အခြေခံအမျိုးအစားများ

အခြေခံအမျိုးအစားများ အများအပြားရှိပါသည်။ နောက်ဆက်တွဲ B တွင် ဖော်ပြထားသော စံချိန်စံညွှန်း header သည် local implementation တွင် တစ်ခုစီ၏ အကြီးဆုံးနှင့် အသေးဆုံးတန်ဖိုးများကို သတ်မှတ်ပါသည်။ နောက်ဆက်တွဲ B တွင် ပေးထားသော ကိန်းဂဏန်းများသည် အသေးဆုံး လက်ခံနိုင်သော ပမာဏများကို ပြသပါသည်။

အက္ခရာများအဖြစ် (char) ကြေညာထားသော objects များသည် အသုံးပြုမှု အက္ခရာစနစ်၏ အဖွဲ့ဝင်တစ်ခုခစီကို သိမ်းဆည်းရန် လုံလောက်သော ကြီးမားမှုရှိပါသည်။ အကယ်၍ ထိုစနစ်မှ စစ်မှန်သော အက္ခရာတစ်ခုကို char object တွင် သိမ်းဆည်းလျှင်၊ ၎င်း၏တန်ဖိုးသည် အက္ခရာအတွက် ကိန်းပြည့်ကုဒ်နှင့် ညီမျှပြီး၊ အနုတ်လက္ခဏာမဟုတ်ပါ။ အခြားပမာဏများကို char ကိန်းရှင်များ (variables) တွင် သိမ်းဆည်းနိုင်ပါသည်၊ သို့သော် ရရှိနိုင်သော တန်ဖိုးအပိုင်းအခြား၊ အထူးသဖြင့် တန်ဖိုးသည် signed ဖြစ်ခြင်း ရှိမရှိသည် implementation-dependent ဖြစ်ပါသည်။

unsigned char ဖြင့် ကြေညာထားသော Unsigned characters များသည် plain characters များကဲ့သို့ ပမာဏတူ နေရာယူပြီး၊ အမြဲ အနုတ်လက္ခဏာမဟုတ်သည့် ပုံစံဖြင့် ပေါ်လာပါသည်၊ signed char ဖြင့် ကြေညာထားသော တိကျစွာ signed characters များသည် plain characters များကဲ့သို့ နေရာတူယူပါသည်။

unsigned char အမျိုးအစားသည် ဤစာအုပ်၏ ပထမထုတ်ဝေမှုတွင် မပေါ်ပါ၊ သို့သော် အများအသုံးပြုကြပါသည်။ signed char သည် အသစ်ဖြစ်ပါသည်။

char အမျိုးအစားများအပြင်၊ ကိန်းပြည့်၏ အရွယ်အစား သုံးမျိုးအထိ - short int၊ int၊ နှင့် long int ဖြင့် ကြေညာထားသော - ရရှိနိုင်ပါသည်။ Plain int objects များသည် host စက် architecture က အကြံပြုသော သဘာဝအရွယ်အစားရှိပါသည်၊ အခြားအရွယ်အစားများကို အထူးလိုအပ်ချက်များ ဖြည့်ဆည်းရန် ပံ့ပိုးပေးထားပါသည်။ ပိုရှည်သော ကိန်းပြည့်များသည် ပိုတိုသောအရာများထက် အနည်းဆုံး ပမာဏတူ သိုလှောင်မှု ပံ့ပိုးပေးပြီး၊ implementation သည် plain integers များကို short integers သို့မဟုတ် long integers များနှင့် ညီမျှစေနိုင်ပါသည်။ int အမျိုးအစားအားလုံးသည် အခြားနည်း မသတ်မှတ်ပါက signed တန်ဖိုးများကို ကိုယ်စားပြုပါသည်။

unsigned သော့ချက်စကားလုံးဖြင့် ကြေညာထားသော Unsigned integers များသည် arithmetic modulo 2n ၏ နိယာမများကို လိုက်နာပြီး n သည် ကိုယ်စားပြုမှုတွင် bits အရေအတွက်ဖြစ်ပြီး၊ ထို့ကြောင့် unsigned ပမာဏများပေါ်တွင် arithmetic သည် ဘယ်တော့မှ overflow မဖြစ်နိုင်ပါ။ signed object တစ်ခုတွင် သိမ်းဆည်းနိုင်သော အနုတ်လက္ခဏာမဟုတ်သော တန်ဖိုးများ၏ အစုသည် သက်ဆိုင်ရာ unsigned object တွင် သိမ်းဆည်းနိုင်သော တန်ဖိုးများ၏ အစုခွဲတစ်ခုဖြစ်ပြီး၊ ထပ်နေသော တန်ဖိုးများအတွက် ကိုယ်စားပြုမှုသည် တူညီပါသည်။

single precision floating point (float)၊ double precision floating point (double)၊ နှင့် extra precision floating point (long double) တို့မှ တစ်ခုခုသည် တူညီနိုင်ပါသည်၊ သို့သော် စာရင်းတွင် နောက်ပိုင်းရှိသောအရာများသည် အနည်းဆုံး ရှေ့က အရာများကဲ့သို့ တိကျမှန်ကန်ပါသည်။

long double သည် အသစ်ဖြစ်ပါသည်။ ပထမထုတ်ဝေမှုသည် long float ကို double နှင့် ညီမျှစေခဲ့ပြီး၊ ထိုအသုံးအနှုန်းကို ရုပ်သိမ်းလိုက်ပါသည်။

Enumerations များသည် ကိန်းပြည့်တန်ဖိုးများရှိသော ထူးခြားသော အမျိုးအစားများဖြစ်ပါသည်၊ enumeration တစ်ခုစီနှင့်အတူ အမည်ရှိသော ကိန်းသေများ၏ အစုတစ်ခု (Par.A.8.4) ဆက်စပ်နေပါသည်။ Enumerations များသည် ကိန်းပြည့်များကဲ့သို့ အပြုအမူပြုပါသည်၊ သို့သော် သီးခြား enumeration တစ်ခု၏ object ကို ၎င်း၏ ကိန်းသေများထဲမှ တစ်ခု သို့မဟုတ် ၎င်း၏အမျိုးအစား၏ expression မှအပ အခြားအရာတစ်ခုခု သတ်မှတ်လိုက်သောအခါ compiler က သတိပေးချက်တစ်ခု ထုတ်ပေးခြင်းသည် အဖြစ်များပါသည်။

ဤအမျိုးအစားများ၏ objects များကို ကိန်းဂဏန်းများအဖြစ် အဓိပ္ပါယ်ဖွင့်ဆိုနိုင်သောကြောင့်၊ ၎င်းတို့ကို arithmetic အမျိုးအစားများအဖြစ် ရည်ညွှန်းမည်ဖြစ်ပါသည်။ char နှင့် အရွယ်အစားအားလုံး၏ int များ၊ တစ်ခုစီတွင် လက္ခဏာနှင့်တကွ သို့မဟုတ် မပါဘဲ၊ နှင့် enumeration အမျိုးအစားများကိုလည်း စုပေါင်း၍ integral အမျိုးအစားများဟု ခေါ်မည်ဖြစ်ပါသည်။ float၊ double၊ နှင့် long double အမျိုးအစားများကို floating အမျိုးအစားများဟု ခေါ်မည်ဖြစ်ပါသည်။

void အမျိုးအစားသည် တန်ဖိုးများ၏ အလွတ်အစုတစ်ခုကို သတ်မှတ်ပါသည်။ ၎င်းကို တန်ဖိုးမထုတ်လုပ်သော functions များက ပြန်ပို့သော အမျိုးအစားအဖြစ် အသုံးပြုပါသည်။

## A.4.3 ဆင်းသက်လာသော အမျိုးအစားများ

အခြေခံအမျိုးအစားများအပြင်၊ အောက်ပါနည်းလမ်းများဖြင့် အခြေခံအမျိုးအစားများမှ တည်ဆောက်ထားသော ဆင်းသက်လာသော အမျိုးအစားများ၏ အယူအဆအရ အဆုံးမဲ့ အတန်းအစားတစ်ခု ရှိပါသည် -

ပေးထားသော အမျိုးအစားတစ်ခု၏ objects များ၏ arrays များ၊

ပေးထားသော အမျိုးအစားတစ်ခု၏ objects များသို့ pointers များ၊

အမျိုးမျိုးသော အမျိုးအစားများ၏ objects များထဲမှ တစ်ခုခုကို ပါဝင်နိုင်သော unions များ။

## A.4.4 အမျိုးအစား အရည်အသွေးသတ်မှတ်ချက်များ

Object တစ်ခု၏ အမျိုးအစားတွင် နောက်ထပ် အရည်အသွေးသတ်မှတ်ချက်များ (qualifiers) ရှိနိုင်ပါသည်။ Object တစ်ခုကို const ဖြင့် ကြေညာခြင်းသည် ၎င်း၏တန်ဖိုး ပြောင်းလဲမည်မဟုတ်ကြောင်း ကြေငြာခြင်းဖြစ်ပါသည်၊ ၎င်းကို volatile ဖြင့် ကြေညာခြင်းသည် optimization နှင့် သက်ဆိုင်သော အထူးဂုဏ်သတ္တိများ ရှိကြောင်း ကြေငြာခြင်းဖြစ်ပါသည်။ မည်သည့် qualifier မျှ object ၏ တန်ဖိုးအပိုင်းအခြား သို့မဟုတ် arithmetic ဂုဏ်သတ္တိများကို မထိခိုက်ပါ။ Qualifiers များကို Par.A.8.2 တွင် ဆွေးနွေးထားပါသည်။
