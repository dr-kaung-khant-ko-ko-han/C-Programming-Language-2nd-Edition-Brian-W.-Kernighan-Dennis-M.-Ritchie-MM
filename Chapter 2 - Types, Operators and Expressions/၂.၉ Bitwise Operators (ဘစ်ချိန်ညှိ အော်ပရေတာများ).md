# ၂.၉ Bitwise Operators (ဘစ်ချိန်ညှိ အော်ပရေတာများ)

C သည် bit manipulation (ဘစ်ပြောင်းလဲခြင်း) အတွက် အော်ပရေတာခြောက်မျိုးကို ပံ့ပိုးပေးပါသည်။ ဤအော်ပရေတာများကို integral operands (ကိန်းပြည့်ကိန်းရှင်များ) ဖြစ်သော char, short, int, နှင့် long တို့တွင်သာ အသုံးပြုနိုင်ပြီး signed (လက်မှတ်ပါသော) သို့မဟုတ် unsigned (လက်မှတ်မပါသော) ဖြစ်နိုင်ပါသည်။

```
&  bitwise AND (ဘစ်ချိန်ညှိ AND)
| bitwise inclusive OR (ဘစ်ချိန်ညှိ inclusive OR)
^ bitwise exclusive OR (ဘစ်ချိန်ညှိ exclusive OR)
<< left shift (ဘယ်ဘက်ရွှေ့ခြင်း)
>> right shift (ညာဘက်ရွှေ့ခြင်း)
~ one's complement (ဖြည့်စွက်ခြင်း - unary)
```

## Bitwise AND Operator (&)

Bitwise AND အော်ပရေတာ & ကို အချို့သော ဘစ်အစုများကို ဖျောက်ရန် (mask off) အသုံးများပါသည်။ ဥပမာ -

```
n = n & 0177;
```

ဤကုဒ်သည် n ၏ အောက်ဆုံး ၇ ဘစ်များမှလွဲ၍ ကျန်ဘစ်များအားလုံးကို သုညသို့ သတ်မှတ်ပေးပါသည်။

## Bitwise OR Operator (|)

Bitwise OR အော်ပရေတာ | ကို ဘစ်များကို ဖွင့်ရန် အသုံးပြုပါသည် -

```
x = x | SET_ON;
```

ဤကုဒ်သည် SET_ON တွင် တစ်ဖြစ်နေသော ဘစ်များကို x တွင် တစ်အဖြစ် သတ်မှတ်ပေးပါသည်။

## Bitwise Exclusive OR Operator (^)

Bitwise exclusive OR အော်ပရေတာ ^ သည် operands နှစ်ခု၏ ဘစ်များ မတူညီသည့် နေရာများတွင် တစ်ကို သတ်မှတ်ပြီး တူညီသည့် နေရာများတွင် သုညကို သတ်မှတ်ပါသည်။

**သတိပြုရန်** - bitwise အော်ပရေတာများဖြစ်သော & နှင့် | တို့ကို logical အော်ပရေတာများဖြစ်သော && နှင့် || တို့နှင့် ခွဲခြားသိရှိရပါမည်။ Logical အော်ပရေတာများသည် ဘယ်မှညာသို့ အမှန်တန်ဖိုး (truth value) ကို တွက်ချက်ပါသည်။ ဥပမာ - x သည် ၁ ဖြစ်ပြီး y သည် ၂ ဖြစ်လျှင် x & y သည် သုညဖြစ်သော်လည်း x && y သည် တစ်ဖြစ်ပါသည်။

## Shift Operators (<< နှင့် >>)

Shift အော်ပရေတာများ << နှင့် >> သည် ၎င်းတို့၏ ဘယ်ဘက် operand ကို ညာဘက် operand ပေးထားသော ဘစ်အနေအထားအရေအတွက်အတိုင်း ဘယ်နှင့် ညာဘက်သို့ ရွှေ့ပေးပါသည်။ ညာဘက် operand သည် အနုတ်မဖြစ်ရပါ။ ထို့ကြောင့် x << 2 သည် x ၏တန်ဖိုးကို အနေအထားနှစ်ခု ရွှေ့ပေးပြီး အလွတ်ဖြစ်သွားသော ဘစ်များကို သုညဖြင့် ဖြည့်ပေးပါသည်။ ဤသည်မှာ ၄ ဖြင့် မြှောက်ခြင်းနှင့် ညီမျှပါသည်။

Unsigned ပမာဏကို ညာဘက်သို့ ရွှေ့ခြင်းသည် အလွတ်ဘစ်များကို အမြဲတမ်း သုညဖြင့် ဖြည့်ပါသည်။ Signed ပမာဏကို ညာဘက်သို့ ရွှေ့ခြင်းသည် စက်အချို့တွင် လက်မှတ်ဘစ်များဖြင့် ဖြည့်သည် ("arithmetic shift") နှင့် အချို့စက်များတွင် ၀-ဘစ်များဖြင့် ဖြည့်သည် ("logical shift")။

## One's Complement Operator (~)

Unary အော်ပရေတာ ~ သည် ကိန်းပြည့်တစ်ခု၏ one's complement (ဖြည့်စွက်) ကို ပေးပါသည်။ ဆိုလိုသည်မှာ ၁-ဘစ်တိုင်းကို ၀-ဘစ်အဖြစ် ပြောင်းလဲပြီး ပြောင်းပြန်လည်လုပ်ပါသည်။ ဥပမာ -

```
x = x & ~077
```

သည် x ၏ နောက်ဆုံး ခြောက်ဘစ်များကို သုညသို့ သတ်မှတ်ပေးပါသည်။ x & ~077 သည် စကားလုံးအရှည် (word length) နှင့် မဆိုင်ဘဲ လွတ်လပ်သောကြောင့် x & 0177700 (x သည် ၁၆-ဘစ်ပမာဏဟု ယူဆထားသော) ထက် ပို၍ ကောင်းပါသည်။ Portable ပုံစံသည် ထပ်ဆောင်းကုန်ကျစရိတ်မရှိပါ၊ အဘය်ကြောင့်ဆိုသော် ~077 သည် compile time တွင် တွက်ချက်နိုင်သော constant expression ဖြစ်သောကြောင့်ဖြစ်ပါသည်။

## getbits() Function ဥပမာ

Bit အော်ပရေတာအချို့၏ သရုပ်ပြချက်အဖြစ် getbits(x,p,n) function ကို စဉ်းစားကြည့်ပါ။ ဤ function သည် x ၏ အနေအထား p မှ စတင်သော n-ဘစ် field ကို (ညာဘက်ညှိပြီး) ပြန်ပေးပါသည်။ ဘစ်အနေအထား ၀ သည် ညာဘက်အဆုံးတွင်ရှိပြီး n နှင့် p တို့သည် သင့်လျော်သော အပေါင်းတန်ဖိုးများဟု ယူဆထားပါသည်။ ဥပမာ - getbits(x,4,3) သည် အနေအထား ၄၊ ၃ နှင့် ၂ ရှိ ဘစ်သုံးခုကို ညာဘက်ညှိပြီး ပြန်ပေးပါသည်။

```c
/* getbits: get n bits from position p */
unsigned getbits(unsigned x, int p, int n)
{
    return (x >> (p+1-n)) & ~(~0 << n);
}
```

Expression x >> (p+1-n) သည် လိုချင်သော field ကို စကားလုံး၏ ညာဘက်အဆုံးသို့ ရွှေ့ပေးပါသည်။ ~0 သည် ဘစ်အားလုံး ၁ ဖြစ်ပါသည်။ ၎င်းကို ဘယ်ဘက်သို့ n အနေအထား ရွှေ့ခြင်း (~0 << n) သည် အညာဆုံး n ဘစ်များတွင် သုညများ ထားပေးပါသည်။ ထိုအရာကို ~ ဖြင့် complementing လုပ်ခြင်းသည် အညာဆုံး n ဘစ်များတွင် တစ်များပါသော mask တစ်ခုကို ဖန်တီးပေးပါသည်။

# ၂.၁၀ Assignment Operators နှင့် Expressions (တန်ဖိုးသတ်မှတ်ခြင်း အော်ပရေတာများနှင့် အသုံးအနှုန်းများ)

ဤကဲ့သို့သော expression တစ်ခုတွင် -

```
i = i + 2
```

ဘယ်ဘက်ခြမ်းရှိ variable သည် ညာဘက်ခြမ်းတွင် ချက်ချင်း ထပ်ပေါ်လာပါက အောက်ပါ ချုံ့ပုံစံ (compressed form) ဖြင့် ရေးနိုင်ပါသည် -

```
i += 2
```

အော်ပရေတာ += ကို assignment operator (တန်ဖိုးသတ်မှတ်ခြင်း အော်ပရေတာ) ဟုခေါ်ပါသည်။

## Assignment Operators စာရင်း

Binary အော်ပရေတာအများစု (ဘယ်နှင့် ညာ operand ရှိသော အော်ပရေတာများဖြစ်သော +) တွင် သက်ဆိုင်ရာ assignment operator op= ရှိပါသည်။ op သည် အောက်ပါတို့ထဲမှ တစ်ခုခုဖြစ်သည် -

```
+   -   *   /   %   <<   >>   &   ^   |
```

expr1 နှင့် expr2 သည် expressions များဖြစ်လျှင် -

```
expr1 op= expr2
```

သည် အောက်ပါနှင့် ညီမျှပါသည် -

```
expr1 = (expr1) op (expr2)
```

သို့သော် expr1 ကို တစ်ကြိမ်သာ တွက်ချက်ပါသည်။ expr2 ၏ ပတ်လည်တွင် ကွင်းပိတ်များ သတိပြုပါ -

```
x *= y + 1
```

ဆိုလိုသည်မှာ -

```
x = x * (y + 1)
```

ဖြစ်ပြီး အောက်ပါမဟုတ်ပါ -

```
x = x * y + 1
```

## bitcount() Function ဥပမာ

ဥပမာအနေဖြင့် bitcount function သည် ၎င်း၏ integer argument တွင် ရှိသော ၁-ဘစ်များကို ရေတွက်ပါသည်။

```c
/* bitcount: count 1 bits in x */
int bitcount(unsigned x)
{
    int b;
    for (b = 0; x != 0; x >>= 1)
        if (x & 01)
            b++;
    return b;
}
```

Argument x ကို unsigned အဖြစ် ကြေညာခြင်းသည် ၎င်းကို ညာဘက်သို့ ရွှေ့သောအခါ အလွတ်ဘစ်များကို လက်မှတ်ဘစ်များဖြင့် မဟုတ်ဘဲ သုညများဖြင့် ဖြည့်မည်ဖြစ်ကြောင်း အာမခံပေးပါသည်၊ ပရိုဂရမ်ကို မည်သည့်စက်ပေါ်တွင် run လုပ်သည်ဖြစ်စေ။

## Assignment Operators ၏ အားသာချက်များ

တိုတောင်းခြင်းထက် လုံးဝခွဲခြားပြီး assignment operators သည် လူများ တွေးခေါ်ပုံနှင့် ပို၍ ကိုက်ညီသည့် အားသာချက်ရှိပါသည်။ ကျွန်ုပ်တို့က "i သို့ ၂ ကို ပေါင်းပါ" သို့မဟုတ် "i ကို ၂ ဖြင့် တိုးပါ" ဟု ပြောတတ်ကြပြီး "i ကို ယူပါ၊ ၂ ကို ပေါင်းပါ၊ ထို့နောက် ရလဒ်ကို i ထဲ ပြန်ထည့်ပါ" ဟု မပြောတတ်ကြပါ။ ထို့ကြောင့် expression i += 2 သည် i = i+2 ထက် ပို၍ သင့်လျော်ပါသည်။

ထို့အပြင် ရှုပ်ထွေးသော expression ဖြစ်သည့် -

```
yyval[yypv[p3+p4] + yypv[p1]] += 2
```

အတွက် assignment operator သည် ကုဒ်ကို ပို၍ နားလည်လွယ်အောင် လုပ်ပေးပါသည်။ အဘယ်ကြောင့်ဆိုသော် ဖတ်သူသည် ရှည်လျားသော expressions နှစ်ခု အမှန်တကယ် တူညီသလား သို့မဟုတ် အဘယ်ကြောင့် မတူညီသနည်းဎ သေချာစွာ စစ်ဆေးရန် မလိုအပ်တော့ပါ။ Assignment operator သည် compiler အား ထိရောက်သော ကုဒ်ထုတ်လုပ်ရန်လည်း ကူညီပေးနိုင်ပါသည်။

## Assignment Expression ၏ တန်ဖိုး

Assignment statement တွင် တန်ဖိုးရှိပြီး expressions များတွင် ပေါ်နိုင်ကြောင်း ကျွန်ုပ်တို့ မြင်ဖူးပြီးဖြစ်သည်။ အသုံးအများဆုံး ဥပမာမှာ -

```c
while ((c = getchar()) != EOF)
    ...
```

အခြား assignment operators များ (+=, -=, စသည်တို့) သည်လည်း expressions များတွင် ပေါ်နိုင်သော်လည်း ဤအသုံးပြုမှုမှာ နည်းပါးပါသည်။

ဤကဲ့သို့သော expressions အားလုံးတွင် assignment expression ၏ အမျိုးအစားသည် ၎င်း၏ ဘယ်ဘက် operand ၏ အမျိုးအစားဖြစ်ပြီး တန်ဖိုးသည် assignment လုပ်ပြီးနောက်ရသော တန်ဖိုးဖြစ်ပါသည်။

### လေ့ကျင့်ခန်း
