# ၂.၄ Declarations (ကြေညာခြင်းများ)

Variable အားလုံးကို အသုံးမပြုမီ ကြေညာရမည်ဖြစ်သော်လည်း အချို့သော ကြေညာချက်များကို အကြောင်းအရာအရ သွယ်ဝိုက်၍ ပြုလုပ်နိုင်ပါသည်။ ကြေညာချက်တစ်ခုသည် အမျိုးအစားတစ်ခုကို သတ်မှတ်ပြီး ထိုအမျိုးအစား၏ variable တစ်ခု သို့မဟုတ် တစ်ခုထက်ပို၍ စာရင်းတစ်ခု ပါဝင်ပါသည် -

```c
int lower, upper, step;
char c, line[1000];
```

Variable များကို မည်သည့်ပုံစံဖြင့်မဆို ကြေညာချက်များကြားတွင် ခွဲဝေနိုင်ပါသည်။ အထက်ပါ စာရင်းများကို အောက်ပါအတိုင်း ရေးနိုင်ပါသည် -

```c
int lower;
int upper;
int step;
char c;
char line[1000];
```

နောက်ပိုင်းပုံစံသည် နေရာပိုယူသော်လည်း နောက်ပိုင်း ပြင်ဆင်မှုများအတွက် ကြေညာချက်တိုင်းတွင် မှတ်ချက် (comment) ထည့်ရန် အဆင်ပြေပါသည်။

## Variable Initialization (ကြိုတင်သတ်မှတ်ခြင်း)

Variable တစ်ခုကို ၎င်း၏ ကြေညာချက်တွင် ကြိုတင်သတ်မှတ် (initialize) နိုင်ပါသည်။ အမည်နောက်တွင် ညီမျှခြင်းသင်္ကေတနှင့် expression တစ်ခု ရှိပါက ထို expression သည် initializer အဖြစ် ဆောင်ရွက်ပါသည် -

```c
char esc = '\\';
int i = 0;
int limit = MAXLINE+1;
float eps = 1.0e-5;
```

မေးခွန်းထုတ်ထားသော variable သည် automatic မဟုတ်ပါက initialization သည် တစ်ကြိမ်သာ ပြုလုပ်ပြီး အယူအဆအရ ပရိုဂရမ် စတင်လုပ်ဆောင်မီ လုပ်ဆောင်ကြောင်း နှင့် initializer သည် constant expression ဖြစ်ရမည်ဖြစ်ပါသည်။ ပြတ်ပြတ်သားသား initialize လုပ်ထားသော automatic variable သည် ၎င်းတည်ရှိသော function သို့မဟုတ် block ထဲသို့ ဝင်သည့်အချိန်တိုင်း initialize ပြုလုပ်ပါသည်။ Initializer သည် မည်သည့် expression မဆို ဖြစ်နိုင်ပါသည်။

External နှင့် static variable များကို default အားဖြင့် သုညအဖြစ် initialize လုပ်ပါသည်။ ပြတ်သားသော initializer မရှိသော automatic variable များတွင် မသတ်မှတ်ထားသော (ဆိုလိုသည်မှာ အမှိုက်) တန်ဖိုးများ ရှိပါသည်။

## const Qualifier

Qualifier const ကို variable မည်သည့်ကြေညာချက်တွင်မဆို အသုံးပြု၍ ၎င်း၏ တန်ဖိုးကို ပြောင်းလဲမည် မဟုတ်ကြောင်း သတ်မှတ်နိုင်ပါသည်။ Array တစ်ခုအတွက် const qualifier သည် ဒြပ်စင်များကို ပြောင်းလဲမည် မဟုတ်ကြောင်း ဆိုလိုပါသည်။

```c
const double e = 2.71828182845905;
const char msg[] = "warning: ";
```

const ကြေညာချက်ကို array arguments များနှင့်လည်း အသုံးပြု၍ function သည် ထို array ကို မပြောင်းလဲကြောင်း ညွှန်ပြနိုင်ပါသည် -

```c
int strlen(const char[]);
```

const ကို ပြောင်းလဲရန် ကြိုးစားပါက ရလဒ်သည် implementation-defined ဖြစ်ပါသည်။

# ၂.၅ Arithmetic Operators (ဂဏန်းတွက်ချက်မှု အော်ပရေတာများ)

Binary arithmetic အော်ပရေတာများမှာ +, -, *, /, နှင့် modulus အော်ပရေတာ % တို့ဖြစ်ပါသည်။ Integer စားခြင်းသည် အပိုင်းကိန်း (fractional part) မည်သည့်အပိုင်းကိုမဆို ဖြတ်တောက်ပစ်ပါသည်။ Expression -

```c
x % y
```

သည် x ကို y ဖြင့် စားသောအခါ ကျန်ကိန်းကို ထုတ်ပေးပြီး x သည် y ဖြင့် အတိအကျ စားနိုင်သောအခါ သုညဖြစ်ပါသည်။ ဥပမာ - နှစ်တစ်နှစ်သည် ၄ ဖြင့် စားလို့ရသော်လည်း ၁၀၀ ဖြင့် မစားနိုင်ပါက leap year (ရက်ထပ်နှစ်) ဖြစ်ပါသည်၊ သို့သော် ၄၀၀ ဖြင့် စားနိုင်သော နှစ်များသည် leap year များဖြစ်ပါသည်။ ထို့ကြောင့် -

```c
if ((year % 4 == 0 && year % 100 != 0) || year % 400 == 0)
    printf("%d is a leap year\n", year);
else
    printf("%d is not a leap year\n", year);
```

% အော်ပရေတာကို float သို့မဟုတ် double တွင် အသုံးမပြုနိုင်ပါ။ / အတွက် ဖြတ်တောက်ခြင်း၏ ဦးတည်ရာနှင့် % အတွက် ရလဒ်၏ လက်မှတ်သည် အနုတ် operands များအတွက် စက်ပေါ်မူတည်ပြီး overflow သို့မဟုတ် underflow တွင် လုပ်ဆောင်သော လုပ်ဆောင်ချက်လည်း ထိုနည်းတူဖြစ်ပါသည်။

Binary + နှင့ - အော်ပရေတာများတွင် တူညီသော precedence (ဦးစားပေးမှု) ရှိပြီး ၎င်းတို့သည် *, /, နှင့ % တို့၏ precedence ထက် နိမ့်ကာ unary + နှင့ - တို့ထက် နိမ့်ပါသည်။ Arithmetic အော်ပရေတာများသည် ဘယ်မှညာသို့ associate (ပေါင်းစည်း) ပါသည်။

ဤအခန်း၏ အဆုံးရှိ ဇယား ၂.၁ တွင် အော်ပရေတာအားလုံး၏ precedence နှင့် associativity ကို အကျဉ်းချုပ် ဖော်ပြထားပါသည်။

# ၂.၆ Relational and Logical Operators (နှိုင်းယှဉ်ခြင်းနှင့် ယုတ္တိဗေဒ အော်ပရေတာများ)

Relational အော်ပရေတာများမှာ -

```
>  >=  <  <=
```

၎င်းတို့အားလုံးတွင် တူညီသော precedence ရှိပါသည်။ ၎င်းတို့၏ အောက်တွင် precedence အရ equality (တူညီမှု) အော်ပရေတာများ ရှိပါသည် -

```
==  !=
```

Relational အော်ပရေတာများတွင် arithmetic အော်ပရေတာများထက် precedence နိမ့်သောကြောင့် i < lim-1 ကဲ့သို့သော expression ကို မျှော်လင့်သည့်အတိုင်း i < (lim-1) အဖြစ် ယူဆပါသည်။

## Logical Operators (&& နှင့် ||)

ပို၍ စိတ်ဝင်စားဖွယ်ကောင်းသည်မှာ logical အော်ပရေတာများ && နှင့် || ဖြစ်ပါသည်။ && သို့မဟုတ် || ဖြင့် ချိတ်ဆက်ထားသော expressions များကို ဘယ်မှညာသို့ တွက်ချက်ပြီး ရလဒ်၏ အမှန် သို့မဟုတ် အမှား သိရှိသည်နှင့်တပြိုင်နက် တွက်ချက်ခြင်းကို ရပ်တန့်ပါသည်။ C ပရိုဂရမ်အများစုတွင် ဤဂုဏ်သတ္တိများအပေါ် မှီခိုအားထားကြပါသည်။

ဥပမာ - အခန်း ၁ တွင် ကျွန်ုပ်တို့ရေးခဲ့သော input function getline မှ loop တစ်ခုမှာ -

```c
for (i=0; i < lim-1 && (c=getchar()) != '\n' && c != EOF; ++i)
    s[i] = c;
```

အက္ခရာအသစ်တစ်ခု မဖတ်မီ ၎င်းကို array s တွင် သိမ်းဆည်းရန် နေရာရှိမရှိ စစ်ဆေးရန် လိုအပ်သောကြောင့် i < lim-1 စစ်ဆေးမှုကို ပထမဦးစွာ လုပ်ရပါမည်။ ထို့အပြင် ဤစစ်ဆေးမှု မအောင်မြင်ပါက အက္ခရာအခြားတစ်ခုကို မဖတ်ရန် လိုအပ်ပါသည်။

အလားတူပင် getchar ကို မခေါ်မီ c ကို EOF နှင့် စစ်ဆေးခြင်းသည် မလိုလားအပ်ပါ။ ထို့ကြောင့် ခေါ်ဆိုမှုနှင့် တန်ဖိုးသတ်မှတ်မှုသည် c ရှိ အက္ခရာကို မစစ်ဆေးမီ ဖြစ်ပေါ်ရပါမည်။

&& ၏ precedence သည် || ထက် မြင့်ပြီး နှစ်ခုစလုံးသည် relational နှင့် equality အော်ပရေတာများထက် နိမ့်သောကြောင့် အောက်ပါကဲ့သို့သော expressions များတွင် -

```c
i < lim-1 && (c=getchar()) != '\n' && c != EOF
```

ထပ်ဆောင်း ကွင်းပိတ်များ မလိုအပ်ပါ။ သို့သော် != ၏ precedence သည် assignment ထက် မြင့်သောကြောင့် -

```c
(c=getchar()) != '\n'
```

တွင် c သို့ တန်ဖိုးသတ်မှတ်ပြီးနောက် '\n' နှင့် နှိုင်းယှဉ်ရန် လိုလားသော ရလဒ်ရရှိစေရန် ကွင်းပိတ်များ လိုအပ်ပါသည်။

## Relational/Logical Expression ၏ တန်ဖိုး

အဓိပ္ပာယ်ဖွင့်ဆိုချက်အရ relational သို့မဟုတ် logical expression ၏ ကိန်းဂဏန်းတန်ဖိုးသည် ဆက်နွယ်မှု အမှန်ဖြစ်ပါက ၁ ဖြစ်ပြီး အမှား ဖြစ်ပါက ၀ ဖြစ်ပါသည်။

## Unary Negation Operator (!)

Unary negation အော်ပရေတာ ! သည် သုညမဟုတ်သော operand ကို ၀ အဖြစ်သို့ ပြောင်းလဲပြီး သုည operand ကို ၁ အဖြစ်သို့ ပြောင်းလဲပါသည်။ ! ၏ အသုံးများသော အသုံးပြုမှုမှာ အောက်ပါကဲ့သို့သော တည်ဆောက်မှုများဖြစ်ပါသည် -

```c
if (!valid)
```

အစား -

```c
if (valid == 0)
```

မည်သည့်ပုံစံက ပိုကောင်းသည်ကို ယေဘူယျ ပြောရန် ခက်ခဲပါသည်။ !valid ကဲ့သို့သော တည်ဆောက်မှုများကို ကောင်းစွာ ဖတ်နိုင်ပါသည် ("if not valid")၊ သို့သော် ပို၍ ရှုပ်ထွေးသော တည်ဆောက်မှုများကို နားလည်ရန် ခက်ခဲနိုင်ပါသည်။
