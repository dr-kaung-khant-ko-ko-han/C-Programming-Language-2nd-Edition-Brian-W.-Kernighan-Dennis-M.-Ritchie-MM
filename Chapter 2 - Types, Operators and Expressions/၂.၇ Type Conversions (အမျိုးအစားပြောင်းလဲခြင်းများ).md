# ၂.၇ Type Conversions (အမျိုးအစားပြောင်းလဲခြင်းများ)

အော်ပရေတာတစ်ခုတွင် မတူညီသော အမျိုးအစားများ၏ operands များရှိသောအခါ ၎င်းတို့ကို စည်းမျဉ်းအနည်းငယ်အရ အများနှင့်သုံးသော အမျိုးအစားတစ်ခုသို့ ပြောင်းလဲပါသည်။ ယေဘုယျအားဖြင့် အလိုအလျောက် ပြောင်းလဲမှုများသည် "ကျဉ်းမြောင်းသော" operand ကို "ကျယ်ပြန့်သော" operand အဖြစ်သို့ အချက်အလက်ဆုံးရှုံးခြင်းမရှိဘဲ ပြောင်းလဲပေးသော ပြောင်းလဲမှုများဖြစ်ပြီး f + i ကဲ့သို့သော expression တွင် integer ကို floating point အဖြစ်သို့ ပြောင်းလဲခြင်းကဲ့သို့ဖြစ်ပါသည်။

Float ကို subscript အဖြစ် အသုံးပြုခြင်းကဲ့သို့ အဓိပ္ပာယ်မရှိသော expressions များကို ခွင့်မပြုပါ။ ပို၍ရှည်လျားသော integer အမျိုးအစားကို ပို၍တိုသော အမျိုးအစားသို့ သို့မဟုတ် floating-point အမျိုးအစားကို integer သို့ သတ်မှတ်ခြင်းကဲ့သို့ အချက်အလက်ဆုံးရှုံးနိုင်သော expressions များသည် သတိပေးချက်ပေါ်နိုင်သော်လည်း တရားမဝင်ခြင်း မဟုတ်ပါ။

## char နှင့် Arithmetic Expressions

char သည် ကိန်းပြည့်ငယ်တစ်ခုသာဖြစ်သောကြောင့် chars များကို arithmetic expressions များတွင် လွတ်လပ်စွာ အသုံးပြုနိုင်ပါသည်။ ဤသည်မှာ အချို့သော အက္ခရာပြောင်းလဲမှုများတွင် များစွာသော လိုက်လျောညီထွေမှုကို ခွင့်ပြုပေးပါသည်။ တစ်ခုမှာ ဂဏန်းကြိုးတစ်ခုကို ၎င်း၏ ကိန်းဂဏန်းညီမျှမှုသို့ ပြောင်းလဲပေးသော function atoi ၏ ရိုးရှင်းသော လုပ်ဆောင်မှုဖြစ်ပါသည်။

```c
/* atoi: convert s to integer */
int atoi(char s[])
{
    int i, n;
    n = 0;
    
    for (i = 0; s[i] >= '0' && s[i] <= '9'; ++i)
        n = 10 * n + (s[i] - '0');
    return n;
}
```

အခန်း ၁ တွင် ဆွေးနွေးခဲ့သည့်အတိုင်း expression -

```c
s[i] - '0'
```

သည် s[i] တွင် သိမ်းဆည်းထားသော အက္ခရာ၏ ကိန်းဂဏန်းတန်ဖိုးကို ပေးပါသည်၊ အဘယ်ကြောင့်ဆိုသော် '0', '1' စသည်တို့၏ တန်ဖိုးများသည် ကပ်လျက်တိုးလာသော အစီအစဉ်တစ်ခု ဖြစ်သောကြောင့်ဖြစ်ပါသည်။

## lower() Function ဥပမာ

char မှ int ပြောင်းလဲခြင်း၏ နောက်ထပ်ဥပမာမှာ function lower ဖြစ်ပြီး ASCII character set အတွက် အက္ခရာတစ်လုံးကို စာလုံးသေးအဖြစ်သို့ ပြောင်းပေးပါသည်။ အက္ခရာသည် စာလုံးကြီး မဟုတ်ပါက lower သည် ၎င်းကို မပြောင်းလဲဘဲ ပြန်ပေးပါသည်။

```c
/* lower: convert c to lower case; ASCII only */
int lower(int c)
{
    if (c >= 'A' && c <= 'Z')
        return c + 'a' - 'A';
    else
        return c;
}
```

ဤသည်မှာ ASCII အတွက် အလုပ်ဖြစ်ပါသည်၊ အဘယ်ကြောင့်ဆိုသော် သက်ဆိုင်ရာ စာလုံးကြီးနှင့် စာလုံးသေးများသည် ကိန်းဂဏန်းတန်ဖိုးများအဖြစ် ပုံသေအကွာအဝေးရှိပြီး အက္ခဘောတစ်ခုစီသည် ကပ်လျက်ရှိသောကြောင့်ဖြစ်ပါသည် -- A နှင့် Z ကြားတွင် အက္ခရာများမှလွဲ၍ ဘာမျှမရှိပါ။ သို့သော် ဤနောက်ဆုံး လေ့လာတွေ့ရှိချက်သည် EBCDIC character set အတွက် မမှန်ကန်ပါ၊ ထို့ကြောင့် ဈကုဒ်သည် EBCDIC တွင် အက္ခရာများထက် ပို၍ ပြောင်းလဲပေးမည်ဖြစ်ပါသည်။

Standard header `<ctype.h>` (နောက်ဆက်တွဲ B တွင် ဖော်ပြထားသည်) သည် character set နှင့် လွတ်လပ်သော စစ်ဆေးမှုများနှင့် ပြောင်းလဲမှုများ ပံ့ပိုးပေးသော functions မိသားစုတစ်ခုကို သတ်မှတ်ပါသည်။ ဥပမာ - function tolower သည် အထက်တွင် ပြသခဲ့သော function lower အတွက် portable အစားထိုးဖြစ်ပါသည်။ အလားတူပင် စစ်ဆေးမှု -

```c
c >= '0' && c <= '9'
```

ကို အောက်ပါဖြင့် အစားထိုးနိုင်ပါသည် -

```c
isdigit(c)
```

ယခုမှစ၍ `<ctype.h>` မှ functions များကို အသုံးပြုပါမည်။

## char မှ int သို့ ပြောင်းလဲခြင်း၏ သိမ်မွေ့သောအချက်

အက္ခရာများကို integers များသို့ ပြောင်းလဲခြင်း၏ သိမ်မွေ့သောအချက်တစ်ခုရှိပါသည်။ ဘာသာစကားသည် char အမျိုးအစား၏ variables များသည် signed သို့မဟုတ် unsigned ပမာဏများ ဖြစ်မဖြစ်ကို မသတ်မှတ်ပါ။ char ကို int သို့ ပြောင်းလဲသောအခါ အနုတ် integer ထုတ်နိုင်ပါသလား? အဖြေသည် စက်မှ စက်သို့ ကွဲပြားပြီး အဆောက်အအုံ၏ ကွဲပြားမှုများကို ထင်ဟပ်စေပါသည်။

အချို့စက်များတွင် ဘယ်ဘက်ဆုံးဘစ်သည် ၁ ဖြစ်သော char ကို အနုတ် integer အဖြစ်သို့ ပြောင်းလဲပါသည် ("sign extension")။ အခြားစက်များတွင် char ကို ဘယ်ဘက်အဆုံးတွင် သုညများ ပေါင်းထည့်ခြင်းဖြင့် int အဖြစ်သို့ မြှင့်တင်ပြီး အမြဲတမ်း အပေါင်းဖြစ်ပါသည်။

C ၏ အဓိပ္ပာယ်ဖွင့်ဆိုချက်သည် စက်၏ standard printing character set ရှိ အက္ခရာမည်သည်မျှ အနုတ်မဖြစ်ကြောင်း အာမခံပြီး ဤအက္ခရာများသည် expressions များတွင် အမြဲတမ်း အပေါင်းပမာဏများ ဖြစ်ပါသည်။ သို့သော် character variables များတွင် သိမ်းဆည်းထားသော ဘစ်ပုံစံများသည် အချို့စက်များတွင် အနုတ်ဖြစ်နိုင်ပြီး အခြားစက်များတွင် အပေါင်းဖြစ်နိုင်ပါသည်။ Portability အတွက် char variables များတွင် အက္ခရာမဟုတ်သော ဒေတာများကို သိမ်းဆည်းမည်ဆိုလျှင် signed သို့မဟုတ် unsigned ကို သတ်မှတ်ပါ။

## Relational နှင့် Logical Expressions များ၏ တန်ဖိုးများ

i > j ကဲ့သို့သော relational expressions နှင့် && နှင့် || ဖြင့် ချိတ်ဆက်ထားသော logical expressions များသည် အမှန်ဖြစ်လျှင် တန်ဖိုး ၁ ရှိပြီး အမှားဖြစ်လျှင် ၀ ရှိကြောင်း သတ်မှတ်ထားပါသည်။ ထို့ကြောင့် assignment -

```c
d = c >= '0' && c <= '9'
```

သည် c သည် ဂဏန်းဖြစ်လျှင် d ကို ၁ သတ်မှတ်ပြီး မဟုတ်လျှင် ၀ သတ်မှတ်ပါသည်။ သို့သော် isdigit ကဲ့သို့သော functions များသည် အမှန်အတွက် သုညမဟုတ်သော တန်ဖိုးမည်သည်ကိုမဆို ပြန်ပေးနိုင်ပါသည်။ if, while, for စသည်တို့၏ စစ်ဆေးမှုအပိုင်းတွင် "အမှန်" ဆိုသည်မှာ "သုညမဟုတ်ခြင်း" ကိုသာ ဆိုလိုသောကြောင့် ဤသည်မှာ ကွာခြားမှု မရှိပါ။

## Implicit Arithmetic Conversions (သွယ်ဝိုက်သော ဂဏန်းပြောင်းလဲမှုများ)

Implicit arithmetic conversions များသည် မျှော်လင့်သည့်အတိုင်း များစွာ အလုပ်ဖြစ်ပါသည်။ ယေဘုယျအားဖြင့် operands နှစ်ခု (binary operator) ယူသော + သို့မဟုတ် * ကဲ့သို့သော အော်ပရေတာတစ်ခုတွင် မတူညီသော အမျိုးအစားများရှိလျှင် လုပ်ဆောင်ချက် မစတင်မီ "နိမ့်သော" အမျိုးအစားကို "မြင့်သော" အမျိုးအစားသို့ မြှင့်တင်ပါသည်။ ရလဒ်သည် integer အမျိုးအစားဖြစ်ပါသည်။

နောက်ဆက်တွဲ A ၏ အပိုင်း ၆ တွင် ပြောင်းလဲမှု စည်းမျဉ်းများကို အတိအကျ ဖော်ပြထားပါသည်။ Unsigned operands များ မရှိပါက အောက်ပါ အလွတ်သဘော စည်းမျဉ်းများသည် လုံလောက်ပါသည် -

- Operand တစ်ခုခုသည် long double ဖြစ်လျှင် အခြားတစ်ခုကို long double သို့ ပြောင်းလဲပါ။
- သို့မဟုတ်ပါက operand တစ်ခုခုသည် double ဖြစ်လျှင် အခြားတစ်ခုကို double သို့ ပြောင်းလဲပါ။
- သို့မဟုတ်ပါက operand တစ်ခုခုသည် float ဖြစ်လျှင် အခြားတစ်ခုကို float သို့ ပြောင်းလဲပါ။
- သို့မဟုတ်ပါက char နှင့် short ကို int သို့ ပြောင်းလဲပါ။
- ထို့နောက် operand တစ်ခုခုသည် long ဖြစ်လျှင် အခြားတစ်ခုကို long သို့ ပြောင်းလဲပါ။

Expression တစ်ခုရှိ floats များကို အလိုအလျောက် double သို့ မပြောင်းလဲကြောင်း သတိပြုပါ။ ဤသည်မှာ မူလအဓိပ္ပာယ်ဖွင့်ဆိုချက်မှ ပြောင်းလဲမှုတစ်ခုဖြစ်ပါသည်။ ယေဘုယျအားဖြင့် `<math.h>` ရှိ သင်္ချာ functions များသည် double precision ကို အသုံးပြုပါမည်။ Float အသုံးပြုရန် အဓိက အကြောင်းရင်းမှာ array အကြီးများတွင် သိုလှောင်မှုကို သက်သာစေရန် သို့မဟုတ် နည်းနည်းဖြစ်သော double-precision arithmetic သည် အထူးကုန်ကျသော စက်များတွင် အချိန်သက်သာစေရန်ဖြစ်ပါသည်။

Unsigned operands များပါဝင်သောအခါ ပြောင်းလဲမှု စည်းမျဉ်းများသည် ပို၍ ရှုပ်ထွေးပါသည်။ ပြဿနာမှာ signed နှင့် unsigned တန်ဖိုးများကြား နှိုင်းယှဉ်မှုများသည် စက်ပေါ်မူတည်သောကြောင့်ဖြစ်ပြီး ၎င်းတို့သည် integer အမျိုးအစားအမျိုးမျိုး၏ အရွယ်အစားများပေါ် မူတည်သောကြောင့်ဖြစ်ပါသည်။

## Assignment များတွင် ပြောင်းလဲမှုများ

Conversions များသည် assignments များတစ်လျှောက် ဖြစ်ပေါ်ပါသည်။ ညာဘက်ခြမ်း၏ တန်ဖိုးကို ဘယ်ဘက်ခြမ်း၏ အမျိုးအစားသို့ ပြောင်းလဲပြီး ၎င်းသည် ရလဒ်၏ အမျိုးအစားဖြစ်ပါသည်။

အက္ခရာကို integer အဖြစ်သို့ အထက်တွင် ဖော်ပြထားသည့်အတိုင်း sign extension ဖြင့် သို့မဟုတ် မပါဘဲ ပြောင်းလဲပါသည်။

ရှည်သော integers များကို တိုသော integers များ သို့မဟုတ် chars များသို့ အလွန်အကျွံ high-order bits များကို ဖယ်ထုတ်ခြင်းဖြင့် ပြောင်းလဲပါသည်။ ထို့ကြောင့် -

```c
int i;
char c;

i = c;
c = i;
```

တွင် c ၏ တန်ဖိုးသည် မပြောင်းလဲပါ။ ဤသည်မှာ sign extension ပါဝင်သည်ဖြစ်စေ မပါဝင်သည်ဖြစ်စေ မှန်ကန်ပါသည်။ သို့သော် assignments များ၏ အစီအစဉ်ကို ပြောင်းပြန်လုပ်ခြင်းသည် အချက်အလက် ဆုံးရှုံးနိုင်ပါသည်။

x သည် float ဖြစ်ပြီး i သည် int ဖြစ်လျှင် x = i နှင့် i = x နှစ်ခုစလုံးသည် ပြောင်းလဲမှုများ ဖြစ်စေပါသည်။ float မှ int သည် အပိုင်းကိန်းအပိုင်းမည်သည်ကိုမဆို ဖြတ်တောက်ပစ်ပါသည်။ Double ကို float သို့ ပြောင်းလဲသောအခါ တန်ဖိုးကို အဝိုင်းချခြင်း သို့မဟုတ် ဖြတ်တောက်ခြင်းသည် implementation dependent ဖြစ်ပါသည်။

## Function Arguments နှင့် Type Conversions

Function call ၏ argument သည် expression တစ်ခုဖြစ်သောကြောင့် arguments များကို functions များသို့ ပေးသောအခါ type conversion လည်း ဖြစ်ပေါ်ပါသည်။ Function prototype မရှိပါက char နှင့် short သည် int ဖြစ်လာပြီး float သည် double ဖြစ်လာပါသည်။ ဤကြောင့် function ကို char နှင့် float ဖြင့် ခေါ်သောအခါတွင်ပင် function arguments များကို int နှင့် double အဖြစ် ကြေညာထားခဲ့ခြင်းဖြစ်ပါသည်။

## Explicit Type Conversions (Cast)

နောက်ဆုံးအနေဖြင့် explicit type conversions များကို မည်သည့် expression တွင်မဆို cast ဟုခေါ်သော unary operator ဖြင့် အတင်းအကျပ် ("coerced") လုပ်နိုင်ပါသည်။ တည်ဆောက်မှုတွင် -

```c
(type name) expression
```

expression ကို အထက်ပါ ပြောင်းလဲမှု စည်းမျဉ်းများဖြင့် အမည်ပေးထားသော အမျိုးအစားသို့ ပြောင်းလဲပါသည်။ Cast ၏ တိကျသော အဓိပ္ပာယ်မှာ expression ကို သတ်မှတ်ထားသော အမျိုးအစား၏ variable တစ်ခုသို့ သတ်မှတ်ထားသကဲ့သို့ ဖြစ်ပြီး ထို့နောက် တည်ဆောက်မှုတစ်ခုလုံး၏ နေရာတွင် အသုံးပြုပါသည်။

ဥပမာ - library routine sqrt သည် double argument တစ်ခုကို မျှော်လင့်ပြီး မတော်တဆ တစ်ခုခုအခြား ကိုင်တွယ်ပါက အဓိပ္ပာယ်မရှိသော အရာထုတ်ပေးပါမည်။ (sqrt ကို `<math.h>` တွင် ကြေညာထားပါသည်။) ထို့ကြောင့် n သည် integer ဖြစ်လျှင် -

```c
sqrt((double) n)
```

ကို အသုံးပြု၍ n ၏ တန်ဖိုးကို sqrt သို့ မပေးမီ double သို့ ပြောင်းလဲနိုင်ပါသည်။ Cast သည် n ၏ တန်ဖိုးကို သင့်လျော်သော အမျိုးအစားတွင် ထုတ်ပေးကြောင်း သတိပြုပါ။ n ကိုယ်တိုင်ကို မပြောင်းလဲပါ။ Cast operator သည် အခြား unary operators များကဲ့သို့ တူညီသော မြင့်မားသော precedence ရှိပြီး ဤအခန်း၏ အဆုံးရှိ ဇယားတွင် အကျဉ်းချုပ် ဖော်ပြထားပါသည်။

Arguments များကို function prototype ဖြင့် ကြေညာထားပါက ပုံမှန်အတိုင်း ဖြစ်သင့်သည့်အတိုင်း function ကို ခေါ်သောအခါ ကြေညာချက်သည် arguments မည်သည်ကိုမဆို အလိုအလျောက် coercion ဖြစ်စေပါသည်။ ထို့ကြောင့် sqrt အတွက် function prototype ပေးထားပါက -

```c
double sqrt(double)
```

ခေါ်ဆိုမှု -

```c
root2 = sqrt(2)
```

သည် integer 2 ကို cast မလိုအပ်ဘဲ double တန်ဖိုး 2.0 အဖြစ်သို့ coerces ပြုလုပ်ပါသည်။

## Pseudo-random Number Generator ဥပမာ

Standard library တွင် pseudo-random number generator ၏ portable implementation တစ်ခုနှင့် seed ကို initialize လုပ်ရန် function တစ်ခု ပါဝင်ပါသည်။ ရှေ့တစ်ခုသည် cast ကို သရုပ်ပြပါသည် -

```c
unsigned long int next = 1;

/* rand: return pseudo-random integer on 0..32767 */
int rand(void)
{
    next = next * 1103515245 + 12345;
    return (unsigned int)(next/65536) % 32768;
}

/* srand: set seed for rand() */
void srand(unsigned int seed)
{
    next = seed;
}
```

# ၂.၈ Increment and Decrement Operators (တိုးခြင်းနှင့် လျှော့ခြင်း အော်ပရေတာများ)

C သည် variables များကို တိုးခြင်းနှင့် လျှော့ခြင်းအတွက် ထူးခြားသော အော်ပရေတာနှစ်ခု ပံ့ပိုးပေးပါသည်။ Increment အော်ပရေတာ ++ သည် ၎င်း၏ operand သို့ ၁ ပေါင်းထည့်ပြီး decrement အော်ပရေတာ -- သည် ၁ နုတ်ပါသည်။ ကျွန်ုပ်တို့က ++ ကို variables များတိုးရန် မကြာခဏ အသုံးပြုခဲ့ပြီးဖြစ်ပါသည် -

```c
if (c == '\n')
    ++nl;
```

## Prefix နှင့် Postfix Operators

ထူးခြားသော ရှုထောင့်မှာ ++ နှင့် -- ကို prefix operators (variable ရှေ့တွင်၊ ++n ကဲ့သို့) သို့မဟုတ် postfix operators (variable နောက်တွင်: n++) အဖြစ် အသုံးပြုနိုင်ပါသည်။ နှစ်ခုစလုံးတွင် အကျိုးသက်ရောက်မှုမှာ n ကို တိုးခြင်းဖြစ်ပါသည်။

သို့သော် expression ++n သည် n ကို ၎င်း၏ တန်ဖိုးမသုံးမီ တိုးပြီး n++ သည် n ကို ၎င်း၏ တန်ဖိုး အသုံးပြုပြီးနောက် တိုးပါသည်။ ဆိုလိုသည်မှာ တန်ဖိုးကို အသုံးပြုနေသော အကြောင်းအရာတွင် အကျိုးသက်ရောက်မှုသာမဟုတ်ဘဲ ++n နှင့် n++ သည် ကွဲပြားပါသည်။ n သည် ၅ ဖြစ်လျှင် -

```c
x = n++;
```

သည် x ကို ၅ သတ်မှတ်သော်လည်း -

```c
x = ++n;
```

သည် x ကို ၆ သတ်မှတ်ပါသည်။ နှစ်ခုစလုံးတွင် n သည် ၆ ဖြစ်လာပါသည်။

Increment နှင့် decrement အော်ပရေတာများကို variables များတွင်သာ အသုံးပြုနိုင်ပါသည်။ (i+j)++ ကဲ့သို့သော expression သည် တရားမဝင်ပါ။

## squeeze() Function ဥပမာ

တန်ဖိုး မလိုချင်သော အကြောင်းအရာတွင် တိုးခြင်း အကျိုးသက်ရောက်မှုသာ လိုချင်သည့်အခါ -

```c
if (c == '\n')
    nl++;
```

prefix နှင့် postfix သည် တူညီပါသည်။ သို့သော် တစ်ခု သို့မဟုတ် အခြားတစ်ခုကို အထူးတလည် ခေါ်သုံးသော အခြေအနေများရှိပါသည်။ ဥပမာ - string s မှ အက္ခရာ c ၏ ပေါ်လာမှုအားလုံးကို ဖယ်ရှားပေးသော function squeeze(s,c) ကို စဉ်းစားကြည့်ပါ။

```c
/* squeeze: delete all c from s */
void squeeze(char s[], int c)
{
    int i, j;
    for (i = j = 0; s[i] != '\0'; i++)
        if (s[i] != c)
            s[j++] = s[i];
    s[j] = '\0';
}
```

c မဟုတ်သော တစ်ခုချင်းစီ ပေါ်လာသည့်အခါ ၎င်းကို လက်ရှိ j အနေအထားသို့ ကူးယူပြီး ထိုအချိန်မှသာ j ကို နောက် အက္ခရာအတွက် အဆင်သင့်ဖြစ်အောင် တိုးပါသည်။ ဤသည်မှာ အောက်ပါနှင့် အတိအကျ ညီမျှပါသည် -

```c
if (s[i] != c) {
    s[j] = s[i];
    j++;
}
```

## strcat() Function ဥပမာ

အလားတူ တည်ဆောက်မှု၏ နောက်ထပ်ဥပမာမှာ အခန်း ၁ တွင် ကျွန်ုပ်တို့ရေးခဲ့သော getline function မှ လာပြီး အောက်ပါကို -

```c
if (c == '\n') {
    s[i] = c;
    ++i;
}
```

ပို၍ ကျစ်လစ်သော အောက်ပါဖြင့် အစားထိုးနိုင်ပါသည် -

```c
if (c == '\n')
    s[i++] = c;
```

တတိယဥပမာအနေဖြင့် string t ကို string s ၏ အဆုံးသို့ ချိတ်ဆက်ပေးသော standard function strcat(s,t) ကို စဉ်းစားကြည့်ပါ။ strcat သည် s တွင် ပေါင်းစပ်မှုကို ထည့်ရန် လုံလောက်သော နေရာရှိကြောင်း ယူဆထားပါသည်။ ကျွန်ုပ်တို့ရေးခဲ့သည့်အတိုင်း strcat သည် တန်ဖိုးမရှိပါ။ Standard library ဗားရှင်းသည် ရလဒ် string သို့ pointer တစ်ခု ပြန်ပေးပါသည်။

```c
/* strcat: concatenate t to end of s; s must be big enough */
void strcat(char s[], char t[])
{
    int i, j;
    i = j = 0;
    
    while (s[i] != '\0')    /* find end of s */
        i++;
    
    while ((s[i++] = t[j++]) != '\0')  /* copy t */
        ;
}
```

အဖွဲ့ဝင်တစ်ခုစီကို t မှ s သို့ ကူးယူသောအခါ postfix ++ ကို i နှင့် j နှစ်ခုစလုံးတွင် အသုံးပြု၍ loop မှတဆင့် နောက်တစ်ကြိမ်အတွက် ၎င်းတို့ အနေအထားတွင်ရှိကြောင်း သေချာစေပါသည်။
