အခန်း ၂ - အမျိုးအစားများ၊ အော်ပရေတာများနှင့် ခိုင်းစေချက်များ

ပရိုဂရမ်တစ်ခုတွင် ကိုင်တွယ်အသုံးပြုသော အခြေခံဒေတာအရာဝတ္ထုများမှာ ကိန်းရှင်များနှင့် ကိန်းသေများဖြစ်ပါသည်။ ကြေငြာချက်များသည် အသုံးပြုမည့်ကိန်းရှင်များကို စာရင်းပြုစုပြီး ၎င်းတို့၏ အမျိုးအစားနှင့် ကနဦးတန်ဖိုးများကို ဖော်ပြပေးပါသည်။ အော်ပရေတာများသည် ၎င်းတို့ကို မည်သို့လုပ်ဆောင်ရမည်ကို သတ်မှတ်ပေးပါသည်။ ခိုင်းစေချက်များသည် ကိန်းရှင်များနှင့် ကိန်းသေများကို ပေါင်းစပ်၍ တန်ဖိုးအသစ်များ ထုတ်လုပ်ပေးပါသည်။ အရာဝတ္ထုတစ်ခု၏ အမျိုးအစားသည် ၎င်းပိုင်ဆိုင်နိုင်သော တန်ဖိုးများနှင့် ၎င်းအပై လုပ်ဆောင်နိုင်သော လုပ်ငန်းများကို သတ်မှတ်ပေးပါသည်။ ဤအခန်းတွင် အဆိုပါ အခြေခံအရာများကို ဆွေးနွေးထားပါသည်။

ANSI စံသတ်မှတ်ချက်သည် အခြေခံအမျိုးအစားများနှင့် ခိုင်းစေချက်များတွင် အသေးစိတ်ပြောင်းလဲမှုများနှင့် ဖြည့်စွက်မှုများစွာကို ပြုလုပ်ထားပါသည်။ ယခုအခါ ကိန်းပြည့်အမျိုးအစားအားလုံးတွင် signed နှင့် unsigned ပုံစံများရှိပြီး၊ unsigned ကိန်းသေများနှင့် hexadecimal စာလုံးကိန်းသေများအတွက် သင်္ကေတများလည်း ရှိပါသည်။ ဒသမကိန်းလုပ်ငန်းများကို single precision ဖြင့် လုပ်ဆောင်နိုင်ပြီး၊ တိုးချဲ့ တိကျမှုအတွက် long double အမျိုးအစားလည်း ရှိပါသည်။ စာကြောင်းကိန်းသေများကို compile အချိန်တွင် ပေါင်းစပ်နိုင်ပါသည်။ Enumeration များသည် ဘာသာစကား၏ တစ်စိတ်တစ်ပိုင်းဖြစ်လာပြီး၊ ကြာရှည်စွာတည်ရှိခဲ့သော လုပ်ဆောင်ချက်ကို တရားဝင်ဖြစ်စေပါသည်။ အရာဝတ္ထုများကို const အဖြစ် ကြေငြာနိုင်ပြီး၊ ၎င်းတို့အား ပြောင်းလဲခြင်းမှ ကာကွယ်ပေးပါသည်။ ပိုမိုကျယ်ပြန့်သော အမျိုးအစားများကို ကိုင်တွယ်ရန် ဂဏန်းတွက်ချက်မှုအမျိုးအစားများကြား အလိုအလျောက် အတင်းအကျပ်ပြောင်းလဲမှုဆိုင်ရာ စည်းမျဉ်းများကို တိုးချဲ့ထားပါသည်။

## ၂.၁ ကိန်းရှင်အမည်များ

အခန်း ၁ တွင် ပြောမထားပါသော်လည်း၊ ကိန်းရှင်များနှင့် သင်္ကေတကိန်းသေများ၏ အမည်များတွင် ကန့်သတ်ချက်အချို့ရှိပါသည်။ အမည်များသည် စာလုံးများနှင့် ဂဏန်းများဖြင့် ဖွဲ့စည်းထားပြီး၊ ပထမစာလုံးမှာ letter ဖြစ်ရမည်။ underscore ``_'' သည် စာလုံးတစ်လုံးအဖြစ် သတ်မှတ်ပြီး၊ ရှည်လျားသော ကိန်းရှင်အမည်များ၏ ဖတ်ရှုနိုင်မှုကို တိုးတက်စေရန် တစ်ခါတစ်ရံ အသုံးဝင်ပါသည်။ သို့သော် library routine များသည် များသောအားဖြင့် ထိုသို့သော အမည်များကို အသုံးပြုသောကြောင့် ကိန်းရှင်အမည်များကို underscore ဖြင့် မစရန် အကြံပြုပါသည်။ စာလုံးကြီးနှင့် စာလုံးသغ်များသည် ကွဲပြားပြီး၊ x နှင့် X သည် အမည်နှစ်ခုဖြစ်ပါသည်။ ရိုးရာ C လက်တွေ့အားဖြင့် ကိန်းရှင်အမည်များအတွက် စာလုံးသေးများကိုသုံးပြီး၊ သင်္ကေတကိန်းသေများအတွက် စာလုံးကြီးအားလုံးကို သုံးပါသည်။

အတွင်းပိုင်းအမည်တစ်ခု၏ အနည်းဆုံး ပထမ ၃၁ စာလုံးသည် အရေးပါပါသည်။ function အမည်များနှင့် ပြင်ပကိန်းရှင်များအတွက်မူ၊ ၃၁ ထက် နည်းနိုင်ပါသည်၊ အကြောင်းမှာ ပြင်ပအမည်များကို ဘာသာစကား၏ ထိန်းချုပ်မှုမရှိသော assembler များနှင့် loader များက အသုံးပြုနိုင်သောကြောင့်ဖြစ်ပါသည်။ ပြင်ပအမည်များအတွက် စံသတ်မှတ်ချက်သည် ၆ စာလုံးနှင့် စာလုံးအကြီးအသေးတစ်မျိုးတည်းအတွက်သာ ထူးခြားမှုကို အာမခံပေးပါသည်။ if၊ else၊ int၊ float စသည့် keyword များသည် သီးသန့်ထားရှိပြီး ကိန်းရှင်အမည်အဖြစ် မသုံးနိုင်ပါ။ ၎င်းတို့သည် စာလုံးသေးဖြင့် ဖြစ်ရမည်။

ကိန်းရှင်အမည်များကို ၎င်း၏ရည်ရွယ်ချက်နှင့် ဆက်စပ်ပြီး၊ စာရိုက်ရာတွင် ရှုပ်ထွေးခြင်း မရှိစေရန် ရွေးချယ်သင့်ပါသည်။ ကျွန်ုပ်တို့သည် local ကိန်းရှင်များအတွက်၊ အထူးသဖြင့် loop index များအတွက် အမည်တိုများကို အသုံးပြုပြီး၊ ပြင်ပကိန်းရှင်များအတွက် အမည်ရှည်များကို အသုံးပြုတတ်ပါသည်။

## ၂.၂ ဒေတာအမျိုးအစားများနှင့် အရွယ်အစားများ

C တွင် အခြေခံဒေတာအမျိုးအစားအနည်းငယ်သာ ရှိပါသည်:

```
char - byte တစ်ခု၊ local စာလုံးစနစ်ရှိ စာလုံးတစ်လုံးကို သိမ်းဆည်းနိုင်သည်

int - ကိန်းပြည့်တစ်ခု၊ များသောအားဖြင့် host စက်ရှိ ကိန်းပြည့်များ၏ 
သဘာဝအရွယ်အစားကို ထင်ဟပ်သည်

float - single-precision ဒသမကိန်း

double - double-precision ဒသမကိန်း
```

ထို့အပြင် ဤအခြေခံအမျိုးအစားများတွင် အသုံးပြုနိုင်သော qualifier အများအပြားရှိပါသည်။ short နှင့် long သည် ကိန်းပြည့်များတွင် အသုံးပြုပါသည်:

```
short int sh;

long int counter;
```

ဤကြေငြာချက်များတွင် int ဟူသော စကားလုံးကို ချန်လှပ်နိုင်ပြီး၊ များသောအားဖြင့် ချန်လှပ်လေ့ရှိပါသည်။

ရည်ရွယ်ချက်မှာ short နှင့် long သည် လက်တွေ့ကျသော နေရာတွင် ကိန်းပြည့်များ၏ မတူညီသော အရှည်များကို ပံ့ပိုးပေးရန်ဖြစ်ပြီး၊ int သည် ပုံမှန်အားဖြင့် စက်အတွက် သဘာဝအရွယ်အစားဖြစ်ပါသည်။ short သည် များသောအားဖြင့် ၁၆ bit ရှည်ပြီး၊ int သည် ၁၆ သို့မဟုတ် ၃၂ bit ဖြစ်ပါသည်။ compiler တစ်ခုချင်းစီသည် ကိုယ့်စက်ပစ္စည်းအတွက် သင့်လျော်သော အရွယ်အစားများကို လွတ်လပ်စွာ ရွေးချယ်နိုင်ပြီး၊ short နှင့် int များသည် အနည်းဆုံး ၁၆ bit၊ long များသည် အနည်းဆုံး ၃၂ bit ဖြစ်ရမည်ဟူသော ကန့်သတ်ချက်သာ ရှိပြီး၊ short သည် int ထက် ရှည်မဖြစ်ရ၊ int သည် long ထက် ရှည်မဖြစ်ရပါ။

signed သို့မဟုတ် unsigned qualifier သည် char သို့မဟုတ် မည်သည့်ကိန်းပြည့်ကိုမဆို အသုံးပြုနိုင်ပါသည်။ unsigned ဂဏန်းများသည် အမြဲ အပေါင်း သို့မဟုတ် သုည ဖြစ်ပြီး၊ 2n modulo (n သည် အမျိုးအစားရှိ bit အရေအတွက်) ဂဏန်းသင်္ချာနိယာမများကို လိုက်နာပါသည်။ ဥပမာ၊ char များသည် ၈ bit ဖြစ်လျှင်၊ unsigned char ကိန်းရှင်များသည် ၀ မှ ၂၅၅ အထိ တန်ဖိုးရှိပြီး၊ signed char များသည် -၁၂၈ မှ ၁၂၇ အထိ တန်ဖိုးရှိပါသည် (two's complement စက်တွင်)။ plain char များသည် signed သို့မဟုတ် unsigned ဖြစ်ခြင်းမှာ စက်အပေါ် မူတည်ပြီး၊ ပုံနှိပ်နိုင်သော စာလုံးများသည် အမြဲ အပေါင်းဖြစ်ပါသည်။

long double အမျိုးအစားသည် တိုးချဲ့ တိကျမှုရှိ ဒသမကိန်းကို သတ်မှတ်ပေးပါသည်။ ကိန်းပြည့်များကဲ့သို့ပင်၊ ဒသမကိန်းအရာဝတ္ထုများ၏ အရွယ်အစားများမှာ implementation-defined ဖြစ်ပြီး၊ float၊ double နှင့် long double တို့သည် အရွယ်အစားတစ်ခု၊ နှစ်ခု သို့မဟုတ် သုံးခုကို ကိုယ်စားပြုနိုင်ပါသည်။

စံ header များဖြစ်သော <limits.h> နှင့် <float.h> တွင် ဤအရွယ်အစားအားလုံးအတွက် သင်္ကေတကိန်းသေများနှင့်အတူ စက်နှင့် compiler ၏ အခြားဂုဏ်သတ္တိများ ပါဝင်ပါသည်။ ၎င်းတို့ကို Appendix B တွင် ဆွေးနွေးထားပါသည်။

လေ့ကျင့်ခန်း ၂-၁။ စံ header များမှ သင့်လျော်သော တန်ဖိုးများကို ပုံနှိပ်ခြင်းဖြင့်နှင့် တိုက်ရိုက်တွက်ချက်ခြင်းဖြင့် char၊ short၊ int နှင့် long ကိန်းရှင်များ၊ signed နှင့် unsigned နှစ်မျိုးစလုံး၏ အပိုင်းအခြားများကို ဆုံးဖြတ်သော ပရိုဂရမ်တစ်ခု ရေးပါ။ သင်ကိုယ်တိုင် တွက်ချက်လျှင် ပိုခက်ခဲသည် - မတူညီသော ဒသမကိန်းအမျိုးအစားများ၏ အပိုင်းအခြားများကို ဆုံးဖြတ်ပါ။

## ၂.၃ ကိန်းသေများ

၁၂၃၄ ကဲ့သို့သော ကိန်းပြည့်ကိန်းသေတစ်ခုသည် int ဖြစ်ပါသည်။ long ကိန်းသေတစ်ခုကို အဆုံး l (ell) သို့မဟုတ် L ဖြင့် ရေးပြီး၊ ၁၂၃၄၅၆၇၈၉L ကဲ့သို့ဖြစ်သည်။ int တွင် မဝင်နိုင်လောက်အောင် ကြီးသော ကိန်းပြည့်ကိန်းသေကိုလည်း long အဖြစ် သတ်မှတ်ပါသည်။ Unsigned ကိန်းသေများကို အဆုံး u သို့မဟုတ် U ဖြင့် ရေးပြီး၊ ul သို့မဟုတ် UL ဆက်ရန်သည် unsigned long ကို ညွှန်ပြပါသည်။

ဒသမကိန်းကိန်းသေများတွင် ဒသမအမှတ် (၁၂၃.၄) သို့မဟုတ် ထပ်ညွှန်း (၁e-၂) သို့မဟုတ် နှစ်ခုစလုံး ပါဝင်ပြီး၊ ၎င်းတို့၏ အမျိုးအစားမှာ double ဖြစ်ပါသည်၊ ဆက်ရန်မပါလျှင်။ f သို့မဟုတ် F ဆက်ရန်သည် float ကိန်းသေကို ညွှန်ပြပြီး၊ l သို့မဟုတ် L သည် long double ကို ညွှန်ပြပါသည်။

ကိန်းပြည့်တစ်ခု၏ တန်ဖိုးကို ဒဿမအစား octal သို့မဟုတ် hexadecimal ဖြင့် သတ်မှတ်နိုင်ပါသည်။ ကိန်းပြည့်ကိန်းသေတွင် ရှေ့တွင် ၀ (သုည) ပါလျှင် octal ကို ဆိုလိုပြီး၊ ၀x သို့မဟုတ် ၀X ပါလျှင် hexadecimal ကို ဆိုလိုပါသည်။ ဥပမာ၊ ဒဿမ ၃၁ ကို octal တွင် ၀၃၇ အဖြစ်နှင့် hex တွင် ၀x၁f သို့မဟုတ် ၀x၁F အဖြစ် ရေးနိုင်ပါသည်။ Octal နှင့် hexadecimal ကိန်းသေများကိုလည်း L ဖြင့် long ဖြစ်စေရန်နှင့် U ဖြင့် unsigned ဖြစ်စေရန် သုံးနိုင်ပါသည် - ၀XFUL သည် ဒဿမ ၁၅ တန်ဖိုးရှိသော unsigned long ကိန်းသေဖြစ်ပါသည်။

စာလုံးကိန်းသေတစ်ခုသည် ကိန်းပြည့်တစ်ခုဖြစ်ပြီး၊ 'x' ကဲ့သို့ single quote အတွင်းရှိ စာလုံးတစ်လုံးအဖြစ် ရေးပါသည်။ စာလုံးကိန်းသေ၏ တန်ဖိုးမှာ စက်၏ စာလုံးစနစ်ရှိ စာလုံး၏ ဂဏန်းတန်ဖိုးဖြစ်ပါသည်။ ဥပမာ၊ ASCII စာလုံးစနစ်တွင် စာလုံးကိန်းသေ '၀' သည် ၄၈ တန်ဖိုးရှိပြီး၊ ဂဏန်းတန်ဖိုး ၀ နှင့် မသက်ဆိုင်ပါ။ စာလုံးစနစ်အပေါ် မူတည်သော ၄၈ ကဲ့သို့သော ဂဏန်းတန်ဖိုးအစား '၀' ကို ရေးလျှင်၊ ပရိုဂရမ်သည် ထိုတန်ဖိုးနှင့် လွတ်လပ်ပြီး ဖတ်ရှုရလွယ်ကူပါသည်။ စာလုံးကိန်းသေများသည် အခြားကိန်းပြည့်များကဲ့သို့ ဂဏန်းတွက်ချက်မှုများတွင် ပါဝင်နိုင်သော်လည်း၊ ၎င်းတို့ကို အခြားစာလုံးများနှင့် နှိုင်းယှဉ်ခြင်းတွင် အများဆုံး အသုံးပြုပါသည်။

အချို့သောစာလုံးများကို စာလုံးကိန်းသေများနှင့် စာကြောင်းကိန်းသေများတွင် \n (newline) ကဲ့သို့သော escape sequence များဖြင့် ကိုယ်စားပြုနိုင်ပြီး၊ ဤ sequence များသည် စာလုံးနှစ်လုံးကဲ့သို့ ရေးသော်လည်း တစ်လုံးသာ ကိုယ်စားပြုပါသည်။ ထို့အပြင်၊ အလိုအလျောက် byte-sized bit pattern တစ်ခုကို

```
'\ooo'
```

ဖြင့် သတ်မှတ်နိုင်ပါသည်၊ ooo သည် octal ဂဏန်း (၀...၇) တစ်လုံးမှ သုံးလုံးဖြစ်သည်၊ သို့မဟုတ်

```
'\xhh'
```

ဖြင့် သတ်မှတ်နိုင်ပါသည်၊ hh သည် hexadecimal ဂဏန်း (၀...၉၊ a...f၊ A...F) တစ်လုံး သို့မဟုတ် ပို၍ ဖြစ်ပါသည်။ ထို့ကြောင့် ကျွန်ုပ်တို့သည်

```
#define VTAB '\013' /* ASCII vertical tab */
#define BELL '\007' /* ASCII bell character */ 
```

သို့မဟုတ် hexadecimal တွင်၊

```
#define VTAB '\xb' /* ASCII vertical tab */
#define BELL '\x7' /* ASCII bell character */ 
```

Escape sequence များ၏ အပြည့်အစုံစာရင်းမှာ

|\a|alert (bell) character|\|backslash|
|---|---|---|---|
|\b|backspace|\ ?|question mark|
|\f|formfeed|\ '|single quote|
|\n|newline|\ "|double quote|
|\r|carriage return|\ooo|octal number|
|\t|horizontal tab|\xhh|hexadecimal number|
|\v|vertical tab|||

စာလုံးကိန်းသေ '\0' သည် တန်ဖိုးသုည၊ null character ကို ကိုယ်စားပြုပါသည်။ '\0' ကို အချို့သော ခိုင်းစေချက်၏ စာလုံးသဘောသဘာဝကို အလေးပေးရန် ၀ အစား မကြာခဏ ရေးသော်လည်း ဂဏန်းတန်ဖိုးမှာ ၀ သာဖြစ်ပါသည်။

ကိန်းသေခိုင်းစေချက်တစ်ခုသည် ကိန်းသေများသာ ပါဝင်သော ခိုင်းစေချက်ဖြစ်ပါသည်။ ထိုသို့သော ခိုင်းစေချက်များကို run-time အစား compilation အတွင်း တွက်ချက်နိုင်ပြီး၊ ထို့ကြောင့် ကိန်းသေတစ်ခု ဖြစ်ပေါ်နိုင်သော မည်သည့်နေရာတွင်မဆို အသုံးပြုနိုင်ပါသည်၊ ဥပမာ

```
#define MAXLINE 1000
char line[MAXLINE+1];
```

သို့မဟုတ်

```
#define LEAP 1 /* in leap years */
int days[31+28+LEAP+31+30+31+30+31+31+30+31+30+31];
```

စာသားကိန်းသေ (string constant) သို့မဟုတ် စာသားတန်ဖောက (string literal) ဆိုသည်မှာ နှစ်ထပ်ကွပ်ကွင်းများဖြင့် ဝန်းရံထားသော သုည သို့မဟုတ် သုညထက်ပို သော စာလုံးများ၏ အစဉ်အလာဖြစ်သည်။ ဥပမာ

```
"I am a string"
```

သို့မဟုတ်

"" /* စာလုံးမဲ့ စာသား */

ကွပ်ကွင်းများသည် စာသား၏ အစိတ်အပိုင်းမဟုတ်ည၊ ၎င်းကို ခွဲခြားသတ်မှတ်ရန်သာ အသုံးပြုသည်။ စာလုံးကိန်းသေများတွင် အသုံးပြုသော escape sequence များကို စာသားများတွင်လည်း အသုံးပြုနိုင်သည်။ \" သည် နှစ်ထပ်ကွပ်ကွင်းစာလုံးကို ကိုယ်စားပြုသည်။ စာသားကိန်းသေများကို compile လုပ်ချိန်တွင် ပေါင်းစပ်နိုင်သည်-

"hello, " "world"

သည်

"hello, world"

နှင့် ညီမျှသည်။
ဤနည်းလမ်းသည် ရှည်လျားသော စာသားများကို source code မျဉ်းများစွာတွင် ခွဲ၍ရေးရန် အသုံးဝင်သည်။

နည်းပညာအရ၊ စာသားကိန်းသေသည် စာလုံးများ၏ array တစ်ခုဖြစ်သည်။ စာသား၊ အတွင်းပိုင်း ကိုယ်စားပြုမှုတွင် အဆုံးတွင် null စာလုံး '\0' ရှိသည်။ ထို့ကြောင့် လိုအပ်သော ရုပ်ပိုင်းဆိုင်ရာ သိမ်းဆည်းမှုနေရာသည် ကွပ်ကွင်းများကြားတွင် ရေးထားသော စာလုံးအရေအတွက်ထက် တစ်ခု ပို၍ ရှိသည်။ ဤကိုယ်စားပြုမှုက စာသားသည် မည်မျှရှည်လျားနိုင်သည်ကို ကန့်သတ်ချက်မရှိကြောင်း ဆိုလိုသည်။ သို့သော် program များသည် ၎င်း၏ အလျားကို ဆုံးဖြတ်ရန် စာသားတစ်ခုလုံးကို scan လုပ်ရမည်။ standard library function ဖြစ်သော strlen(s) သည် ၎င်း၏ စာလုံးစာသား argument s ၏ အလျားကို ပြန်ပေးသည်၊ terminal '\0' ကို မထည့်တွက်ပါ။ ကျွန်ုပ်တို့၏ ဗားရှင်းမှာ-

```
/* strlen: s ၏ အလျားကို ပြန်ပေးသည် */ int strlen(char s[]) { int i; while (s[i] != '\0') ++i; return i; }
```

strlen နှင့် အခြားသော string function များကို standard header <string.h> တွင် ကြေညာထားသည်။

စာလုံးကိန်းသေနှင့် စာလုံးတစ်လုံးတည်း ပါဝင်သော စာသားကြား ခွဲခြားသတ်မှတ်ရာတွင် သတိထားပါ- 'x' သည် "x" နှင့် မတူပါ။ ပထမတစ်ခုသည် integer ဖြစ်ပြီး၊ machine ၏ character set တွင် x စာလုံး၏ ဂဏန်းတန်ဖိုးကို ထုတ်ပေးရန် အသုံးပြုသည်။ နောက်တစ်ခုသည် စာလုံးတစ်လုံး (x စာလုံး) နှင့် '\0' ပါဝင်သော စာလုံးများ၏ array ဖြစ်သည်။

နောက်ထပ် ကိန်းသေအမျိုးအစားတစ်ခုမှာ enumeration constant ဖြစ်သည်။ Enumeration ဆိုသည်မှာ ကိန်းသေ integer တန်ဖိုးများ၏ စာရင်းဖြစ်သည်၊ ဥပမာ-

```
enum boolean { NO, YES };
```

enum တွင် ပထမအမည်သည် တန်ဖိုး 0 ရှိပြီး၊ နောက်တစ်ခုသည် 1 ရှိသည်၊ အစရှိသဖြင့်၊ သီးခြားတန်ဖိုးများ သတ်မှတ်မထားလျှင်။ တန်ဖိုးများအားလုံးကို သတ်မှတ်မထားလျှင်၊ သတ်မှတ်မထားသော တန်ဖိုးများသည် နောက်ဆုံးသတ်မှတ်ထားသော တန်ဖိုးမှ ဆက်လက်တိုးပွားသည်၊ ဤဥပမာနှစ်ခုတွင် ဒုတိယတစ်ခုကဲ့သို့-

```
enum escapes { BELL = '\a', BACKSPACE = '\b', TAB = '\t', NEWLINE = '\n', VTAB = '\v', RETURN = '\r' }; enum months { JAN = 1, FEB, MAR, APR, MAY, JUN, JUL, AUG, SEP, OCT, NOV, DEC }; /* FEB = 2, MAR = 3, စသည်ဖြင့် */
```

မတူညီသော enumeration များတွင် အမည်များသည် ကွဲပြားရမည်။ တူညီသော enumeration တွင် တန်ဖိုးများသည် ကွဲပြားရန် မလိုအပ်ပါ။
Enumeration များသည် ကိန်းသေတန်ဖိုးများကို အမည်များနှင့် ဆက်စပ်ရန် အဆင်ပြေသော နည်းလမ်းကို ပေးသည်၊ #define ၏ အစားထိုးတစ်ခုဖြစ်ပြီး တန်ဖိုးများကို သင့်အတွက် အလိုအလျောက် ထုတ်ပေးနိုင်သည့် အားသာချက်ရှိသည်။ enum အမျိုးအစားများ၏ variable များကို ကြေညာနိုင်သော်လည်း၊ compiler များသည် သင် variable တွင် သိမ်းဆည်းထားသော တန်ဖိုးသည် enumeration အတွက် တရားဝင်တန်ဖိုးဖြစ်သည်ကို စစ်ဆေးရန် မလိုအပ်ပါ။ မည်သို့ပင်ဆိုစေကာမူ၊ enumeration variable များသည် စစ်ဆေးခွင့်အခွင့်အလမ်းကို ပေးသည်၊ ထို့ကြောင့် #define များထက် ပိုကောင်းလေ့ရှိသည်။ ထို့အပြင်၊ debugger သည် enumeration variable များ၏ တန်ဖိုးများကို ၎င်းတို့၏ သင်္ကေတပုံစံဖြင့် print ထုတ်နိုင်သည်။
