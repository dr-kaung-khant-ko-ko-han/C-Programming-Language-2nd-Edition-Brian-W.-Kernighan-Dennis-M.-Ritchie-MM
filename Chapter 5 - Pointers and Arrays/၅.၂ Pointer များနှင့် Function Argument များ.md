## ၅.၂ Pointer များနှင့် Function Argument များ

C သည် argument များကို function များသို့ value ဖြင့် ပို့သောကြောင့် ခေါ်ဆိုထားသော function က ခေါ်ဆိုသည့် function ရှိ variable တစ်ခုကို တိုက်ရိုက်ပြောင်းလဲပေးနိုင်သော နည်းလမ်းမရှိပါ။ ဥပမာအားဖြင့် sorting routine တစ်ခုသည် အစီအစဉ်မမှန်သော argument နှစ်ခုကို swap ဟုခေါ်သော function ဖြင့် လဲလှယ်နိုင်သည်။ ဤသို့ရေးရုံမျှဖြင့် မလုံလောက်ပါ

`swap(a, b);`

swap function ကို ဤသို့ သတ်မှတ်ထားလျှင်

```
void swap(int x, int y) /* မှားသည် */

{
	int temp;
	temp = x;
	x = y;
	y = temp;
}
```

Call by value ကြောင့် swap သည် ၎င်းကိုခေါ်ဆိုသော routine ရှိ argument a နှင့် b တို့ကို ထိခိုက်စေ၍မရပါ။ အထက်ပါ function သည် a နှင့် b တို့၏ copy များကို လဲလှယ်ရုံသာဖြစ်သည်။

လိုချင်သော အကျိုးသက်ရောက်မှုကို ရရှိရန် နည်းလမ်းမှာ ခေါ်ဆိုသော program က ပြောင်းလဲလိုသော တန်ဖိုးများသို့ pointer များကို ပို့ပေးရမည်:

```
swap(&a, &b);
```

Operator & သည် variable တစ်ခု၏ address ကို ထုတ်ပေးသောကြောင့် &a သည် a သို့ point လုပ်သော pointer ဖြစ်သည်။ swap ကိုယ်တိုင်တွင် parameter များကို pointer များအဖြစ် declare လုပ်ထားပြီး operand များကို ၎င်းတို့မှတဆင့် သွယ်ဝိုက်၍ ရယူအသုံးပြုသည်။

```
void swap(int *px, int *py) /* *px နှင့် *py ကို လဲလှယ်သည် */

{
	int temp;
	temp = *px;
	*px = *py;
	*py = temp;
}

```

ပုံဖြင့်ဖော်ပြလျှင်:

Pointer argument များသည် function တစ်ခုကို ၎င်းကိုခေါ်ဆိုသော function ရှိ object များကို ရယူပြီး ပြောင်းလဲနိုင်စေသည်။ ဥပမာအနေဖြင့် getint function တစ်ခုကို စဉ်းစားကြည့်ပါ။ ၎င်းသည် character များ၏ stream တစ်ခုကို integer တန်ဖိုးများအဖြစ် ခွဲခြမ်းခြင်းဖြင့် free-format input conversion လုပ်ပေးသည်၊ တစ်ခေါ်တစ်ကြိမ်လျှင် integer တစ်ခုစီ။ getint သည် ၎င်းတွေ့ရှိသော တန်ဖိုးကို return ပြန်ရမည်သာမက input မရှိတော့သောအခါ end of file ကိုလည်း အချက်ပြရမည်။ ဤတန်ဖိုးများကို သီးခြား လမ်းကြောင်းများဖြင့် ပြန်ပို့ရမည်၊ အကြောင်းမှာ EOF အတွက် မည်သည့်တန်ဖိုးကို အသုံးပြုသည်ဖြစ်စေ ၎င်းသည် input integer တစ်ခု၏ တန်ဖိုးလည်းဖြစ်နိုင်သောကြောင့်ဖြစ်သည်။

ဖြေရှင်းချက်တစ်ခုမှာ getint က end of file status ကို ၎င်း၏ function value အဖြစ် return ပြန်စေပြီး pointer argument ကို အသုံးပြု၍ ပြောင်းလဲပြီးသော integer ကို ခေါ်ဆိုသော function သို့ ပြန်သိမ်းစေရန်ဖြစ်သည်။ ဤနည်းလမ်းကို scanf ကလည်း အသုံးပြုသည်။ Section 7.4 ကိုကြည့်ပါ။

အောက်ပါ loop သည် getint ကိုခေါ်ဆိုခြင်းဖြင့် array တစ်ခုကို integer များဖြင့် ဖြည့်သည်:

```
int n, array[SIZE], getint(int *);

for (n = 0; n < SIZE && getint(&array[n]) != EOF; n++);
```

တစ်ခေါ်တစ်ခါလျှင် array[n] ကို input တွင်တွေ့ရှိသော နောက်ထပ် integer သို့ သတ်မှတ်ပြီး n ကို တိုးသည်။ array[n] ၏ address ကို getint သို့ ပို့ပေးရန် မရှိမဖြစ်လိုအပ်သည်ကို သတိပြုပါ။ သို့မဟုတ်လျှင် getint က ပြောင်းလဲပြီးသော integer ကို caller သို့ ပြန်ပို့ပေးနိုင်သော နည်းလမ်းမရှိပါ။

ကျွန်ုပ်တို့၏ getint ဗားရှင်းသည် end of file အတွက် EOF ကို return ပြန်သည်၊ နောက်ထပ် input သည် နံပါတ်မဟုတ်လျှင် သုည (zero) ကို return ပြန်သည်၊ ပြီးနောက် input တွင် တရားဝင်နံပါတ်ပါရှိလျှင် အပေါင်းကိန်း (positive value) ကို return ပြန်သည်။

```
#include

int getch(void);

void ungetch(int);

/* getint: input မှ နောက်ထပ် integer ကို *pn သို့ ရယူသည် */

int getint(int *pn)

{

	int c, sign;

	while (isspace(c = getch())) /* white space ကို ကျော်သည် */;

	if (!isdigit(c) && c != EOF && c != '+' && c != '-') {

	ungetch(c); /* ၎င်းသည် နံပါတ်မဟုတ်ပါ */ return 0;

}

sign = (c == '-') ? -1 : 1;

if (c == '+' || c == '-')

	c = getch();

for (*pn = 0; isdigit(c), c = getch())

	*pn = 10 * *pn + (c - '0');

	*pn *= sign;

if (c != EOF)

	ungetch(c);

return c;

}
```

getint တစ်ခုလုံးတွင် *pn ကို သာမန် int variable တစ်ခုအဖြစ် အသုံးပြုထားသည်။ ကျွန်ုပ်တို့သည် getch နှင့် ungetch (Section 4.3 တွင် ဖော်မြူထားသည်) ကိုလည်း အသုံးပြုထားသည် ထို့ကြောင့် ဖတ်ရမည့် အပိုထပ် character တစ်ခုကို input အပေါ်သို့ push back လုပ်နိုင်သည်။

**လေ့ကျင့်ခန်း ၅-၁။** ရေးထားသည့်အတိုင်းဆိုလျှင် getint သည် + သို့မဟုတ် - နောက်တွင် ဂဏန်းမပါလျှင် သုည (zero) ၏ တရားဝင်ကိုယ်စားပြုမှုအဖြစ် သတ်မှတ်သည်။ ထိုကဲ့သို့ character ကို input အပေါ်သို့ push back လုပ်ရန် ပြင်ဆင်ပါ။

**လေ့ကျင့်ခန်း ၅-၂။** getint ၏ floating-point အလားတူ getfloat ကို ရေးပါ။ getfloat သည် ၎င်း၏ function value အဖြစ် မည်သည့် အမျိုးအစားကို return ပြန်သနည်း?
