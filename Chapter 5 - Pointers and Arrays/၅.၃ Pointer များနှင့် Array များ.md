## ၅.၃ Pointer များနှင့် Array များ

C တွင် pointer များနှင့် array များအကြား အားကောင်းသော ဆက်နွယ်မှုတစ်ခုရှိသည်၊ အလွန်အားကောင်းသောကြောင့် pointer များနှင့် array များကို တစ်ပြိုင်နက်တည်း ဆွေးနွေးသင့်သည်။ Array subscripting ဖြင့် ရရှိနိုင်သော လုပ်ဆောင်ချက်မှန်သမျှကို pointer များဖြင့်လည်း လုပ်ဆောင်နိုင်သည်။ Pointer ဗားရှင်းသည် ယေဘုယျအားဖြင့် ပို၍မြန်ဆန်မည်ဖြစ်သော်လည်း အနည်းဆုံး မကျွမ်းကျင်သေးသူများအတွက် နားလည်ရန် အနည်းငယ် ခက်ခဲသည်။

Declaration

```
int a[10];
```

သည် အရွယ်အစား ၁၀ ရှိသော array တစ်ခုကို သတ်မှတ်သည်၊ ဆိုလိုသည်မှာ a[0], a[1], ..., a[9] ဟုအမည်ပေးထားသော ဆက်တိုက် object ၁၀ ခု၏ block တစ်ခုဖြစ်သည်။

Notation a[i] သည် array ၏ i-th element ကို ရည်ညွှန်းသည်။ pa သည် integer တစ်ခုသို့ point လုပ်သော pointer ဖြစ်ပြီး

```
int *pa;
```

ဟု declare လုပ်ထားလျှင် assignment

```
pa = &a[0];
```

သည် pa ကို a ၏ element သုည (zero) သို့ point လုပ်စေသည်၊ ဆိုလိုသည်မှာ pa တွင် a[0] ၏ address ပါရှိသည်။

ယခု assignment

```
x = *pa;
```

သည် a[0] ၏ အကြောင်းအရာများကို x သို့ ကူးယူမည်ဖြစ်သည်။

pa သည် array တစ်ခု၏ သီးခြား element တစ်ခုကို point လုပ်နေလျှင် သတ်မှတ်ချက်အရ pa+1 သည် နောက်ထပ် element ကို point လုပ်သည်၊ pa+i သည် pa နောက်တွင် i element များကို point လုပ်သည်၊ ပြီးနောက် pa-i သည် pa ရှေ့တွင် i element များကို point လုပ်သည်။ ထို့ကြောင့် pa သည် a[0] ကို point လုပ်နေလျှင်

```
*(pa+1)
```

သည် a[1] ၏ အကြောင်းအရာများကို ရည်ညွှန်းသည်၊ pa+i သည် a[i] ၏ address ဖြစ်သည်၊ ပြီးနောက် *(pa+i) သည် a[i] ၏ အကြောင်းအရာများဖြစ်သည်။

ဤမှတ်ချက်များသည် array a ရှိ variable များ၏ အမျိုးအစား သို့မဟုတ် အရွယ်အစားကို မထည့်တွက်ဘဲ မှန်ကန်သည်။ "pointer တစ်ခုသို့ ၁ ပေါင်းခြင်း" ၏ အဓိပ္ပာယ်နှင့် ချဲ့ထွင်လျှင် pointer arithmetic အားလုံးမှာ pa+1 သည် နောက်ထပ် object ကို point လုပ်သည်၊ ပြီးနောက် pa+i သည် pa ကျော်သော i-th object ကို point လုပ်သည်။

Indexing နှင့် pointer arithmetic အကြား ဆက်စပ်မှုသည် အလွန်နီးစပ်သည်။ သတ်မှတ်ချက်အရ array အမျိုးအစား variable သို့မဟုတ် expression တစ်ခု၏ တန်ဖိုးသည် array ၏ element သုည (zero) ၏ address ဖြစ်သည်။ ထို့ကြောင့် assignment

```
pa = &a[0];
```

ပြီးနောက် pa နှင့် a တို့တွင် တူညီသော တန်ဖိုးများရှိသည်။ Array တစ်ခု၏ အမည်သည် ကနဦး element ၏ တည်နေရာအတွက် ဆင်တူသံကွဲစကားလုံး (synonym) ဖြစ်သောကြောင့် assignment pa=&a[0] ကို

```
pa = a;
```

ဟုလည်း ရေးနိုင်သည်။

ပထမတွေ့ရှိချိန်တွင် ပို၍အံ့သြဖွယ်ကောင်းသည်မှာ a[i] သို့ reference လုပ်ခြင်းကို *(a+i) ဟုလည်း ရေးနိုင်သည့် အချက်ဖြစ်သည်။ a[i] ကို အကဲဖြတ်သောအခါ C သည် ၎င်းကို *(a+i) သို့ ချက်ချင်းပြောင်းလဲသည်။ ပုံစံနှစ်ခုသည် ညီမျှသည်။ ဤညီမျှမှု၏ အစိတ်အပိုင်းနှစ်ခုလုံးသို့ operator & ကို အသုံးပြုလျှင် &a[i] နှင့် a+i တို့သည်လည်း တူညီသည်ဟု နောက်ဆက်တွဲဖြစ်သည်- a+i သည် a ကျော်သော i-th element ၏ address ဖြစ်သည်။ ဤအကြောင်းအရာ၏ အခြားတစ်ဖက်အနေဖြင့် pa သည် pointer တစ်ခုဖြစ်လျှင် expression များသည် ၎င်းကို subscript ဖြင့် အသုံးပြုနိုင်သည်၊ pa[i] သည် *(pa+i) နှင့် တူညီသည်။ အတိုချုပ်ဆိုရလျှင် array-and-index expression သည် pointer and offset အဖြစ် ရေးထားသော expression နှင့် ညီမျှသည်။

Array name နှင့် pointer အကြား မှတ်သားထားရမည့် ကွာခြားချက်တစ်ခုရှိသည်။ Pointer သည် variable တစ်ခုဖြစ်သောကြောင့် pa=a နှင့် pa++ တို့သည် တရားဝင်သည်။ သို့သော် array name သည် variable တစ်ခုမဟုတ်ပါ၊ a=pa နှင့် a++ ကဲ့သို့သော တည်ဆောက်မှုများသည် တရားမဝင်ပါ။

Array name တစ်ခုကို function တစ်ခုသို့ ပို့သောအခါ ပို့သည့်အရာမှာ ကနဦး element ၏ တည်နေရာဖြစ်သည်။ ခေါ်ဆိုထားသော function အတွင်း၌ ဤ argument သည် local variable တစ်ခုဖြစ်သည်၊ ထို့ကြောင့် array name parameter သည် pointer တစ်ခုဖြစ်သည်၊ ဆိုလိုသည်မှာ address ပါရှိသော variable တစ်ခုဖြစ်သည်။ ကျွန်ုပ်တို့သည် ဤအချက်အလက်ကို အသုံးပြု၍ strlen ၏ နောက်ထပ်ဗားရှင်းတစ်ခုကို ရေးနိုင်သည်၊ ၎င်းသည် string တစ်ခု၏ အရှည်ကို တွက်ချက်သည်။

```
/* strlen: string s ၏ အရှည်ကို return ပြန်သည် */

int strlen(char *s)
{
int n;
	for (n = 0; *s != '\0', s++)
		n++;
	return n;
}
```

s သည် pointer တစ်ခုဖြစ်သောကြောင့် ၎င်းကို တိုးခြင်းသည် လုံးဝတရားဝင်သည်၊ s++ သည် strlen ကို ခေါ်ဆိုသော function ရှိ character string အပေါ် သက်ရောက်မှုမရှိပါ၊ သို့သော် strlen ၏ ကိုယ်ပိုင် pointer copy ကိုသာ တိုးပေးသည်။ ဆိုလိုသည်မှာ ဤကဲ့သို့ ခေါ်ဆိုခြင်းများ

```
strlen("hello, world"); /* string constant */

strlen(array); /* char array[100]; */

strlen(ptr); /* char *ptr; */ အားလုံး အလုပ်လုပ်သည်။
```

Function definition တွင် formal parameter များအနေဖြင့်

```
char s[];
```

နှင့်

```
char *s;
```

တို့သည် ညီမျှသည်၊ ကျွန်ုပ်တို့သည် နောက်ဆုံးတစ်ခုကို ပို၍နှစ်သက်သည် အကြောင်းမှာ ၎င်းသည် variable သည် pointer တစ်ခုဖြစ်သည်ကို ပို၍ တိကျစွာပြောသောကြောင့်ဖြစ်သည်။ Array name တစ်ခုကို function တစ်ခုသို့ ပို့သောအခါ function သည် ၎င်း၏ အဆင်ပြေသလို

array သို့မဟုတ် pointer ကို လက်ခံရရှိခဲ့သည်ဟု ယုံကြည်နိုင်ပြီး ထိုနည်းအတိုင်း ကိုင်တွယ်နိုင်သည်။ သင့်လျော်ပြီး ရှင်းလင်းသည်ဟု ထင်ရလျှင် notation နှစ်ခုစလုံးကို ပင် အသုံးပြုနိုင်သည်။

Array တစ်ခု၏ အစိတ်အပိုင်းတစ်ခုကို function တစ်ခုသို့ ပို့ရန် ဖြစ်နိုင်သည်၊ subarray ၏ အစသို့ pointer တစ်ခုကို ပို့ပေးခြင်းဖြင့်ဖြစ်သည်။ ဥပမာအားဖြင့် a သည် array တစ်ခုဖြစ်လျှင်

```
f(&a[2])
```

နှင့်

```
f(a+2)
```

နှစ်ခုစလုံးသည် function f သို့ a[2] တွင် စတင်သော subarray ၏ address ကို ပို့ပေးသည်။ f အတွင်း၌ parameter declaration သည်

```
f(int arr[]) { ... }
```

သို့မဟုတ်

```
f(int *arr) { ... }
```

ဟု ဖတ်နိုင်သည်။

ထို့ကြောင့် f နှင့်သက်ဆိုင်သလောက်တွင် parameter သည် ပို၍ကြီးမားသော array တစ်ခု၏ အစိတ်အပိုင်းကို ရည်ညွှန်းသည့် အချက်သည် အရေးမကြီးပါ။

Element များ ရှိသည်ဟု သေချာလျှင် array တွင် နောက်ပြန်လှည့် index လုပ်ရန်လည်း ဖြစ်နိုင်သည်၊ p[-1], p[-2] စသည်ဖြင့် သည် syntax အရ တရားဝင်ပြီး p[0] ကို တိုက်ရိုက်ရှေ့က element များကို ရည်ညွှန်းသည်။ သို့သော် array bounds အတွင်း မရှိသော object များကို ရည်ညွှန်းခြင်းသည် တရားမဝင်သည့်အရာမှာ ဖြစ်သည်။
