## ၅.၄ Address Arithmetic

p သည် array တစ်ခု၏ element အချို့သို့ point လုပ်သော pointer ဖြစ်လျှင် p++ သည် p ကို နောက်ထပ် element သို့ point လုပ်ရန် တိုးသည်၊ ပြီးနောက် p+=i သည် ၎င်းလက်ရှိရှိနေသော နေရာမှ i element များကျော်၍ point လုပ်ရန် တိုးသည်။ ဤနှင့်တူသော တည်ဆောက်မှုများသည် pointer သို့မဟုတ် address arithmetic ၏ ရိုးရှင်းဆုံး ပုံစံများဖြစ်သည်။

C သည် address arithmetic ကို ချဉ်းကပ်ရာတွင် လိုက်လျောညီထွေနှင့် ပုံမှန်ဖြစ်သည်။ ၎င်း၏ pointer များ၊ array များနှင့် address arithmetic ပေါင်းစပ်မှုသည် ဘာသာစကား၏ အားသာချက်များထဲမှ တစ်ခုဖြစ်သည်။ အခြေခံ storage allocator တစ်ခု ရေးခြင်းဖြင့် ဖော်ပြကြပါစို့။ Routine နှစ်ခုရှိသည်။ ပထမတစ်ခုမှာ alloc(n) ဖြစ်ပြီး ဆက်တိုက် character position n ခုသို့ pointer တစ်ခုကို return ပြန်သည်၊ ၎င်းကို alloc ၏ caller က character များကို သိမ်းဆည်းရန် အသုံးပြုနိုင်သည်။ ဒုတိယတစ်ခုမှာ afree(p) ဖြစ်ပြီး ရရှိထားသော storage ကို ထုတ်လွှတ်ပေးသည် ထို့ကြောင့် နောက်တွင် ပြန်လည်အသုံးပြုနိုင်သည်။ Routine များသည် "အခြေခံ" ဖြစ်ကြသည် အကြောင်းမှာ afree သို့ ခေါ်ဆိုခြင်းများကို alloc အပေါ် ပြုလုပ်သော ခေါ်ဆိုမှုများနှင့် ပြောင်းပြန်အစီအစဉ်ဖြင့် ပြုလုပ်ရမည်။ ဆိုလိုသည်မှာ alloc နှင့် afree ဖြင့် စီမံခန့်ခွဲသော storage သည် stack သို့မဟုတ် last-in, first-out ဖြစ်သည်။ Standard library တွင် ထိုကဲ့သို့ ကန့်သတ်ချက်များမရှိသော malloc နှင့် free ဟုခေါ်သော အလားတူ function များ ပံ့ပိုးပေးထားသည်။ Section 8.7 တွင် ၎င်းတို့ကို မည်သို့ အကောင်အထည်ဖော်နိုင်သည်ကို ပြသမည်။

အလွယ်ကူဆုံး အကောင်အထည်ဖော်မှုမှာ alloc က allocbuf ဟု ခေါ်မည့် ကြီးမားသော character array တစ်ခု၏ အစိတ်အပိုင်းများကို ခွဲဝေပေးစေရန်ဖြစ်သည်။ ဤ array သည် alloc နှင့် afree အတွက် private ဖြစ်သည်။ ၎င်းတို့သည် pointer များဖြင့် ကိုင်တွယ်ကြပြီး array index များဖြင့် မဟုတ်သောကြောင့် အခြား routine များက array ၏ အမည်ကို သိရန် မလိုအပ်ပါ၊ ၎င်းကို alloc နှင့် afree ပါဝင်သော source file တွင် static အဖြစ် declare လုပ်နိုင်ပြီး ထို့ကြောင့် ၎င်း၏ အပြင်ဘက်တွင် မမြင်ရပါ။ လက်တွေ့ အကောင်အထည်ဖော်မှုများတွင် array သည် အမည်တောင်မရှိနိုင်ပါ။ ၎င်းအစား malloc ကို ခေါ်ဆိုခြင်းဖြင့် သို့မဟုတ် operating system ကို အမည်မဲ့ storage block တစ်ခုသို့ pointer အတွက် တောင်းဆိုခြင်းဖြင့် ရရှိနိုင်သည်။

လိုအပ်သော အခြားအချက်အလက်မှာ allocbuf ၏ မည်မျှကို အသုံးပြုထားပြီးသည်ဆိုသည့်အရာဖြစ်သည်။ ကျွန်ုပ်တို့သည် allocp ဟုခေါ်သော pointer တစ်ခုကို အသုံးပြုသည်၊ ၎င်းသည် နောက်ထပ် လွတ်လပ်သော element ကို point လုပ်သည်။ alloc ကို n character များအတွက် တောင်းဆိုလာသောအခါ ၎င်းသည် allocbuf တွင် လုံလောက်သော နေရာကျန်ရှိမရှိ စစ်ဆေးသည်။ ရှိလျှင် alloc သည် allocp ၏ လက်ရှိတန်ဖိုး (ဆိုလိုသည်မှာ လွတ်လပ်သော block ၏ အစ) ကို return ပြန်ပြီး နောက်ထပ် လွတ်လပ်သော ဧရိယာသို့ point လုပ်ရန် ၎င်းကို n ဖြင့် တိုးသည်။ နေရာမရှိလျှင် alloc သည် သုည (zero) ကို return ပြန်သည်။ afree(p) သည် p သည် allocbuf အတွင်း၌ ရှိလျှင် allocp ကို p သို့ သတ်မှတ်ရုံသာဖြစ်သည်။

```
#define ALLOCSIZE 10000 /* ရရှိနိုင်သော နေရာ၏ အရွယ်အစား */

static char allocbuf[ALLOCSIZE]; /* alloc အတွက် storage */

static char *allocp = allocbuf; /* နောက်ထပ် လွတ်လပ်သော position */

char *alloc(int n) /* n character များသို့ pointer ကို return ပြန်သည် */

{

if (allocbuf + ALLOCSIZE - allocp >= n) { /* ကိုက်ညီသည် */

	allocp += n;

	return allocp - n; /* old p */

} else /* လုံလောက်သော နေရာမရှိပါ */

	return 0;

}

	void afree(char *p) /* p ဖြင့် point လုပ်ထားသော storage ကို လွတ်လပ်စေသည် */

{

if (p >= allocbuf && p < allocbuf + ALLOCSIZE)

	allocp = p;

}
```

ယေဘုယျအားဖြင့် pointer တစ်ခုကို အခြား variable များကဲ့သို့ပင် initialize လုပ်နိုင်သည်၊ သို့သော် ပုံမှန်အားဖြင့် အဓိပ္ပာယ်ရှိသော တန်ဖိုးများမှာ သုည (zero) သို့မဟုတ် သင့်လျော်သော အမျိုးအစား၏ ကြိုတင်သတ်မှတ်ထားသော data ၏ address ပါဝင်သော expression ဖြစ်သည်။ Declaration

```
static char *allocp = allocbuf;
```

သည် allocp ကို character pointer တစ်ခုအဖြစ် သတ်မှတ်ပြီး allocbuf ၏ အစသို့ point လုပ်ရန် initialize လုပ်သည်၊ ၎င်းသည် program စတင်သောအခါ နောက်ထပ် လွတ်လပ်သော position ဖြစ်သည်။ ဤကိုလည်း

```
static char *allocp = &allocbuf[0];
```

ဟု ရေးနိုင်သည်၊ အကြောင်းမှာ array name သည် zeroth element ၏ address ဖြစ်သောကြောင့်ဖြစ်သည်။

သည် n character များအတွက် တောင်းဆိုချက်ကို ပြည့်မီစေရန် လုံလောက်သော နေရာရှိမရှိ စစ်ဆေးသည်။ ရှိလျှင် allocp ၏ တန်ဖိုးအသစ်သည် allocbuf ၏ အဆုံးကျော်၍ တစ်ခုအများဆုံး ရှိမည်ဖြစ်သည်။ တောင်းဆိုချက်ကို ပြည့်မီစေနိုင်လျှင် alloc သည် character များ၏ block တစ်ခု၏ အစသို့ pointer တစ်ခုကို return ပြန်သည် (function ကိုယ်တိုင်၏ declaration ကို သတိပြုပါ)။ မဟုတ်လျှင် alloc သည် နေရာမကျန်တော့ကြောင်း အချက်ပြသည့် signal တစ်ခုကို return ပြန်ရမည်။ C သည် သုည (zero) သည် data အတွက် တရားဝင် address တစ်ခု ဘယ်တော့မှမဖြစ်ကြောင်း အာမခံသည်၊ ထို့ကြောင့် return value သုည (zero) ကို ပုံမှန်မဟုတ်သော ဖြစ်ရပ်တစ်ခု၊ ဤအခြေအနေတွင် နေရာမရှိခြင်း၊ ကို အချက်ပြရန် အသုံးပြုနိုင်သည်။

Pointer များနှင့် integer များသည် အပြန်အလှန်လဲလှယ်၍မရပါ။ သုည (Zero) သည် ခြွင်းချက်တစ်ခုတည်းဖြစ်သည်- constant သုည (zero) ကို pointer သို့ သတ်မှတ်နိုင်ပြီး pointer ကို constant သုည (zero) နှင့် နှိုင်းယှဉ်နိုင်သည်။ Symbolic constant NULL ကို သုည (zero) အစား မကြာခဏ အသုံးပြုသည်၊ ဤသည်မှာ pointer အတွက် အထူးတန်ဖိုးတစ်ခုဖြစ်ကြောင်း ပို၍ ရှင်းလင်းစွာ ညွှန်ပြရန် မှတ်မိလွယ်သော အနေဖြင့်ဖြစ်သည်။ NULL ကို <stdio.h> တွင် သတ်မှတ်ထားသည်။ ကျွန်ုပ်တို့သည် နောက်တွင် NULL ကို အသုံးပြုမည်ဖြစ်သည်။

တို့သည် pointer arithmetic ၏ အရေးကြီးသော ရှုထောင့်များစွာကို ပြသသည်။ ပထမဦးစွာ pointer များကို အချို့အခြေအနေများတွင် နှိုင်းယှဉ်နိုင်သည်။ p နှင့် q သည် တူညီသော array တစ်ခု၏ member များသို့ point လုပ်လျှင် ==, !=, <, >= စသည့် relation များသည် မှန်ကန်စွာ အလုပ်လုပ်သည်။ ဥပမာအားဖြင့်

```
p < q
```

သည် p သည် q ထက် array ၏ အစောပိုင်း element တစ်ခုကို point လုပ်နေလျှင် မှန်သည်။ မည်သည့် pointer မဆို သုည (zero) နှင့် တန်းတူမှု သို့မဟုတ် မတူညီမှုအတွက် အဓိပ္ပာယ်ရှိစွာ နှိုင်းယှဉ်နိုင်သည်။ သို့သော် တူညီသော array တစ်ခု၏ member များသို့ point မလုပ်သော pointer များနှင့် arithmetic သို့မဟုတ် နှိုင်းယှဉ်မှုများအတွက် အပြုအမူကို သတ်မှတ်မထားပါ။ (ခြွင်းချက်တစ်ခုရှိသည်- array ၏ အဆုံးကျော်သော ပထမ element ၏ address ကို pointer arithmetic တွင် အသုံးပြုနိုင်သည်။)

ဒုတိယအချက်မှာ pointer တစ်ခုနှင့် integer တစ်ခုကို ပေါင်း သို့မဟုတ် နုတ်နိုင်သည်ကို ကျွန်ုပ်တို့ လေ့လာခဲ့ပြီးဖြစ်သည်။ တည်ဆောက်မှု

```
p + n
```

သည် p လက်ရှိ point လုပ်နေသောအရာကျော်၍ n-th object ၏ address ကို ဆိုလိုသည်။ ဤသည်မှာ p က မည်သည့်အမျိုးအစား object ကို point လုပ်သည်ဖြစ်စေ မှန်ကန်သည်။ n ကို p point လုပ်သော object များ၏ အရွယ်အစားအတိုင်း scaled လုပ်သည်၊ ၎င်းကို p ၏ declaration ဖြင့် သတ်မှတ်သည်။ ဥပမာအားဖြင့် int သည် လေးဘိုက်ဖြစ်လျှင် int ကို လေးဖြင့် scaled လုပ်မည်ဖြစ်သည်။

Pointer subtraction သည်လည်း တရားဝင်သည်- p နှင့် q သည် တူညီသော array တစ်ခု၏ element များကို point လုပ်ပြီး p<q လျှင် q-p+1 သည် p မှ q အထိ ပါဝင်သော element များ၏ အရေအတွက်ဖြစ်သည်။ ဤအချက်အလက်ကို strlen ၏ နောက်ထပ်ဗားရှင်းတစ်ခု ရေးရန် အသုံးပြုနိုင်သည်:

```
/* strlen: string s ၏ အရှည်ကို return ပြန်သည် */

int strlen(char *s)
{
	char *p = s;
	while (*p != '\0')
		p++;
	return p - s;
}
```

၎င်း၏ declaration တွင် p ကို s သို့ initialize လုပ်သည်၊ ဆိုလိုသည်မှာ string ၏ ပထမ character သို့ point လုပ်သည်။ while loop တွင် character တစ်ခုချင်းစီကို အဆုံးတွင် '\0' မတွေ့မချင်း စမ်းစစ်သည်။ p သည် character များကို point လုပ်သောကြောင့် p++ သည် တစ်ကြိမ်လျှင် p ကို နောက်ထပ် character သို့ တိုးသည်၊ ပြီးနောက် p-s သည် တိုးသွားသော character များ၏ အရေအတွက်၊ ဆိုလိုသည်မှာ string အရှည်ကို ပေးသည်။ (String ရှိ character များ၏ အရေအတွက်သည် int တွင် သိမ်းဆည်းရန် အလွန်ကြီးမားနိုင်သည်။ <stddef.h> header သည် ptrdiff_t အမျိုးအစားတစ်ခုကို သတ်မှတ်ထားပြီး ၎င်းသည် pointer value နှစ်ခု၏ signed difference ကို သိမ်းဆည်းရန် လုံလောက်အောင် ကြီးမားသည်။ သို့သော် ကျွန်ုပ်တို့သည် သတိထားလျှင် standard library ဗားရှင်းနှင့် ကိုက်ညီစေရန် strlen ၏ return value အတွက် size_t ကို အသုံးပြုမည်။ size_t သည် sizeof operator ဖြင့် return ပြန်သော unsigned integer အမျိုးအစားဖြစ်သည်။)

Pointer arithmetic သည် လိုက်လျောညီထွေဖြစ်သည်- ကျွန်ုပ်တို့သည် char များထက် storage ပို၍ယူသော float များနှင့် ကိုင်တွယ်နေလျှင်၊ ပြီးနောက် p သည် float သို့ point လုပ်သော pointer ဖြစ်လျှင် p++ သည် နောက်ထပ် float သို့ တိုးမည်ဖြစ်သည်။ ထို့ကြောင့် ကျွန်ုပ်တို့သည် char များအစား float များကို ထိန်းသိမ်းသော alloc ၏ နောက်ထပ်ဗားရှင်းတစ်ခုကို alloc နှင့် afree တစ်လျှောက်လုံး char ကို float သို့ ပြောင်းခြင်းဖြင့်သာ ရေးနိုင်သည်။ Pointer manipulation အားလုံးသည် point လုပ်ထားသော object များ၏ အရွယ်အစားကို အလိုအလျောက် ထည့်တွက်သည်။

တရားဝင် pointer operation များမှာ တူညီသော အမျိုးအစား၏ pointer များ assignment လုပ်ခြင်း၊ pointer တစ်ခုနှင့် integer တစ်ခုကို ပေါင်း သို့မဟုတ် နုတ်ခြင်း၊ တူညီသော array တစ်ခု၏ member များသို့ pointer နှစ်ခုကို နုတ် သို့မဟုတ် နှိုင်းယှဉ်ခြင်း၊ ပြီးနောက် သုည (zero) သို့ assign လုပ် သို့မဟုတ် နှိုင်းယှဉ်ခြင်း တို့ဖြစ်သည်။ အခြား pointer arithmetic အားလုံးသည် တရားမဝင်ပါ။ Pointer နှစ်ခုကို ပေါင်းခြင်း သို့မဟုတ် ၎င်းတို့ကို မြှောက်၊ စား၊ shift သို့မဟုတ် mask လုပ်ခြင်း သို့မဟုတ် ၎င်းတို့သို့ float သို့မဟုတ် double ပေါင်းခြင်း သို့မဟုတ် void * မှလွဲ၍ cast မပါဘဲ အမျိုးအစားတစ်ခု၏ pointer ကို အခြားအမျိုးအစားတစ်ခု၏ pointer သို့ assign လုပ်ခြင်းတောင်မှ တရားမဝင်ပါ။
