# အခန်း ၅ - Pointers နှင့် Arrays များ

Pointer ဆိုသည်မှာ တစ်ခြား variable တစ်ခု၏ address ကို သိမ်းဆည်းထားသော variable တစ်ခုဖြစ်သည်။ C ဘာသာစကားတွင် pointer များကို များစွာအသုံးပြုကြသည်၊ တစ်ခါတစ်ရံတွင် computation တစ်ခုကို ဖော်ပြရန် တစ်ခုတည်းသောနည်းလမ်းဖြစ်သောကြောင့်လည်းကောင်း၊ များသောအားဖြင့် အခြားနည်းလမ်းများထက် ပို၍ ကျစ်လစ်ပြီး ထိရောက်သော code များ ရရှိနိုင်သောကြောင့်လည်းကောင်း ဖြစ်သည်။ Pointer များနှင့် array များသည် အနီးကပ်ဆက်စပ်နေကြသည်။ ဤအခန်းတွင် ထိုဆက်နွယ်မှုကို စူးစမ်းလေ့လာပြီး မည်သို့အသုံးချရမည်ကို ပြသပေးမည်။

Pointer များကို goto statement များနှင့်အတူ နားလည်ရန်မဖြစ်နိုင်သော program များ ဖန်တီးနိုင်သည့် အံ့သြဖွယ်နည်းလမ်းအဖြစ် အတူတကွထည့်သွင်းလေ့ရှိကြသည်။ ၎င်းတို့ကို ဂရုမစိုက်ဘဲ အသုံးပြုလျှင် ဤသို့ဖြစ်နိုင်သည်မှာ အမှန်ပင်ဖြစ်ပြီး မမျှော်လင့်သောနေရာသို့가리키는point လုပ်သော pointer များကို ဖန်တီးရန် လွယ်ကူသည်။ သို့သော် စည်းကမ်းတကျဖြင့် အသုံးပြုလျှင် pointer များသည် ရှင်းလင်းမှုနှင့် ရိုးရှင်းမှုကို ရရှိစေနိုင်သည်။ ဤအချက်ကို ဖော်ပြပေးရန် ကြိုးစားမည်ဖြစ်သည်။

ANSI C တွင် အဓိကပြောင်းလဲမှုမှာ pointer များကို မည်သို့ကိုင်တွယ်အသုံးပြုနိုင်သည်ဆိုသည့် စည်းမျဉ်းများကို တိကျစွာဖော်ပြခြင်းဖြစ်သည်၊ တကယ်တမ်းတွင် ကောင်းမွန်သော programmer များ လက်တွေ့လုပ်ဆောင်နေပြီးသားနှင့် ကောင်းမွန်သော compiler များ စိစစ်နေပြီးသားကို တရားဝင်ပြဋ္ဌာန်းခြင်းဖြစ်သည်။ ထို့အပြင် void * အမျိုးအစား (pointer to void) သည် generic pointer အတွက် သင့်လျော်သော အမျိုးအစားအဖြစ် char * ကို အစားထိုးထားသည်။

## ၅.၁ Pointer များနှင့် Address များ

Memory ကို မည်သို့ စီစဉ်ထားသည်ဆိုသည့် ရိုးရှင်းသော ပုံရိပ်ဖြင့် စတင်ကြပါစို့။ ပုံမှန် machine တစ်ခုတွင် အစဉ်လိုက် နံပါတ်တပ်ထားသော သို့မဟုတ် address ပေးထားသော memory cell များ၏ array တစ်ခုရှိပြီး ၎င်းတို့ကို တစ်ခုချင်းစီ သို့မဟုတ် ဆက်တိုက်အုပ်စုများအဖြစ် ကိုင်တွယ်နိုင်သည်။ ဘုံအခြေအနေတစ်ခုမှာ byte တစ်ခုသည် char တစ်ခုဖြစ်နိုင်သည်၊ one-byte cell နှစ်ခုကို short integer အဖြစ် သတ်မှတ်နိုင်သည်၊ ပြီးနောက် byte လေးခုသည် long တစ်ခုကို ဖွဲ့စည်းသည်။ Pointer သည် address တစ်ခုကို သိမ်းဆည်းနိုင်သော cell များ၏ အုပ်စု (များသောအားဖြင့် နှစ်ခု သို့မဟုत် လေးခု) ဖြစ်သည်။ ထို့ကြောင့် c သည် char တစ်ခုဖြစ်ပြီး p သည် ၎င်းကို가리키는pointing လုপ်သော pointer တစ်ခုဖြစ်လျှင် အခြေအနေကို ဤသို့ဖော်ပြနိုင်သည်:

Unary operator & သည် object တစ်ခု၏ address ကို ပေးသည်၊ ထို့ကြောင့် statement

`p = &c;`

သည် c ၏ address ကို variable p သို့ သတ်မှတ်ပေးသည်၊ ပြီးနောက် p သည် c ကို "point လုပ်သည်" ဟု ဆိုနိုင်သည်။ & operator သည် memory တွင်ရှိသော object များ- variable များနှင့် array element များ- ကိုသာ အသုံးပြုနိုင်သည်။ ၎င်းကို expression များ၊ constant များ သို့မဟုတ် register variable များအပေါ် အသုံးမပြုနိုင်ပါ။

Unary operator * သည် indirection သို့မဟုတ် dereferencing operator ဖြစ်သည်။ pointer တစ်ခုအပေါ် အသုံးပြုလျှင် pointer가리키는pointing လုပ်နေသော object ကို ရယူအသုံးပြုနိုင်သည်။ x နှင့် y သည် integer များဖြစ်ပြီး ip သည် int သို့ point လုပ်သော pointer ဖြစ်သည်ဟု ဆိုကြပါစို့။ ဤအတုအယောင် အစီအစဉ်သည် pointer တစ်ခုကို မည်သို့ declare လုပ်ရမည်နှင့် & နှင့် * ကို မည်သို့အသုံးပြုရမည်ကို ပြသထားသည်:

```
int x = 1, y = 2, z[10];

int *ip; /* ip သည် int သို့ point လုပ်သော pointer ဖြစ်သည် */

ip = &x; /* ip သည် ယခု x ကို point လုပ်သည် */

y = *ip; /* y သည် ယခု ၁ ဖြစ်သည် */

*ip = 0; /* x သည် ယခု ၀ ဖြစ်သည် */

ip = &z[0]; /* ip သည် ယခု z[0] ကို point လုပ်သည် */ 

```

x၊ y နှင့် z တို့၏ declaration သည် ကျွန်ုပ်တို့ အမြဲတွေ့နေရသော ပုံစံဖြစ်သည်။ Pointer ip ၏ declaration,

```
int *ip;
```

ကို မှတ်မိလွယ်အောင် ရည်ရွယ်ထားခြင်းဖြစ်သည်။ ၎င်းက expression *ip သည် int တစ်ခုဖြစ်သည်ဟု ဆိုလိုသည်။ Variable တစ်ခုအတွက် declaration ၏ syntax သည် variable ပေါ်လာနိုင်သော expression များ၏ syntax နှင့် တူညီသည်။ ဤအကြောင်းပြချက်သည် function declaration များအတွက်လည်း အသုံးချရသည်။ ဥပမာအားဖြင့်၊

`double *dp, atof(char *);`

က expression တစ်ခုတွင် *dp နှင့် atof(s) တို့တွင် double တန်ဖိုးများရှိပြီး atof ၏ argument သည် char သို့ point လုပ်သော pointer ဖြစ်သည်ဟု ဆိုလိုသည်။

ထို့အပြင် pointer တစ်ခုသည် သီးခြား object အမျိုးအစားတစ်ခုကိုသာ point လုပ်ရန် ကန့်သတ်ထားသည်ဟူသော အဓိပ္ပာယ်ကိုလည်း သတိပြုသင့်သည်- pointer တိုင်းသည် သီးခြား data type တစ်ခုကို point လုပ်သည်။ (ခြွင်းချက်တစ်ခုရှိသည်- "pointer to void" ကို မည်သည့် pointer အမျိုးအစားကိုမဆို သိမ်းဆည်းရန် အသုံးပြုသည် သို့သော် ၎င်းကိုယ်တိုင်ကို dereference လုပ်၍မရပါ။ Section 5.11 တွင် ပြန်လာမည်ဖြစ်သည်။)

ip သည် integer x ကို point လုပ်နေလျှင် *ip ကို x အသုံးပြုနိုင်သော context မည်သည့်နေရာတွင်မဆို အသုံးပြုနိုင်သည်၊ ထို့ကြောင့်

`*ip = *ip + 10;`

သည် *ip ကို ၁၀ ဖြင့် တိုးသည်။

Unary operator * နှင့် & တို့သည် arithmetic operator များထက် ပို၍ တင်းကျပ်စွာ bind လုပ်သည်၊ ထို့ကြောင့် assignment

```
y = *ip + 1
```

သည် ip가리키는pointing လုပ်နေသည့်အရာကို ယူပြီး ၁ ထပ်ပေါင်းကာ ရလဒ်ကို y သို့ သတ်မှတ်သည်၊ အတူတူပင်

```
*ip += 1
```

သည် ip가리키는pointing လုပ်နေသည့်အရာကို တိုးသည်၊ အောက်ပါများလည်း တူညီသည်

```
++*ip
```

နှင့်

```
(*ip)++
```

ဤနောက်ဆုံးဥပမာတွင် ကွင်းစကွင်းပိတ်များ လိုအပ်သည်။ ၎င်းတို့မပါဘဲဆိုလျှင် expression သည် ip가리키는pointing လုပ်နေသည့်အရာကို တိုးမည့်အစား ip ကိုပင် တိုးမည်ဖြစ်သည်၊ အကြောင်းမှာ * နှင့် ++ ကဲ့သို့သော unary operator များသည် ညာဘက်မှ ဘယ်ဘက်သို့ ပေါင်းစပ်သောကြောင့်ဖြစ်သည်။

နောက်ဆုံးအနေဖြင့် pointer များသည် variable များဖြစ်သောကြောင့် ၎င်းတို့ကို dereference မလုပ်ဘဲ အသုံးပြုနိုင်သည်။ ဥပမာအားဖြင့် iq သည် int သို့ point လုပ်သော နောက်ထပ် pointer တစ်ခုဖြစ်လျှင်၊

```
iq = ip
```

သည် ip ၏ အကြောင်းအရာများကို iq သို့ ကူးယူသည်၊ ထို့ကြောင့် iq သည် ip가리키는pointing လုပ်ခဲ့သည့်အရာကို pointing လုပ်သွားသည်။
