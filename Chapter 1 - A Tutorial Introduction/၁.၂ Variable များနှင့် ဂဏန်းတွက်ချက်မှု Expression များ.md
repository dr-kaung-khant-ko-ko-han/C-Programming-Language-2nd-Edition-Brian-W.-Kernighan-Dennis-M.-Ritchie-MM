# 1.2 ကိန်းရှင်များနှင့် ဂဏန်းတွက်ချက်မှု ဖော်ပြချက်များ

နောက်ထပ် ပရိုဂရမ်သည် °C=(5/9)(F-32) ဖော်မြူလာကို အသုံးပြု၍ အောက်ပါ ဖာရင်ဟိုက် အပူချိန်ဇယားနှင့် ၎င်းတို့၏ စင်တီဂရိတ် သို့မဟုတ် ဆဲလ်စီးယပ်စ် ညီမျှမှုများကို ပုံနှိပ်ဖော်ပြသည်:

|1|-17|
|---|---|
|20|-6|
|40|4|
|60|15|
|80|26|
|100|37|
|120|48|
|140|60|
|160|71|
|180|82|
|200|93|
|220|104|
|240|115|
|260|126|
|280|137|
|300|148|

ပရိုဂရမ်သည် main အမည်တွင်သော function တစ်ခု၏ အဓိပ္ပါယ်ဖော်ပြချက်သာ ပါဝင်နေဆဲဖြစ်သည်။ ၎င်းသည် "hello, world" ကို ပုံနှိပ်ခဲ့သည့်အရာထက် ရှည်လျားသော်လည်း ရှုပ်ထွေးခြင်းမရှိပါ။ ၎င်းသည် မှတ်ချက်များ၊ ကြေညာချက်များ၊ ကိန်းရှင်များ၊ ဂဏန်းတွက်ချက်မှု ဖော်ပြချက်များ၊ loops များနှင့် ပုံစံချထားသော output တို့အပါအဝင် အကြံအစည်အသစ်များစွာကို မိတ်ဆက်ပေးသည်။

```
#include

/* print Fahrenheit-Celsius table

for fahr = 0, 20, ..., 300 */

main()

{

int fahr, celsius;
int lower, upper, step;

lower = 0; /* lower limit of temperature scale */ upper = 300; /* upper limit */ step = 20; /* step size */

fahr = lower;

	while (fahr <= upper) {

		celsius = 5 * (fahr-32) / 9;

		printf("%d\t%d\n", fahr, celsius);

		fahr = fahr + step;

	}

}
```

အောက်ပါ စာကြောင်းနှစ်ကြောင်း

```
/* print Fahrenheit-Celsius table

for fahr = 0, 20, ..., 300 */
```

သည် မှတ်ချက် (comment) ဖြစ်ပြီး၊ ဤကိစ္စတွင် ပရိုဂရမ်က ဘာလုပ်ဆောင်သည်ကို အတိုချုပ်ရှင်းပြထားသည်။ /* နှင့် */ အကြား အက္ခရာများသည် compiler မှ လျစ်လျူရှုခြင်းခံရသည်။ ၎င်းတို့ကို ပရိုဂရမ်တစ်ခု နားလည်ရန် ပိုလွယ်ကူစေရန် လွတ်လပ်စွာ အသုံးပြုနိုင်သည်။ မှတ်ချက်များသည် blank၊ tab သို့မဟုတ် newline ထားနိုင်သည့် နေရာတိုင်းတွင် ပေါ်လာနိုင်သည်။

C တွင်၊ ကိန်းရှင်အားလုံးကို အသုံးမပြုမီ ကြေညာရမည်၊ များသောအားဖြင့် function ၏အစတွင် executable statements များမတိုင်မီ ကြေညာသည်။ ကြေညာချက်သည် ကိန်းရှင်များ၏ ဂုဏ်သတ္တိများကို ကြေငြာသည်။ ၎င်းတွင် အမည်တစ်ခုနှင့် ကိန်းရှင်များစာရင်း ပါဝင်သည်၊ ဥပမာ

```
int fahr, celsius;

int lower, upper, step;
```

int အမျိုးအစားက စာရင်းတင်ထားသော ကိန်းရှင်များသည် integers (ကိန်းပြည့်များ) ဖြစ်ကြောင်း ဆိုလိုသည်။ float နှင့် ဆန့်ကျင်ဘက်အားဖြင့်၊ float ဆိုသည်မှာ ဒသမဖြစ်နိုင်သော ကိန်းများ ဖြစ်သည်။ int နှင့် float နှစ်ခုစလုံး၏ အကွာအဝေးသည် သင်အသုံးပြုနေသော စက်ပေါ် မူတည်သည်။ 16-bits ints များသည် -32768 နှင့် +32767 အကြားတွင် ရှိပြီး အဖြစ်များသည်၊ 32-bit ints များလည်း အဖြစ်များသည်။ float နံပါတ်သည် များသောအားဖြင့် 32-bit ပမာණ ဖြစ်ပြီး အနည်းဆုံး ဆယ်ခြောက်လုံးနေ ဂဏန်းခြောက်လုံးနှင့် ပမာဏမှာ ယေဘုယျအားဖြင့် 10^-38 နှင့် 10^38 အကြားတွင် ဖြစ်သည်။

C သည် int နှင့် float အပြင် အခြားဒေတာအမျိုးအစားများစွာကို ပေးပါသည်:

|char|character - a single byte|
|---|---|
|short|short integer|
|long|long integer|
|double|double-precision floating point|

ဤအရာဝတ္ထုများ၏ အရွယ်အစားသည်လည်း စက်ပေါ် မူတည်သည်။ ဤအခြေခံအမျိုးအစားများ၏ arrays၊ structures နှင့် unions များ၊ ၎င်းတို့ထံသို့ pointers များနှင့် ၎င်းတို့ကို ပြန်ပေးသော functions များလည်း ရှိပြီး၊ ၎င်းအားလုံးကို အချိန်တန်လျှင် တွေ့ရမည်။

အပူချိန်ပြောင်းလဲခြင်း ပရိုဂရမ်တွင် တွက်ချက်မှုသည် assignment statements များဖြင့် စတင်သည်

```
lower = 0;
upper = 300;
step = 20;

```

ဤ statements များသည် ကိန်းရှင်များကို ၎င်းတို့၏ ကနဦးတန်ဖိုးများသို့ သတ်မှတ်သည်။ တစ်ခုချင်းစီ statements များကို semicolons များဖြင့် အဆုံးသတ်သည်။

ဇယား၏ စာကြောင်းတစ်ကြောင်းစီကို တူညီသောနည်းဖြင့် တွက်ချက်သည်၊ ထို့ကြောင့် output line တစ်ခုလျှင် တစ်ကြိမ် ထပ်ခါထပ်ခါလုပ်သော loop ကို အသုံးပြုသည်။ ဤသည်မှာ while loop ၏ ရည်ရွယ်ချက် ဖြစ်သည်

```
while (fahr <= upper) {
		...
}
```

while loop သည် အောက်ပါအတိုင်း လည်ပတ်သည်: ကွင်းစကွင်းပိတ်အတွင်းရှි အခြေအနေကို စမ်းသပ်သည်။ ၎င်းသည် မှန်ကန်ပါက (fahr သည် upper ထက် နည်းသည် သို့မဟုတ် ညီသည်)၊ loop ၏ body (ကွင်းကောက်များထဲတွင် ပိတ်ထားသော statements သုံးခု) ကို execute လုပ်သည်။ ထို့နောက် အခြေအနေကို ပြန်စမ်းသည်၊ မှန်ကန်ပါက body ကို ထပ်မံ execute လုပ်သည်။ စမ်းသပ်မှု မှားသောအခါ (fahr သည် upper ထက် ကျော်လွန်သောအခါ) loop သည် ပြီးဆုံးပြီး၊ loop နောက်က statement တွင် execution ဆက်လက်လုပ်ဆောင်သည်။ ဤပရိုဂရမ်တွင် နောက်ထပ် statements များ မရှိသောကြောင့် ၎င်းသည် ရပ်တန့်သည်။

while ၏ body တွင် ကွင်းကောက်များဖြင့် ပိတ်ထားသော statements တစ်ခု သို့မဟုတ် တစ်ခုထက်ပိုနိုင်သည်၊ အပူချိန်ပြောင်းစက်တွင် ကဲ့သို့၊ သို့မဟုတ် ကွင်းကောက်များမပါဘဲ statement တစ်ခုတည်းနိုင်သည်၊ ဥပမာ

```
while (i < j)
i = 2 * i;

```

မည်သည့်ကိစ္စတွင်မဆို၊ while မှ ထိန်းချုပ်ထားသော statements များကို tab stop တစ်ခုဖြင့် အမြဲ indent လုပ်မည် (ကျွန်ုပ်တို့က spaces လေးခုအဖြစ် ပြသထားသည်) သို့မှသာ မည်သည့် statements များသည် loop အတွင်းတွင် ရှိကြောင်း ချက်ချင်းမြင်နိုင်မည်။ indentation သည် ပရိုဂရမ်၏ ယုတ္တိရှိသော တည်ဆောက်ပုံကို အလေးပေးသည်။ C compilers များသည် ပရိုဂရမ် ပုံစံအကြောင်း ဂရုမစိုက်သော်လည်း၊ သင့်လျော်သော indentation နှင့် spacing သည် လူတို့အတွက် ပရိုဂရမ်များကို ဖတ်ရန် လွယ်ကူစေရန် အရေးကြီးသည်။ စာကြောင်းတစ်ကြောင်းလျှင် statement တစ်ခုသာ ရေးရန်နှင့် operators များ ပတ်လည်တွင် blanks များ အသုံးပြု၍ အုပ်စုဖွဲ့မှုကို ရှင်းလင်းစေရန် ကျွန်ုပ်တို့ အကြံပြုပါသည်။ ကွင်းကောက်များ၏ အနေအထားသည် အရေးမကြီးသော်လည်း လူတို့တွင် စိတ်အားထက်သန်သော ယုံကြည်ချက်များ ရှိကြသည်။ ကျွန်ုပ်တို့သည် လူကြိုက်များသော style အများအပြားထဲမှ တစ်ခုကို ရွေးချယ်ထားသည်။ သင့်နှင့် သင့်လျော်သော style တစ်ခုကို ရွေးချယ်ပါ၊ ထို့နောက် ၎င်းကို တသမတ်တည်း အသုံးပြုပါ။

အလုပ်အများစုကို loop ၏ body တွင် လုပ်ဆောင်သည်။ Celsius အပူချိန်ကို တွက်ချက်ပြီး celsius ကိန်းရှင်သို့ assign လုပ်သည့် statement က

```
celsius = 5 * (fahr-32) / 9;
```

5/9 ဖြင့် မြှောက်မည့်အစား 5 ဖြင့် မြှောက်ပြီး 9 ဖြင့် စားခြင်း၏ အကြောင်းရင်းမှာ C တွင် အခြားဘာသာစကားများစွာတွင်ကဲ့သို့ integer ခွဲခြင်းသည် ဖြတ်တောက်သည်: ဒသမအစိတ်အပိုင်းများကို စွန့်ပစ်သည်။ 5 နှင့် 9 သည် integers များဖြစ်သောကြောင့်၊ 5/9 သည် သုညသို့ ဖြတ်တောက်ခံရမည်ဖြစ်ပြီး Celsius အပူချိန်အားလုံး သုညအဖြစ် အစီရင်ခံခံရမည်။

ဤဥပမာသည် printf မည်သို့ အလုပ်လုပ်သည်ကို နောက်ထပ် အနည်းငယ် ပြသသည်။ printf သည် အထွေထွေရည်ရွယ်ချက် output formatting function တစ်ခုဖြစ်ပြီး Chapter 7 တွင် အသေးစိတ် ဖော်ပြမည်။ ၎င်း၏ပထမ argument သည် ပုံနှိပ်ရမည့် characters များ၏ string ဖြစ်ပြီး၊ % တစ်ခုချင်းစီသည် အခြား (ဒုတိယ၊ တတိယ၊ ...) arguments များထဲမှ တစ်ခုကို မည်သည့်နေရာတွင် အစားထိုးရမည်နှင့် မည်သည့်ပုံစံဖြင့် ပုံနှိပ်ရမည်ကို ညွှန်ပြသည်။ ဥပမာ၊ %d သည် integer argument တစ်ခုကို သတ်မှတ်သည်၊ ထို့ကြောင့် statement က

```
printf("%d\t%d\n", fahr, celsius);
```

integers နှစ်ခုဖြစ်သော fahr နှင့် celsius ၏ တန်ဖိုးများကို ၎င်းတို့အကြား tab (\t) ဖြင့် ပုံနှိပ်စေသည်။

printf ၏ ပထမ argument ရှိ % တည်ဆောက်မှုတစ်ခုချင်းစီသည် သက်ဆိုင်ရာ ဒုတိယ argument၊ တတိယ argument စသည်တို့နှင့် တွဲဖက်ထားသည်။ ၎င်းတို့သည် နံပါတ်နှင့် အမျိုးအစားအရ သင့်လျော်စွာ ကိုက်ညီရမည်၊ သို့မဟုတ်ပါက အဖြေမှားများ ရရှိမည်။

ခပ်ဖြင့်၊ printf သည် C ဘာသာစကား၏ တစ်စိတ်တစ်ပိုင်း မဟုတ်ပါ။ C ကိုယ်တိုင်တွင် input သို့မဟုတ် output အဓိပ္ပါယ်ဖွင့်ဆိုချက် မရှိပါ။ printf သည် များသောအားဖြင့် C ပရိုဂရမ်များအတွက် ရရှိနိုင်သော functions များ၏ standard library မှ အသုံးဝင်သော function တစ်ခုသာဖြစ်သည်။ printf ၏ အပြုအမူကို ANSI standard တွင် သတ်မှတ်ထားသည်၊ ထို့ကြောင့် ၎င်း၏ ဂုဏ်သတ္တိများသည် standard နှင့် ကိုက်ညီသော compiler နှင့် library တိုင်းတွင် တူညီရမည်။

C ကိုယ်တိုင်အပေါ် အာရုံစိုက်ရန်၊ chapter 7 အထိ input နှင့် output အကြောင်း များစွာ မပြောပါ။ အထူးသဖြင့်၊ formatted input ကို ထိုအချိန်အထိ ရွှေ့ဆိုင်းမည်။ သင် နံပါတ်များ input လုပ်ရန် လိုအပ်ပါက Section 7.4 ရှိ scanf function ၏ ဆွေးနွေးချက်ကို ဖတ်ပါ။ scanf သည် printf ကဲ့သို့ ဖြစ်သည်၊ output ရေးမည့်အစား input ဖတ်သည်မှလွဲ၍။

အပူချိန်ပြောင်းလဲခြင်း ပရိုဂရမ်တွင် ပြဿနာနှစ်ခု ရှိသည်။ ပိုရိုးရှင်းသောပြဿနာမှာ နံပါတ်များသည် ညာဘက်ရှိ မမှန်သောကြောင့် output သည် အလှမရှိခြင်းဖြစ်သည်။ ၎င်းသည် ပြင်ဆင်ရန် လွယ်ကူသည်။ printf statement ရှိ %d တစ်ခုချင်းစီကို width ဖြင့် တိုးချဲ့ပါက ပုံနှိပ်ထားသော နံပါတ်များသည် ၎င်းတို့၏ fields များတွင် ညာဘက်မှ justified ဖြစ်မည်။ ဥပမာ၊ ကျွန်ုပ်တို့က ပြောနိုင်သည်

```
printf("%3d %6d\n", fahr, celsius);
```

စာကြောင်းတစ်ကြောင်းချင်းစီ၏ ပထမနံပါတ်ကို ဂဏန်းသုံးလုံး ကျယ်သော field တစ်ခုတွင် ပုံနှိပ်ရန်နှင့် ဒုတိယကို ဂဏန်းခြောက်လုံး ကျယ်သော field တစ်ခုတွင် ပုံနှိပ်ရန်၊ ဤကဲ့သို့:

|0|-17|
|---|---|
|20|-6|
|40|4|
|60|15|
|80|26|
|100|37|
|...||

ပိုပြင်းထန်သော ပြဿနာမှာ ကျွန်ုပ်တို့သည် integer arithmetic ကို အသုံးပြုထားသောကြောင့် Celsius အပူချိန်များသည် များစွာ တိကျမှန်ကန်မှု မရှိခြင်း ဖြစ်သည်။ ဥပမာ၊ 0°F သည် တကယ်တော့ ခန့်မှန်းခြေ -17.8°C ဖြစ်သည်၊ -17 မဟုတ်ပါ။ ပိုတိကျသော အဖြေများ ရရှိရန်၊ integer အစား floating-point arithmetic ကို အသုံးပြုသင့်သည်။ ဤသည်က ပရိုဂရမ်တွင် အပြောင်းအလဲအချို့ လိုအပ်သည်။ ဒုတိယဗားရှင်းမှာ:

```
#include

/* print Fahrenheit-Celsius table

for fahr = 0, 20, ..., 300; floating-point version */

main()

{

float fahr, celsius;
float lower, upper, step;

lower = 0; /* lower limit of temperatuire scale */ upper = 300; /* upper limit */ step = 20; /* step size */

fahr = lower;

	while (fahr <= upper) {

		celsius = (5.0/9.0) * (fahr-32.0);

		printf("%3.0f %6.1f\n", fahr, celsius);

		fahr = fahr + step;

	}

}
```

ဤသည်မှာ အရင်နှင့် များစွာတူညီသည်၊ fahr နှင့် celsius ကို float အဖြစ် ကြေညာထားခြင်းနှင့် ပြောင်းလဲခြင်းအတွက် ဖော်မြူလာကို သဘာဝကျသော နည်းဖြင့် ရေးထားခြင်း မှလွဲ၍။ integer ခွဲခြင်းက ၎င်းကို သုညသို့ ဖြတ်တောက်မည်ဖြစ်သောကြောင့် အရင်ဗားရှင်းတွင် 5/9 ကို အသုံးမပြုနိုင်ခဲ့ပါ။ သို့သော် constant တစ်ခုတွင် ဒသမအမှတ်က ၎င်းသည် floating point ဖြစ်ကြောင်း ညွှန်ပြသည်၊ ထို့ကြောင့် 5.0/9.0 သည် floating-point တန်ဖိုးနှစ်ခု၏ အချိုး ဖြစ်သောကြောင့် ဖြတ်တောက်ခံရခြင်း မရှိပါ။

ဂဏန်းတွက်ချက်မှု operator တစ်ခုတွင် integer operands များ ရှိပါက integer operation လုပ်ဆောင်သည်။ သို့သော် ဂဏန်းတွက်ချက်မှု operator တစ်ခုတွင် floating-point operand တစ်ခုနှင့် integer operand တစ်ခု ရှိပါက operation မလုပ်မီ integer ကို floating point သို့ ပြောင်းလဲမည်။ (fahr-32) ကို ရေးခဲ့ပါက 32 ကို အလိုအလျောက် floating point သို့ ပြောင်းမည်။ သို့သော်လည်း၊ integral တန်ဖိုးများ ရှိသော်လည်း floating-point constants များကို explicit ဒသမအမှတ်များဖြင့် ရေးခြင်းသည် လူဖတ်သူများအတွက် ၎င်းတို့၏ floating-point သဘောသဘာဝကို အလေးပေးသည်။

integers များကို floating point သို့ အဘယ်အချိန်တွင် ပြောင်းလဲသည်ကို အသေးစိတ်စည်းမျဉ်းများသည် Chapter 2 တွင် ရှိသည်။ ယခုအတွက်၊ assignment ကို သတိပြုပါ

```
fahr = lower;
```

နှင့် စမ်းသပ်မှု

```
while (fahr <= upper)
```

သဘာဝကျသော နည်းဖြင့်လည်း အလုပ်လုပ်သည် - operation မလုပ်မီ int ကို float သို့ ပြောင်းလဲသည်။

printf conversion specification %3.0f က floating-point number တစ်ခု (ဤနေရာတွင် fahr) ကို အနည်းဆုံး သုံးလုံး ကျယ်သော characters များဖြင့် ပုံနှိပ်ရန်၊ ဒသမအမှတ်မရှိ၊ ဒသမဂဏန်းများမရှိ ဟု ပြောသည်။ %6.1f က အခြား number တစ်ခု (celsius) ကို အနည်းဆုံး ခြောက်လုံး ကျယ်သော characters များဖြင့်၊ ဒသမအမှတ်နောက် ဂဏန်း 1 လုံးဖြင့် ပုံနှိပ်ရန် ဖော်ပြသည်။ output က ဤကဲ့သို့ ဖြစ်သည်:

|0|-17.8|
|---|---|
|20|-6.7|
|40|4.4|
|...||

Width နှင့် precision ကို specification မှ ချန်လှပ်နိုင်သည်: %6f က နံပတ်သည် အနည်းဆုံး ခြောက်လုံး ကျယ်ရမည်ဟု ပြောသည်။ %.2f က ဒသမအမှတ်နောက် characters နှစ်လုံး သတ်မှတ်သည်၊ သို့သော် width ကို ကန့်သတ်မထားပါ။ %f က နံပါတ်ကို floating point အဖြစ် ပုံနှိပ်ရန်သာ ပြောသည်။

|%d|decimal integer အဖြစ် ပုံနှိပ်ရန်|
|---|---|
|%6d|decimal integer အဖြစ် ပုံနှိပ်ရန်၊ အနည်းဆုံး 6 characters ကျယ်သော|
|%f|floating point အဖြစ် ပုံနှိပ်ရန်|
|%6f|floating point အဖြစ် ပုံနှိပ်ရန်၊ အနည်းဆုံး 6 characters ကျယ်သော|
|%.2f|floating point အဖြစ် ပုံနှိပ်ရန်၊ ဒသမအမှတ်နောက် 2 characters|
|%6.2f|floating point အဖြစ် ပုံနှိပ်ရန်၊ အနည်းဆုံး 6 ကျယ်သောနှင့် ဒသမအမှတ်နောက် 2 ဂဏန်း|

အခြားတို့အကြားတွင်၊ printf သည် octal အတွက် %o၊ hexadecimal အတွက် %x၊ character အတွက် %c၊ character string အတွက် %s နှင့် ၎င်းကိုယ်တိုင်အတွက် % % ကို အသိအမှတ်ပြုသည်။
