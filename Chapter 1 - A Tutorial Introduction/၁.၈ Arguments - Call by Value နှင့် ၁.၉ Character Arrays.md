## 1.8 Arguments - Call by Value

C functions များ၏ ရှုထောင့်တစ်ခုသည် အခြားဘာသာစကားများ အထူးသဖြင့် Fortran ကို အသုံးပြုကျွမ်းကျင်သော ပရိုဂရမ်မာများအတွက် မရင်းနှီးနိုင်ပါ။ C တွင် function arguments အားလုံးကို "by value" ဖြင့် ပေးပို့သည်။ ဆိုလိုသည်မှာ ခေါ်ဆိုထားသော function သည် မူရင်းများထက် ယာယီ ကိန်းရှင်များတွင် ၎င်း၏ arguments များ၏ တန်ဖိုးများကို ရရှိသည်။ ဤသည်က Fortran ကဲ့သို့ "call by reference" ဘာသာစကားများ သို့မဟုတ် Pascal ရှိ var parameters များနှင့် ကွာခြားသော ဂုဏ်သတ္တိများ ဖြစ်ပေါ်စေပြီး ၎င်းတို့တွင် ခေါ်ဆိုထားသော routine သည် local copy မဟုတ်ဘဲ မူရင်း argument သို့ ဝင်ရောက်ခွင့် ရှိသည်။

သို့သော် Call by value သည် အရင်းအနှီး ဖြစ်သည်၊ တာဝန်ထမ်းရမှု မဟုတ်ပါ။ ၎င်းသည် များသောအားဖြင့် အပိုကိန်းရှင်များ နည်းပါးသော ပိုကျစ်လစ်သော ပရိုဂရမ်များကို ဖြစ်ပေါ်စေသည်၊ အဘယ်ကြောင့်ဆိုသော် parameters များကို ခေါ်ဆိုထားသော routine တွင် အဆင်ပြေစွာ initialize လုပ်ထားသော local ကိန်းရှင်များအဖြစ် ကိုင်တွယ်နိုင်သောကြောင့် ဖြစ်သည်။ ဥပမာ ဤဂုဏ်သတ္တိကို အသုံးပြုသော power ၏ ဗားရှင်းတစ်ခု ရှိသည်:

```
/* power: raise base to n-th power; n >= 0; version 2 */

int power(int base, int n)

{

int p;

for (p = 1; n > 0; --n)

p = p * base;

return p;

}
```

Parameter n ကို ယာယီ ကိန်းရှင်တစ်ခုအဖြစ် အသုံးပြုပြီး သုညဖြစ်သည်အထိ ရေတွက်ဆင်းသည် (နောက်ပြန်လှည့် လည်ပတ်သော for loop)။ ကိန်းရှင် i အတွက် ထပ်မံလိုအပ်ခြင်း မရှိတော့ပါ။ power အတွင်းတွင် n သို့ မည်သို့ပင် လုပ်ဆောင်စေကာမူ power ကို မူလက ခေါ်ဆိုခဲ့သော argument အပေါ် အကျိုးသက်ရောက်မှု မရှိပါ။

လိုအပ်သည့်အခါ calling routine တွင် ကိန်းရှင်တစ်ခုကို ပြုပြင်ရန် function တစ်ခုအတွက် စီစဉ်နိုင်သည်။ Caller သည် သတ်မှတ်ရမည့် ကိန်းရှင်၏ လိပ်စာကို (နည်းပညာအရ ကိန်းရှင်သို့ pointer တစ်ခု) ပေးရမည်ဖြစ်ပြီး ခေါ်ဆိုထားသော function သည် parameter ကို pointer တစ်ခုအဖြစ် ကြေညာပြီး ကိန်းရှင်ကို ၎င်းမှတစ်ဆင့် သွယ်ဝိုက်၍ ဝင်ရောက်အသုံးပြုရမည်။ ကျွန်ုပ်တို့ pointers များကို Chapter 5 တွင် လေ့လာမည်။

Arrays များအတွက် ဇာတ်လမ်းသည် ကွာခြားသည်။ Array တစ်ခု၏ အမည်ကို argument အဖြစ် အသုံးပြုသောအခါ function သို့ ပေးပို့သော တန်ဖိုးသည် array ၏ အစ၏ တည်နေရာ သို့မဟုတ် လိပ်စာ ဖြစ်သည် - array elements များ၏ ကူးယူမှု မရှိပါ။ ဤတန်ဖိုးကို subscript ပြုလုပ်ခြင်းဖြင့် function သည် array ၏ argument တစ်ခုခုကို ဝင်ရောက်ပြီး ပြောင်းလဲနိုင်သည်။ ဤသည်မှာ နောက်အပိုင်း၏ အကြောင်းအရာ ဖြစ်သည်။

## 1.9 Character Arrays

C တွင် array ၏ အဖြစ်အများဆုံး အမျိုးအစားမှာ characters များ၏ array ဖြစ်သည်။ Character arrays များနှင့် ၎င်းတို့ကို ကိုင်တွယ်ရန် functions များ၏ အသုံးပြုမှုကို သရုပ်ပြရန် text lines များ၏ set တစ်ခုကို ဖတ်ပြီး အရှည်ဆုံးကို ပုံနှိပ်သော ပရိုဂရမ်တစ်ခု ရေးကြပါစို့။ အကြမ်းဖျင်းမှာ လုံလောက်သလောက် ရိုးရှင်းသည်:

```
while (there's another line)
	if (it's longer than the previous longest)
		(save it)
		(save its length)
print longest line
```

ဤအကြမ်းဖျင်းက ပရိုဂရမ်သည် သဘာဝအရ အပိုင်းများအဖြစ် ခွဲခြားကြောင်း ရှင်းလင်းစေသည်။ အပိုင်းတစ်ခုက စာကြောင်းအသစ် ရယူသည်၊ အခြားတစ်ခုက ၎င်းကို သိမ်းဆည်းသည်၊ ကျန်ရှိသည်က လုပ်ငန်းစဉ်ကို ထိန်းချုပ်သည်။

အရာများသည် အလွန်ကောင်းစွာ ခွဲခြားသောကြောင့် ၎င်းတို့ကို ထိုနည်းဖြင့်လည်း ရေးရန် ကောင်းလိမ့်မည်။ ထို့ကြောင့် ကျွန်ုပ်တို့ ပထမဦးစွာ input ၏ နောက်ထပ် စာကြောင်းကို ယူရန် သီးခြား function getline ကို ရေးကြပါစို့။ အခြား အကြောင်းအရာများတွင် function ကို အသုံးဝင်စေရန် ကြိုးစားမည်။ အနည်းဆုံးအားဖြင့် getline သည် ဖြစ်နိုင်သော end of file အကြောင်း အချက်ပြမှုတစ်ခု ပြန်ပေးရမည်။ ပို၍ အသုံးဝင်သော ဒီဇိုင်းမှာ စာကြောင်း၏ အရှည်ကို ပြန်ပေးခြင်း သို့မဟုတ် end of file တွေ့ပါက သုည ပြန်ပေးခြင်း ဖြစ်မည်။ သုညသည် လက်ခံနိုင်သော end-of-file return ဖြစ်သည်၊ အဘယ်ကြောင့်ဆိုသော် ၎င်းသည် တရားဝင် စာကြောင်းအရှည် ဘယ်တော့မှ မဟုတ်ပါ။ Text စာကြောင်းတိုင်းတွင် အနည်းဆုံး character တစ်ခု ရှိသည်။ newline သာပါသော စာကြောင်းတောင် အရှည် 1 ရှိသည်။

အရင်ဆုံး အရှည်ဆုံး စာကြောင်းထက် ပိုရှည်သော စာကြောင်းတစ်ခု တွေ့သောအခါ ၎င်းကို တစ်နေရာရာတွင် သိမ်းဆည်းရမည်။ ဤသည်က စာကြောင်းအသစ်ကို ဘေးကင်းသော နေရာသို့ ကူးယူရန် ဒုတိယ function copy ကို အကြံပြုသည်။

နောက်ဆုံးတွင် getline နှင့် copy ကို ထိန်းချုပ်ရန် main ပရိုဂရမ်တစ်ခု လိုအပ်သည်။ ရလဒ်မှာ ဤသို့:

```
#include

#define MAXLINE 1000 /* maximum input line length */

int getline(char line[], int maxline);

void copy(char to[], char from[]);

/* print the longest input line */

main()

{

int len; /* current line length */ int max; /* maximum length seen so far */ char line[MAXLINE]; /* current input line */ char longest[MAXLINE]; /* longest line saved here */

max = 0;

while ((len = getline(line, MAXLINE)) > 0)

if (len > max) {

max = len;

copy(longest, line);

}

if (max > 0) /* there was a line */

	printf("%s", longest);

return 0;

}

/* getline: read a line into s, return length */

int getline(char s[],int lim)

{

int c, i;

for (i=0; i < lim-1 && (c=getchar())!=EOF && c!='\n'; ++i)

	s[i] = c;

if (c == '\n') {

	s[i] = c;

	++i;

}

s[i] = '\0';

	return i;

}

/* copy: copy 'from' into 'to'; assume to is big enough */

void copy(char to[], char from[])

{

int i;
i = 0;

while ((to[i] = from[i]) != '\0')

	++i;

}
```

Functions များ getline နှင့် copy ကို ပရိုဂရမ်၏ အစတွင် ကြေညာထားပြီး file တစ်ခုတည်းတွင် ပါဝင်သည်ဟု ကျွန်ုပ်တို့ ယူဆသည်။

main နှင့် getline သည် arguments များ တစ်စုံနှင့် ပြန်ပေးသော တန်ဖိုးတစ်ခုမှတစ်ဆင့် ဆက်သွယ်သည်။ getline တွင် arguments များကို စာကြောင်းဖြင့် ကြေညာသည်

```
int getline(char s[], int lim);
```

ဤသည်က ပထမ argument s သည် array တစ်ခု ဖြစ်ပြီး ဒုတိယ lim သည် integer တစ်ခု ဖြစ်ကြောင်း သတ်မှတ်သည်။ ကြေညာချက်တွင် array ၏ အရွယ်အစားကို ပေးပို့ခြင်း၏ ရည်ရွယ်ချက်မှာ သိုလှောင်မှုကို ခွဲထုတ်ရန် ဖြစ်သည်။ Array s ၏ အရှည်သည် getline တွင် မလိုအပ်ပါ၊ အဘယ်ကြောင့်ဆိုသော် ၎င်း၏ အရွယ်အစားကို main တွင် သတ်မှတ်ထားသောကြောင့် ဖြစ်သည်။ getline သည် return ကို အသုံးပြု၍

caller သို့ တန်ဖိုးတစ်ခု ပြန်ပို့ပေးသည်၊ function power ကဲ့သို့ပင်။ ဤစာကြောင်းက getline သည် int ပြန်ပေးကြောင်းလည်း ကြေညာသည်။ int သည် default return အမျိုးအစား ဖြစ်သောကြောင့် ၎င်းကို ချန်လှပ်နိုင်သည်။

အချို့ functions များသည် အသုံးဝင်သော တန်ဖိုး ပြန်ပေးသည်။ copy ကဲ့သို့ အခြားတို့သည် ၎င်းတို့၏ အကျိုးသက်ရောက်မှုအတွက်သာ အသုံးပြုပြီး တန်ဖိုး မပြန်ပေးပါ။ copy ၏ return အမျိုးအစားမှာ void ဖြစ်ပြီး တန်ဖိုး မပြန်ပေးကြောင်း အတိအကျ ဖော်ပြသည်။

getline သည် character '\0' (null character၊ ၎င်း၏ တန်ဖိုးမှာ သုည) ကို ၎င်း ဖန်တီးနေသော array ၏ အဆုံးတွင် ထည့်ပြီး characters များ၏ string ၏ အဆုံးကို အမှတ်အသားပြုသည်။ ဤအသွင်ပြောင်းမှုကို C ဘာသာစကားကလည်း အသုံးပြုသည်။ string constant တစ်ခုကဲ့သို့

```
"hello\n"
```

C ပရိုဂရမ်တစ်ခုတွင် ပေါ်လာသောအခါ ၎င်းကို string ထဲရှိ characters များ ပါဝင်ပြီး အဆုံးကို အမှတ်အသားပြုရန် '\0' ဖြင့် ပြီးဆုံးသော characters များ၏ array အဖြစ် သိမ်းဆည်းသည်။

printf ရှိ %s format specification က သက်ဆိုင်သော argument သည် ဤပုံစံဖြင့် ကိုယ်စားပြုထားသော string တစ်ခု ဖြစ်ရန် မျှော်လင့်သည်။ copy သည်လည်း ၎င်း၏ input argument ကို '\0' ဖြင့် ပြီးဆုံးကြောင်းနှင့် ဤ character ကို output သို့ ကူးယူကြောင်း မှီခိုသည်။

ဤကဲ့သို့ သေးငယ်သော ပရိုဂရမ်တောင်မှ ခက်ခဲသော ဒီဇိုင်းပြဿနာအချို့ ပေးကြောင်း ဖြတ်သန်းရာတွင် ထောက်ပြဖွယ်ရာ ရှိသည်။ ဥပမာ ၎င်း၏ ကန့်သတ်ချက်ထက် ကြီးမားသော စာကြောင်းတစ်ခုကို တွေ့ပါက main က ဘာလုပ်သင့်သနည်း။ getline သည် ဘေးကင်းစွာ အလုပ်လုပ်သည်၊ array ပြည့်သောအခါ စုဆောင်းခြင်း ရပ်တန့်သည်၊ newline တစ်ခုမျှ မတွေ့ရသေးသော်လည်း။ အရှည်နှင့် ပြန်ပေးထားသော နောက်ဆုံး character ကို စမ်းသပ်ခြင်းဖြင့် main သည် စာကြောင်းသည် အလွန်ရှည်လွန်းခြင်း ရှိမရှိ ဆုံးဖြတ်နိုင်ပြီး ထို့နောက် ၎င်း လိုချင်သလို ကိုင်တွယ်နိုင်သည်။ တိုတောင်းမှု၏ အကျိုးစီးပွားအတွက် ကျွန်ုပ်တို့ ဤကိစ္စကို လျစ်လျူရှုခဲ့သည်။

getline ၏ အသုံးပြုသူအတွက် input စာကြောင်းတစ်ခု မည်မျှရှည်မည်ကို ကြိုတင်သိရန် နည်းလမ်း မရှိပါ၊ ထို့ကြောင့် getline သည် overflow အတွက် စစ်ဆေးသည်။ အခြားတစ်ဖက်တွင် copy ၏ အသုံးပြုသူသည် strings များ မည်မျှကြီးသည်ကို ယခုပင် သိထားသည် (သို့မဟုတ် ရှာဖွေနိုင်သည်)၊ ထို့ကြောင့် ကျွန်ုပ်တို့ ၎င်းသို့ error checking ထည့်ရန် မရွေးချယ်ခဲ့ပါ။

လေ့ကျင့်ခန်း 1-16. အရှည်ဆုံး-စာကြောင်း ပရိုဂရမ်၏ main routine ကို ပြန်လည်ပြင်ဆင်ပါ သို့မှသာ ၎င်းသည် မတူညီသော ရှည်လျားသော input lines များ၏ အရှည်ကို မှန်ကန်စွာ ပုံနှိပ်မည်ဖြစ်ပြီး စာသား၏ တတ်နိုင်သမျှကို ပုံနှိပ်မည်။

လေ့ကျင့်ခန်း 1-17. 80 characters ထက် ရှည်သော input lines အားလုံးကို ပုံနှိပ်ရန် ပရိုဂရမ်တစ်ခု ရေးပါ။

လေ့ကျင့်ခန်း 1-18. Input ၏ စာကြောင်းတစ်ခုချင်းစီမှ နောက်တွင်ရှိ blanks နှင့် tabs များကို ဖယ်ရှားရန်နှင့် လုံးဝ blank ဖြစ်နေသော စာကြောင်းများကို ဖျက်ပစ်ရန် ပရိုဂရမ်တစ်ခု ရေးပါ။

လေ့ကျင့်ခန်း 1-19. Character string s ကို ပြောင်းပြန်လှန်သော function reverse(s) တစ်ခု ရေးပါ။ ၎င်းကို အသုံးပြု၍ ၎င်း၏ input ကို တစ်ကြိမ်လျှင် စာကြောင်းတစ်ခု ပြောင်းပြန်လှန်သော ပရိုဂရမ်တစ်ခု ရေးပါ။
