# 1.3 for statement

အထူးသီးသန့် လုပ်ငန်းတစ်ခုအတွက် ပရိုဂရမ်တစ်ခုကို ရေးရန် မတူညီသော နည်းလမ်းများစွာ ရှိသည်။ အပူချိန်ပြောင်းစက်တွင် ကွဲပြားမှုတစ်ခုကို စမ်းကြည့်ကြပါစို့။

```
#include

/* print Fahrenheit-Celsius table */

main()

{

int fahr;

for (fahr = 0; fahr <= 300; fahr = fahr + 20)

printf("%3d %6.1f\n", fahr, (5.0/9.0)*(fahr-32));

}
```

ဤသည်က တူညီသော အဖြေများကို ထုတ်ပေးသည်၊ သို့သော် ၎င်းသည် ကွဲပြားစွာ ပုံပေါက်သည်။ အဓိက အပြောင်းအလဲတစ်ခုမှာ ကိန်းရှင်အများစု၏ ဖယ်ရှားမှု ဖြစ်သည်။ fahr သာ ကျန်ရှိပြီး ကျွန်ုပ်တို့က ၎င်းကို int တစ်ခု ပြုလုပ်ထားသည်။ အောက်ခြေနှင့် အထက်ခြေ ကန့်သတ်ချက်များနှင့် step size သည် for statement တွင် constants များအဖြစ်သာ ပေါ်လာသည်၊ ၎င်းကိုယ်တိုင် တည်ဆောက်မှုအသစ် ဖြစ်ပြီး Celsius အပူချိန်ကို တွက်ချက်သည့် expression သည် ယခု သီးခြား assignment statement အစား printf ၏ တတိယ argument အဖြစ် ပေါ်လာသည်။

ဤနောက်ဆုံး အပြောင်းအလဲသည် ယေဘုယျ စည်းမျဉ်းတစ်ခု၏ ဥပမာတစ်ခု ဖြစ်သည် - အမျိုးအစားတစ်ခု၏ တန်ဖိုးကို အသုံးပြုရန် ခွင့်ပြုထားသော မည်သည့် အကြောင်းအရာတွင်မဆို၊ ထိုအမျိုးအစား၏ ပိုရှုပ်ထွေးသော expression ကို အသုံးပြုနိုင်သည်။ printf ၏ တတိယ argument သည် %6.1f နှင့် ကိုက်ညီရန် floating-point တန်ဖိုးတစ်ခု ဖြစ်ရမည်ဖြစ်သောကြောင့်၊ မည်သည့် floating-point expression မဆို ဤနေရာတွင် ဖြစ်ပွားနိုင်သည်။

for statement သည် loop တစ်ခု ဖြစ်ပြီး while ၏ ယေဘုယျပြုမှု ဖြစ်သည်။ ၎င်းကို အစောပိုင်း while နှင့် နှိုင်းယှဉ်ပါက ၎င်း၏ လုပ်ဆောင်ချက်သည် ရှင်းလင်းသင့်သည်။ ကွင်းစကွင်းပိတ်အတွင်းတွင် အပိုင်းသုံးပိုင်း ရှိပြီး semicolons များဖြင့် ခွဲခြားထားသည်။ ပထမအပိုင်း၊ initialization

```
fahr = 0
```

ကို တစ်ကြိမ် လုပ်ဆောင်သည်၊ loop စစ်မှန်သည့် အတွင်းသို့ မဝင်ရောက်မီ။ ဒုတိယအပိုင်းသည် loop ကို ထိန်းချုပ်သော စမ်းသပ်မှု သို့မဟုတ် အခြေအနေ ဖြစ်သည်:

```
fahr <= 300
```

ဤအခြေအနေကို အကဲဖြတ်သည်။ ၎င်းသည် မှန်ကန်ပါက loop ၏ body (ဤနေရာတွင် printf တစ်ခုတည်း) ကို execute လုပ်သည်။ ထို့နောက် increment step

```
fahr = fahr + 20
```

ကို execute လုပ်ပြီး အခြေအနေကို ပြန်လည်အကဲဖြတ်သည်။ အခြေအနေသည် မှားသောအခါ loop သည် ပြီးဆုံးသည်။ while နှင့်အတူ၊ loop ၏ body သည် statement တစ်ခုတည်း သို့မဟုတ် ကွင်းကောက်များဖြင့် ပိတ်ထားသော statements အုပ်စုတစ်ခု ဖြစ်နိုင်သည်။ initialization၊ condition နှင့် increment သည် မည်သည့် expressions များမဆို ဖြစ်နိုင်သည်။

while နှင့် for အကြား ရွေးချယ်မှုသည် မတရားမှု ရှိပြီး မည်သည့်အရာက ပိုရှင်းလင်းသည်ဟု ထင်သည်အပေါ် အခြေခံသည်။ for သည် များသောအားဖြင့် initialization နှင့် increment သည် statement တစ်ခုတည်းဖြစ်ပြီး ယုတ္တိရှိစွာ ဆက်စပ်နေသော loops များအတွက် သင့်လျော်သည်၊ အဘယ်ကြောင့်ဆိုသော် ၎င်းသည် while ထက် ပိုကျစ်လစ်ပြီး loop control statements များကို တစ်နေရာတည်းတွင် ထားရှိသောကြောင့် ဖြစ်သည်။

လေ့ကျင့်ခန်း 1-5. အပူချိန်ပြောင်းလဲခြင်း ပရိုဂရမ်ကို ဇယားကို ပြောင်းပြန်အစဉ်လိုက်ဖြင့် ပုံနှိပ်ရန် ပြုပြင်ပါ၊ ဆိုလိုသည်မှာ 300 ဒီဂရီမှ 0 သို့။

# 1.4 သင်္ကေတ Constants များ

အပူချိန်ပြောင်းလဲခြင်းကို အပြီးတိုင် စွန့်ပစ်မီ နောက်ဆုံး လေ့လာတွေ့ရှိချက်။ ပရိုဂရမ်တစ်ခုတွင် 300 နှင့် 20 ကဲ့သို့ "မှော်နံပါတ်များ" ကို မြှုပ်နှံထားခြင်းသည် ဆိုးသော အလေ့အကျင့် ဖြစ်သည်။ ၎င်းတို့သည် နောက်မှ ပရိုဂရမ်ကို ဖတ်ရန် လိုအပ်နိုင်သူတစ်ဦးဦးထံ သတင်းအချက်အလက် နည်းစွာသာ ပေးပို့ပြီး ၎င်းတို့ကို စနစ်တကျ နည်းလမ်းဖြင့် ပြောင်းလဲရန် ခက်ခဲသည်။ မှော်နံပါတ်များကို ကိုင်တွယ်ရန် နည်းလမ်းတစ်ခုမှာ ၎င်းတို့ကို အဓိပ္ပါယ်ရှိသော အမည်များ ပေးခြင်း ဖြစ်သည်။ #define စာကြောင်းသည် သင်္ကေတအမည် သို့မဟုတ် သင်္ကေတ constant ကို characters များ၏ သီးခြား string တစ်ခုအဖြစ် သတ်မှတ်သည်:

```
#define name replacement list
```

ထို့နောက်၊ name ၏ မည်သည့် ဖြစ်ပေါ်မှုမဆို (ကိုးကားချက်များတွင် မပါသော နှင့် အခြား name ၏ အစိတ်အပိုင်း မဟုတ်သော) ကို သက်ဆိုင်ရာ အစားထိုး စာသားဖြင့် အစားထိုးမည်။ name သည် ကိန်းရှင်အမည် ကဲ့သို့ တူညီသော ပုံစံ ရှိသည်: စာလုံးတစ်လုံးဖြင့် စတင်သော စာလုံးများနှင့် ဂဏန်းများ၏ အစီအစဉ်။ အစားထိုး စာသားသည် characters များ၏ မည်သည့် အစီအစဉ်မဆို ဖြစ်နိုင်သည်။ ၎င်းသည် နံပါတ်များသို့သာ ကန့်သတ်ထားခြင်း မရှိပါ။

```
#include

#define LOWER 0 /* lower limit of table */
#define UPPER 300 /* upper limit */
#define STEP 20 /* step size */

/* print Fahrenheit-Celsius table */

main()

{

int fahr;

for (fahr = LOWER; fahr <= UPPER; fahr = fahr + STEP)

printf("%3d %6.1f\n", fahr, (5.0/9.0)*(fahr-32));

}
```

LOWER၊ UPPER နှင့် STEP ပမာဏများသည် သင်္ကေတ constants များ ဖြစ်ပြီး ကိန်းရှင်များ မဟုတ်ပါ၊ ထို့ကြောင့် ၎င်းတို့သည် ကြေညာချက်များတွင် မပေါ်လာပါ။ သင်္ကေတ constant အမည်များကို အောက်ခြေကိစ္စ ကိန်းရှင်အမည်များနှင့် အလွယ်တကူ ခွဲခြားနိုင်ရန် အပေါ်ခြေကိစ္စဖြင့် ရိုးရာအားဖြင့် ရေးသည်။ #define စာကြောင်း၏ အဆုံးတွင် semicolon မရှိကြောင်း သတိပြုပါ။
