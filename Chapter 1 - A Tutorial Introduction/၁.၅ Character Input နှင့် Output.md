# 1.5 Character Input နှင့် Output

Character ဒေတာကို စီမံဆောင်ရွက်ရန်အတွက် ဆက်စပ်နေသော ပရိုဂရမ်များ မိသားစုကို ကျွန်ုပ်တို့ စဉ်းစားသွားမည်။ များစွာသော ပရိုဂရမ်များသည် ကျွန်ုပ်တို့ ဤနေရာတွင် ဆွေးနွေးသော prototypes များ၏ တိုးချဲ့ထားသော ဗားရှင်းများသာ ဖြစ်ကြောင်း သင်တွေ့ရှိလိမ့်မည်။

Standard library မှ ပံ့ပိုးပေးသော input နှင့် output ၏ ပုံစံသည် အလွန်ရိုးရှင်းသည်။ Text input သို့မဟုတ် output သည် မည်သည့်နေရာမှ စတင်သည်ဖြစ်စေ သို့မဟုတ် မည်သည့်နေရာသို့ သွားသည်ဖြစ်စေ၊ characters များ၏ streams များအဖြစ် ကိုင်တွယ်သည်။ Text stream တစ်ခုသည် စာကြောင်းများအဖြစ် ခွဲခြားထားသော characters များ၏ အစီအစဉ် ဖြစ်သည်။ စာကြောင်းတစ်ကြောင်းစီတွင် သုည သို့မဟုတ် ပိုမိုသော characters များပါဝင်ပြီး နောက်တွင် newline character ပါဝင်သည်။ input သို့မဟုတ် output stream တစ်ခုချင်းစီကို ဈပုံစံအတိုင်း အတည်ပြုစေရန် library ၏ တာဝန်ဖြစ်သည်။ library ကို အသုံးပြုနေသော C ပရိုဂရမ်မာသည် ပရိုဂရမ်ပြင်ပတွင် စာကြောင်းများကို မည်သို့ကိုယ်စားပြုသည်ကို စိုးရိမ်ရန် မလိုအပ်ပါ။

Standard library သည် တစ်ကြိမ်လျှင် character တစ်ခု ဖတ်ရန် သို့မဟုတ် ရေးရန်အတွက် functions များစွာကို ပေးပါသည်၊ ၎င်းတို့ထဲမှ getchar နှင့် putchar တို့သည် အရိုးရှင်းဆုံး ဖြစ်သည်။ ၎င်းကို ခေါ်သည့် အချိန်တိုင်းတွင်၊ getchar သည် text stream မှ နောက်ထပ် input character ကို ဖတ်ပြီး ၎င်း၏ တန်ဖိုးအဖြစ် ပြန်ပေးသည်။ ဆိုလိုသည်မှာ၊

```
c = getchar();
```

ပြီးနောက် ကိန်းရှင် c တွင် input ၏ နောက်ထပ် character ပါဝင်သည်။ Characters များသည် များသောအားဖြင့် keyboard မှ လာသည်။ files များမှ input ကို Chapter 7 တွင် ဆွေးနွေးသည်။

Function putchar သည် ၎င်းကို ခေါ်သည့် အချိန်တိုင်းတွင် character တစ်ခုကို ပုံနှိပ်သည်:

```
putchar(c);
```

integer ကိန်းရှင် c ၏ အကြောင်းအရာများကို character အဖြစ် ပုံနှိပ်သည်၊ များသောအားဖြင့် screen ပေါ်တွင်။ putchar နှင့် printf သို့ ခေါ်ဆိုမှုများကို ရောထွေးနိုင်သည်။ output သည် ခေါ်ဆိုမှုများ ပြုလုပ်သည့် အစဉ်လိုက်အတိုင်း ပေါ်လာမည်။

### 1.5.1 File ကူးယူခြင်း

getchar နှင့် putchar ရရှိပြီးပါက၊ input နှင့် output အကြောင်း နောက်ထပ် မည်သည့်အရာမျှ မသိဘဲ အသုံးဝင်သော ကုဒ်များစွာကို သင်ရေးနိုင်သည်။ အရိုးရှင်းဆုံး ဥပမာမှာ ၎င်း၏ input ကို ၎င်း၏ output သို့ တစ်ကြိမ်လျှင် character တစ်ခု ကူးယူသော ပရိုဂရမ် ဖြစ်သည်:

```
read a character

		while (charater is not end-of-file indicator)

			output the character just read

			read a character
```

ဤသည်ကို C သို့ ပြောင်းလဲခြင်းက:

```
#include

/* copy input to output; 1st version */

main()

	{

	int c;

	c = getchar();

		while (c != EOF) {

			putchar(c);

			c = getchar();

		}

	}

```

Relational operator != က "မညီသော" ကို ဆိုလိုသည်။

Keyboard သို့မဟုတ် screen ပေါ်တွင် character တစ်ခုအဖြစ် ပေါ်လာသောအရာသည် အခြားအရာအားလုံးကဲ့သို့ ဖြစ်ပြီး၊ အတွင်းပိုင်းတွင် bit pattern အဖြစ်သာ သိမ်းဆည်းထားသည်။ char အမျိုးအစားသည် ထိုကဲ့သို့ character ဒေတာကို သိမ်းဆည်းရန်အတွက် အထူးရည်ရွယ်ထားသော်လည်း မည်သည့် integer အမျိုးအစားကိုမဆို အသုံးပြုနိုင်သည်။ ကျွန်ုပ်တို့သည် သိမ်မွေ့သော်လည်း အရေးကြီးသော အကြောင်းရင်းတစ်ခုအတွက် int ကို အသုံးပြုခဲ့သည်။

ပြဿနာမှာ input ၏ အဆုံးကို တရားဝင် ဒေတာနှင့် ခွဲခြားခြင်း ဖြစ်သည်။ ဖြေရှင်းချက်မှာ input ထပ်မံမရှိတော့သောအခါ getchar သည် ထူးခြားသော တန်ဖိုးတစ်ခုကို ပြန်ပေးသည်၊ တကယ့် character တစ်ခုခုနှင့် ရှုပ်ထွေးလို့ မရသော တန်ဖိုးတစ်ခု။ ဤတန်ဖိုးကို "end of file" အတွက် EOF ဟု ခေါ်သည်။ getchar ပြန်ပေးသော မည်သည့်တန်ဖိုးကိုမဆို ထားရှိနိုင်လောက်သော အမျိုးအစားအဖြစ် c ကို ကြေညာရမည်။ c သည် မည်သည့် char ကိုမဆို ထပ်ဆောင်း၍ EOF ကို ထားရှိနိုင်လောက် ကြီးရမည်ဖြစ်သောကြောင့် char ကို အသုံးမပြုနိုင်ပါ။ ထို့ကြောင့် int ကို အသုံးပြုသည်။

EOF သည် <stdio.h> တွင် သတ်မှတ်ထားသော integer တစ်ခု ဖြစ်သည်၊ သို့သော် ၎င်းသည် မည်သည့် char တန်ဖိုးနှင့်မျှ မတူသည်ရှိသမျှ သီးခြား ကိန်းဂဏန်း တန်ဖိုးသည် အရေးမကြီးပါ။ သင်္ကေတ constant ကို အသုံးပြုခြင်းဖြင့်၊ ပရိုဂရမ်တွင် မည်သည့်အရာမျှ သီးခြား ကိန်းဂဏန်း တန်ဖိုးအပေါ် မှီခိုမှု မရှိကြောင်း ကျွန်ုပ်တို့ အာမခံနိုင်သည်။

ကူးယူရန် ပရိုဂရမ်ကို အတွေ့အကြုံရှိသော C ပရိုဂရမ်မာများက ပိုတိုတောင်းစွာ ရေးလိမ့်မည်။ C တွင်၊ မည်သည့် assignment မဆို၊ ဥပမာ

```
c = getchar();
```

သည် expression တစ်ခု ဖြစ်ပြီး တန်ဖိုးတစ်ခု ရှိသည်၊ ၎င်းသည် assignment ပြီးနောက် ဘယ်ဘက်ခြမ်း၏ တန်ဖိုး ဖြစ်သည်။ ဆိုလိုသည်မှာ assignment သည် ပိုကြီးသော expression ၏ တစ်စိတ်တစ်ပိုင်းအဖြစ် ပေါ်လာနိုင်သည်။ c သို့ character ၏ assignment ကို while loop ၏ စမ်းသပ်အပိုင်းအတွင်းတွင် ထည့်ပါက copy ပရိုဂရမ်ကို ဤနည်းဖြင့် ရေးနိုင်သည်:

```
#include

/* copy input to output; 2nd version */

main()

{

int c;



while ((c = getchar()) != EOF)

putchar(c);

}
```

while သည် character တစ်ခု ရယူ၍ c သို့ assign လုပ်ပြီး character သည် end-of-file signal ဖြစ်ခြင်း ရှိမရှိ စမ်းသပ်သည်။ မဟုတ်ပါက while ၏ body ကို execute လုပ်ပြီး character ကို ပုံနှိပ်သည်။ ထို့နောက် while သည် ထပ်လုပ်သည်။ input ၏ အဆုံး နောက်ဆုံး ရောက်သောအခါ while သည် ပြီးဆုံးပြီး main လည်း ပြီးဆုံးသည်။

ဤဗားရှင်းသည် input ကို ဗဟိုချုပ်ကိုင်သည် - ယခု getchar သို့ ကိုးကားချက် တစ်ခုတည်းသာ ရှိပြီး ပရိုဂရမ်ကို ကျုံ့စေသည်။ ရလဒ် ပရိုဂရမ်သည် ပိုကျစ်လစ်ပြီး idiom ကို ကျွမ်းကျင်လျှင် ဖတ်ရန် ပိုလွယ်ကူသည်။ ဤစတိုင်ကို သင် မကြာခဏ မြင်လိမ့်မည်။ (သို့သော် လွန်လွန်းပြီး နားမလည်နိုင်သော ကုဒ်ကို ဖန်တီးနိုင်သည်၊ ကျွန်ုပ်တို့ ဟန့်တားရန် ကြိုးစားလိမ့်မည့် သဘောထားတစ်ခု။)

Condition အတွင်းရှိ assignment ပတ်လည်က ကွင်းစကွင်းပိတ်များသည် လိုအပ်သည်။ != ၏ precedence သည် = ထက် မြင့်သည်၊ ဆိုလိုသည်မှာ ကွင်းစကွင်းပိတ်များ မရှိပါက relational test != ကို assignment = မတိုင်မီ လုပ်မည်။ ထို့ကြောင့် statement

```
c = getchar() != EOF
```

သည် အောက်ပါနှင့် ညီမျှသည်

```
c = (getchar() != EOF)
```

ဤသည်က getchar ၏ ခေါ်ဆိုမှုသည် end of file ကို ပြန်ပေးခြင်း ရှိမရှိပေါ် မူတည်၍ c ကို 0 သို့မဟုတ် 1 သို့ သတ်မှတ်ခြင်း၏ မလိုလားအပ်သော အကျိုးသက်ရောက်မှု ရှိသည်။ (ဤအကြောင်း နောက်ထပ် Chapter 2 တွင်။)

လေ့ကျင့်ခန်း 1-6. Expression getchar() != EOF သည် 0 သို့မဟုတ် 1 ဖြစ်ကြောင်း အတည်ပြုပါ။

လေ့ကျင့်ခန်း 1-7. EOF ၏ တန်ဖိုးကို ပုံနှိပ်ရန် ပရိုဂရမ်တစ်ခု ရေးပါ။

### 1.5.2 Character ရေတွက်ခြင်း

နောက်ထပ် ပရိုဂရမ်သည် characters များကို ရေတွက်သည်။ ၎င်းသည် ကူးယူခြင်း ပရိုဂရမ်နှင့် တူသည်။

```
#include

/* count characters in input; 1st version */

main()

{

long nc;

nc = 0;

while (getchar() != EOF)

++nc;

printf("%ld\n", nc);

}
```

Statement

```
++nc;
```

သည် operator အသစ် ++ ကို တင်ပြပြီး တစ်ခုဖြင့် တိုးမြှင့်ခြင်း ကို ဆိုလိုသည်။ ၎င်းအစား nc = nc + 1 ဟု ရေးနိုင်သော်လည်း ++nc သည် ပိုတိုတောင်းပြီး မကြာခဏ ပိုထိရောက်သည်။ 1 ဖြင့် လျှော့ချရန်အတွက် သက်ဆိုင်သော operator -- ရှိသည်။ Operators ++ နှင့် -- သည် prefix operators (++nc) သို့မဟုတ် postfix operators (nc++) ဖြစ်နိုင်သည်။ ဤပုံစံ နှစ်ခုသည် expressions များတွင် မတူညီသော တန်ဖိုးများ ရှိသည်၊ Chapter 2 တွင် ပြလိမ့်မည်၊ သို့သော် ++nc နှင့် nc++ နှစ်ခုစလုံးသည် nc ကို တိုးမြှင့်သည်။ လောလောဆယ် ကျွန်ုပ်တို့သည် prefix ပုံစံကို သုံးမည်။

Character ရေတွက်ခြင်း ပရိုဂရမ်သည် ၎င်း၏ အရေအတွက်ကို int အစား long ကိန်းရှင်တွင် စုဆောင်းသည်။ Long integers များသည် အနည်းဆုံး 32 bits ဖြစ်သည်။ အချို့ စက်များတွင် int နှင့် long သည် တူညီသော အရွယ်အစား ဖြစ်သော်လည်း၊ အခြားတို့တွင် int သည် 16 bits ဖြစ်ပြီး အများဆုံး တန်ဖိုး 32767 ရှိသည်၊ int counter ကို ကျော်လွှားရန် input အနည်းငယ်သာ လိုအပ်လိမ့်မည်။ Conversion specification %ld က printf ကို သက်ဆိုင်သော argument သည် long integer ဖြစ်ကြောင်း ပြောသည်။

Double (double precision float) ကို အသုံးပြု၍ ပိုကြီးမားသော နံပါတ်များကို ကိုင်တွယ်နိုင်ခြင်း ဖြစ်နိုင်သည်။ loop ကို ရေးရန် အခြားနည်းလမ်းတစ်ခုကို သရုပ်ပြရန် while အစား for statement ကိုလည်း အသုံးပြုမည်။

```
#include

/* count characters in input; 2nd version */

main()

{

double nc;

for (nc = 0; gechar() != EOF; ++nc)

;

printf("%.0f\n", nc);

}
```

printf သည် float နှင့် double နှစ်ခုလုံးအတွက် %f ကို အသုံးပြုသည်။ %.0f က ဒသမအမှတ်နှင့် သုညဖြစ်သော ဒသမအပိုင်း၏ ပုံနှိပ်မှုကို ဖိနှိပ်သည်။

ဤ for loop ၏ body သည် ဗလာဖြစ်သည်၊ အဘယ်ကြောင့်ဆိုသော် လုပ်ဆောင်ချက်အားလုံးကို စမ်းသပ်မှုနှင့် တိုးမြှင့်မှု အပိုင်းများတွင် လုပ်ဆောင်သောကြောင့် ဖြစ်သည်။ သို့သော် C ၏ သဒ္ဒါစည်းမျဉ်းများက for statement တွင် body ရှိရမည်ဟု လိုအပ်သည်။ Null statement ဟု ခေါ်သော သီးခြား semicolon သည် ထိုလိုအပ်ချက်ကို ပြည့်မီရန် ထည့်ထားသည်။ ၎င်းကို မြင်နိုင်စေရန် သီးခြား စာကြောင်းတစ်ကြောင်းတွင် ထားသည်။

Character ရေတွက်ခြင်း ပရိုဂရမ်ကို မစွန့်မီ၊ input တွင် characters များ မပါရှိပါက while သို့မဟုတ် for စမ်းသပ်မှုသည် getchar သို့ ပထမဆုံး ခေါ်ဆိုမှုတွင်ပင် မအောင်မြင်ပြီး ပရိုဂရမ်သည် သုည ကို ထုတ်ပေးသည်၊ မှန်ကန်သော အဖြေ ဖြစ်သည်ကို သတိထားပါ။ ဤသည် အရေးကြီးသည်။ while နှင့် for ၏ ကောင်းသောအရာများထဲမှ တစ်ခုမှာ ၎င်းတို့သည် body နှင့် ဆက်လက်လုပ်ဆောင်မီ loop ၏ ထိပ်တွင် စမ်းသပ်ခြင်း ဖြစ်သည်။ လုပ်ရန် မရှိပါက မည်သည်ကိုမျှ မလုပ်ပါ၊ loop body ကို လုံးဝ မဖြတ်သန်းခြင်း ဖြစ်စေကာမူ။ ပရိုဂရမ်များသည် သုည-အရှည် input ပေးလျှင် ဉာဏ်ရှိစွာ လုပ်ဆောင်သင့်သည်။ while နှင့် for statements များသည် ပရိုဂရမ်များ boundary conditions များနှင့် ကျိုးကြောင်းဆီလျော်သော အရာများ လုပ်ဆောင်ကြောင်း အာမခံရန် ကူညီသည်။

### 1.5.3 Line ရေတွက်ခြင်း

နောက်ထပ် ပရိုဂရမ်သည် input lines များကို ရေတွက်သည်။ အထက်တွင် ကျွန်ုပ်တို့ ဖော်ပြခဲ့သည့်အတိုင်း၊ standard library သည် input text stream သည် စာကြောင်းများ၏ အစီအစဉ်အဖြစ် ပေါ်လာကြောင်း အာမခံသည်၊ တစ်ခုချင်းစီကို newline ဖြင့် အဆုံးသတ်သည်။ ထို့ကြောင့် စာကြောင်းများ ရေတွက်ခြင်းသည် newlines များ ရေတွက်ခြင်းသာ ဖြစ်သည်:

```
#include

/* count lines in input */

main()

{

int c, nl;

nl = 0;

while ((c = getchar()) != EOF)

if (c == '\n')

++nl;

printf("%d\n", nl);

}
```

while ၏ body တွင် ယခု if ပါဝင်ပြီး ၎င်းသည် တိုးမြှင့်မှု ++nl ကို ထိန်းချုပ်သည်။ if statement သည် ကွင်းစကွင်းပိတ်ထဲရှိ အခြေအနေကို စမ်းသပ်ပြီး အခြေအနေ မှန်ကန်ပါက နောက်က statement (သို့မဟုတ် ကွင်းကောက်များထဲရှိ statements အုပ်စု) ကို execute လုပ်သည်။ မည်သည့်အရာက မည်သည့်အရာကို ထိန်းချုပ်သည်ကို ပြရန် ကျွန်ုပ်တို့ ထပ်မံ indent လုပ်ထားသည်။

Double equals sign == သည် "ညီမျှသည်" အတွက် C ဖော်ပြချက် ဖြစ်သည် (Pascal ၏ single = သို့မဟုတ် Fortran ၏ .EQ. ကဲ့သို့)။ ဤသင်္ကေတကို equality စမ်းသပ်မှုကို C က assignment အတွက် အသုံးပြုသော single = နှင့် ခွဲခြားရန် အသုံးပြုသည်။ သတိပေးချက်: C သို့ အသစ်လာသူများသည် == ကို ရည်ရွယ်သည့်အခါ = ဟု ရံဖန်ရံခါ ရေးသည်။ Chapter 2 တွင် မြင်လိမ့်မည့်အတိုင်း၊ ရလဒ်သည် များသောအားဖြင့် တရားဝင် expression ဖြစ်သောကြောင့် သင် သတိပေးချက် မရရှိပါ။

Single quotes များ အကြားတွင် ရေးထားသော character တစ်ခုသည် စက်၏ character set ရှိ character ၏ ကိန်းဂဏန်း တန်ဖိုးနှင့် ညီမျှသော integer တန်ဖိုးကို ကိုယ်စားပြုသည်။ ဤသည်ကို character constant ဟု ခေါ်သည်၊ ၎င်းသည် သေးငယ်သော integer ကို ရေးရန် အခြားနည်းလမ်းသာ ဖြစ်သော်လည်း။ ထို့ကြောင့် ဥပမာ 'A' သည် character constant ဖြစ်သည်။ ASCII character set တွင် ၎င်း၏ တန်ဖိုးမှာ 65 ဖြစ်ပြီး character A ၏ အတွင်းပိုင်း ကိုယ်စားပြုမှု ဖြစ်သည်။ ဧကန်မုချ 'A' သည် 65 ထက် ပိုနှစ်သက်ဖွယ်ရာ ဖြစ်သည်: ၎င်း၏ အဓိပ္ပါယ်သည် ထင်ရှားပြီး သီးခြား character set အပေါ် မမှီခိုပါ။

String constants များတွင် အသုံးပြုသော escape sequences များသည် character constants များတွင်လည်း တရားဝင် ဖြစ်သည်၊





ထို့ကြောင့် '\n' သည် newline character ၏ တန်ဖိုးကို ကိုယ်စားပြုပြီး ASCII တွင် 10 ဖြစ်သည်။ '\n' သည် character တစ်ခုတည်း ဖြစ်ကြောင်း ဂရုတစိုက် သတိပြုသင့်သည်၊ expressions များတွင် integer တစ်ခုသာ ဖြစ်သည်။ အခြားတစ်ဖက်တွင် "\n" သည် character တစ်ခုတည်းသာ ပါဝင်သော string constant ဖြစ်သည်။ Strings နှင့် characters ၏ အကြောင်းအရာကို Chapter 2 တွင် နောက်ထပ် ဆွေးနွေးသည်။

လေ့ကျင့်ခန်း 1-8. Blanks၊ tabs နှင့် newlines များကို ရေတွက်ရန် ပရိုဂရမ်တစ်ခု ရေးပါ။

လေ့ကျင့်ခန်း 1-9. ၎င်း၏ input ကို ၎င်း၏ output သို့ ကူးယူရန် ပရိုဂရမ်တစ်ခု ရေးပါ၊ blank တစ်ခု သို့မဟုတ် တစ်ခုထက်ပို၍ ပါသော string တစ်ခုချင်းစီကို blank တစ်ခုတည်းဖြင့် အစားထိုးပါ။

လေ့ကျင့်ခန်း 1-10. ၎င်း၏ input ကို ၎င်း၏ output သို့ ကူးယူရန် ပရိုဂရမ်တစ်ခု ရေးပါ၊ tab တစ်ခုချင်းစီကို \t ဖြင့်၊ backspace တစ်ခုချင်းစီကို \b ဖြင့်၊ backslash တစ်ခုချင်းစီကို \ ဖြင့် အစားထိုးပါ။ ဤသည်က tabs နှင့် backspaces များကို ရှင်းရှင်းလင်းလင်း မရှုပ်ထွေးစေဘဲ မြင်နိုင်စေသည်။

### 1.5.4 Word ရေတွက်ခြင်း

ကျွန်ုပ်တို့၏ အသုံးဝင်သော ပရိုဂရမ်များ စီးရီး၏ စတုတ္ထမြောက်သည် lines၊ words နှင့် characters များကို ရေတွက်သည်၊ word တစ်ခုသည် blank၊ tab သို့မဟုတ် newline မပါဝင်သော characters များ၏ အစီအစဉ် ဖြစ်သည်ဟူသော လျော့ရဲသော အဓိပ္ပါယ်ဖွင့်ဆိုချက်ဖြင့်။ ဤသည်မှာ UNIX ပရိုဂရမ် wc ၏ bare-bones ဗားရှင်း ဖြစ်သည်။

```
#include

#define IN 1 /* inside a word */
#define OUT 0 /* outside a word */

/* count lines, words, and characters in input */

main()

{

int c, nl, nw, nc, state;

state = OUT;

nl = nw = nc = 0;

while ((c = getchar()) != EOF) {

++nc;

if (c == '\n')

++nl;

if (c == ' ' || c == '\n' || c = '\t')

state = OUT;

else if (state == OUT) {

state = IN;

++nw;

}

}

printf("%d %d %d\n", nl, nw, nc);

}
```

ပရိုဂရမ်သည် word ၏ ပထမ character ကို တွေ့သည့် အချိန်တိုင်းတွင် word တစ်ခုထပ်ပေါင်း ရေတွက်သည်။ ကိန်းရှင် state သည် ပရိုဂရမ်သည် လက်ရှိ word တစ်ခုအတွင်းတွင် ရှိခြင်း ရှိမရှိ မှတ်တမ်းတင်သည်။ ကနဦးတွင် ၎င်းသည် "word အတွင်းတွင် မဟုတ်" ဖြစ်ပြီး OUT တန်ဖိုးကို ပေးထားသည်။ ကျွန်ုပ်တို့သည် literal တန်ဖိုးများ 1 နှင့် 0 ထက် သင်္ကေတ constants IN နှင့် OUT ကို ပိုနှစ်သက်ပြီး ၎င်းတို့က ပရိုဂရမ်ကို ပို၍ ဖတ်နိုင်စေသောကြောင့် ဖြစ်သည်။ ဤကဲ့သို့ သေးငယ်သော ပရိုဂရမ်တစ်ခုတွင် ကွာခြားချက် အနည်းငယ်သာ ရှိသော်လည်း ပိုကြီးသော ပရိုဂရမ်များတွင် ရှင်းလင်းမှု တိုးပွားခြင်းသည် စတင်ရာမှ ဤနည်းဖြင့် ရေးရန် သမာသမတ် အပိုထိုးအားစိုက်မှုထက် တန်ဖိုးရှိသည်။ မှော်နံပါတ်များ သင်္ကေတ constants များအဖြစ်သာ ပေါ်လာသော ပရိုဂရမ်များတွင် ကျယ်ပြန့်သော ပြောင်းလဲမှုများ ပြုလုပ်ရန် ပိုလွယ်ကူကြောင်းလည်း သင်တွေ့ရလိမ့်မည်။

စာကြောင်း

```
nl = nw = nc = 0;
```

သည် ကိန်းရှင် သုံးခုလုံးကို သုညသို့ သတ်မှတ်သည်။ ဤသည်မှာ အထူးကိစ္စ မဟုတ်ဘဲ assignment သည် တန်ဖိုးနှင့်အတူ expression တစ်ခု ဖြစ်ပြီး assignments များသည် ညာဘက်မှ ဘယ်ဘက်သို့ ချိတ်ဆက်ထားခြင်း၏ အကျိုးဆက် ဖြစ်သည်။ ၎င်းသည် ကျွန်ုပ်တို့ ရေးခဲ့သလို ဖြစ်သည်

```
nl = (nw = (nc = 0));
```

Operator || သည် OR ကို ဆိုလိုသည်၊ ထို့ကြောင့် စာကြောင်း

```
if (c == ' ' || c == '\n' || c = '\t')
```

က "c သည် blank ဖြစ်သည် သို့မဟုတ် c သည် newline ဖြစ်သည် သို့မဟုတ် c သည် tab ဖြစ်သည်" ဟု ပြောသည်။ (Escape sequence \t သည် tab character ၏ မြင်နိုင်သော ကိုယ်စားပြုမှု ဖြစ်ကြောင်း သတိရပါ။) AND အတွက် သက်ဆိုင်သော operator && ရှိသည်။ ၎င်း၏ precedence သည် || ထက် အနည်းငယ် မြင့်သည်။ && သို့မဟုတ် || ဖြင့် ချိတ်ဆက်ထားသော expressions များကို ဘယ်ဘက်မှ ညာဘက်သို့ အကဲဖြတ်ပြီး အမှန် သို့မဟုတ် အမှား သိလျှင် evaluation သည် ရပ်တန့်မည်ဟု အာမခံထားသည်။ c သည် blank ဖြစ်ပါက ၎င်းသည် newline သို့မဟုတ် tab ဖြစ်ခြင်း ရှိမရှိ စမ်းသပ်ရန် မလိုအပ်သောကြောင့် ဤစမ်းသပ်မှုများကို မလုပ်ပါ။ ဤသည်မှာ ဤနေရာတွင် အထူးအရေးကြီးခြင်း မရှိသော်လည်း ပိုရှုပ်ထွေးသော အခြေအနေများတွင် အရေးပါပြီး မကြာမီ မြင်လိမ့်မည်။

ဥပမာသည် else ကိုလည်း ပြသပြီး၊ ၎င်းသည် if statement ၏ condition အပိုင်း မှားပါက အခြား လုပ်ဆောင်ချက်ကို သတ်မှတ်သည်။ ယေဘုယျပုံစံမှာ

```
if (expression)
	statement1
else
	statement2
```

if-else နှင့် ချိတ်ဆက်ထားသော statements နှစ်ခုထဲမှ တစ်ခုတည်းကိုသာ လုပ်ဆောင်သည်။ Expression မှန်ကန်ပါက statement 1 ကို execute လုပ်သည်။ မဟုတ်ပါက statement2 ကို execute လုပ်သည်။ Statement တစ်ခုချင်းစီသည် statement တစ်ခုတည်း သို့မဟုတ် ကွင်းကောက်များထဲရှိ အများအပြား ဖြစ်နိုင်သည်။ Word count ပရိုဂရမ်တွင် else နောက်က တစ်ခုသည် ကွင်းကောက်များထဲရှိ statements နှစ်ခုကို ထိန်းချုပ်သော if ဖြစ်သည်။
