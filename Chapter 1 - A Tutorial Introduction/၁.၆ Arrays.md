## 1.6 Arrays

ဂဏန်းတစ်ခုချင်းစီ၊ white space characters များ (blank၊ tab၊ newline) နှင့် အခြား characters အားလုံး၏ ပေါ်လာမှု အကြိမ်အရေအတွက်ကို ရေတွက်ရန် ပရိုဂရမ်တစ်ခု ရေးကြပါစို့။ ဤသည်မှာ အတုအယောင် ဖြစ်သော်လည်း ပရိုဂရမ်တစ်ခုတည်းတွင် C ၏ ရှုထောင့်များစွာကို သရုပ်ပြရန် ခွင့်ပြုသည်။

Input အမျိုးအစား ဆယ့်နှစ်မျိုး ရှိသောကြောင့် တစ်ခုချင်းစီ ကိန်းရှင် ဆယ်ခု ထက် ဂဏန်းတစ်ခုချင်းစီ၏ ပေါ်လာမှု အရေအတွက်ကို ထားရှိရန် array တစ်ခု အသုံးပြုရန် အဆင်ပြေသည်။ ပရိုဂရမ်၏ ဗားရှင်းတစ်ခုမှာ:

```
#include

/* count digits, white space, others */

main()

{

int c, i, nwhite, nother;

int ndigit[10];

nwhite = nother = 0;

for (i = 0; i < 10; ++i)

ndigit[i] = 0;

while ((c = getchar()) != EOF)

if (c >= '0' && c <= '9')

++ndigit[c-'0'];

else if (c == ' ' || c == '\n' || c == '\t')

++nwhite;

else

++nother;

printf("digits =");

for (i = 0; i < 10; ++i)

printf(" %d", ndigit[i]);

printf(", white space = %d, other = %d\n",

nwhite, nother);

}
```

ဤပရိုဂရမ်၏ output ကို ၎င်းကိုယ်တိုင်အပေါ် လုပ်ဆောင်လျှင်:

```
digits = 9 3 0 0 0 0 0 0 0 1, white space = 123, other = 345 
```

ကြေညာချက်

```
int ndigit[10];
```

သည် ndigit ကို integers ဆယ်ခု၏ array တစ်ခုအဖြစ် ကြေညာသည်။ Array subscripts များသည် C တွင် အမြဲတမ်း သုညတွင် စတင်သည်၊ ထို့ကြောင့် elements များမှာ ndigit[0]၊ ndigit[1]၊ ...၊ ndigit[9] ဖြစ်သည်။ ဤသည်ကို array ကို initialize လုပ်ပြီး ပုံနှိပ်သော for loops များတွင် ထင်ဟပ်သည်။

Subscript သည် မည်သည့် integer expression မဆို ဖြစ်နိုင်ပြီး i ကဲ့သို့ integer ကိန်းရှင်များနှင့် integer constants များ ပါဝင်သည်။

ဤသီးခြား ပရိုဂရမ်သည် ဂဏန်းများ၏ character ကိုယ်စားပြုမှု၏ ဂုဏ်သတ္တိများအပေါ် မှီခိုသည်။ ဥပမာ စမ်းသပ်မှု

```
if (c >= '0' && c <= '9')
```

သည် c ထဲရှိ character သည် ဂဏန်းတစ်ခု ဖြစ်ခြင်း ရှိမရှိ ဆုံးဖြတ်သည်။ ဖြစ်ပါက ထိုဂဏန်း၏ ကိန်းဂဏန်း တန်ဖိုးမှာ

```
c - '0'
```

ဖြစ်သည်။ ဤသည်မှာ '0'၊ '1'၊ ...၊ '9' တို့တွင် ဆက်တိုက် တိုးလာသော တန်ဖိုးများ ရှိမှသာ အလုပ်လုပ်သည်။ ကံကောင်းစွာပင်၊ ဤသည်မှာ character sets အားလုံးအတွက် မှန်ကန်သည်။

အဓိပ္ပါယ်ဖွင့်ဆိုချက်အရ chars များသည် သေးငယ်သော integers များသာ ဖြစ်သောကြောင့် char ကိန်းရှင်များနှင့် constants များသည် ဂဏန်းတွက်ချက်မှု expressions များတွင် ints များနှင့် ထပ်တူ ဖြစ်သည်။ ဤသည်မှာ သဘာဝကျပြီး အဆင်ပြေသည်။ ဥပမာ c-'0' သည် c တွင် သိမ်းဆည်းထားသော character '0' မှ '9' နှင့် သက်ဆိုင်သော 0 နှင့် 9 အကြား တန်ဖိုးရှိသော integer expression ဖြစ်ပြီး ထို့ကြောင့် array ndigit အတွက် တရားဝင် subscript ဖြစ်သည်။

Character တစ်ခုသည် ဂဏန်း၊ white space သို့မဟုတ် အခြားတစ်ခုခု ဖြစ်ခြင်း ရှိမရှိ ဆုံးဖြတ်ချက်ကို အစီအစဉ်ဖြင့် ပြုလုပ်သည်

```
if (c >= '0' && c <= '9')

	++ndigit[c-'0'];

else if (c == ' ' || c == '\n' || c == '\t')

	++nwhite;

else

	++nother;
```

ပုံစံ

```
if (condition1)
	statement1
else if (condition2)
	statement2
	...
	...
else
	statementn
```

သည် ပရိုဂရမ်များတွင် multi-way decision ကို ဖော်ပြရန် နည်းလမ်းတစ်ခုအဖြစ် မကြာခဏ ဖြစ်ပေါ်သည်။ Conditions များကို အပေါ်မှ အစဉ်လိုက် အကဲဖြတ်ပြီး condition တစ်ခုခု ပြည့်မီသည်အထိ လုပ်ဆောင်သည်။ ထိုအချိန်တွင် သက်ဆိုင်သော statement အပိုင်းကို execute လုပ်ပြီး တည်ဆောက်မှု တစ်ခုလုံး ပြီးဆုံးသည်။ (မည်သည့် statement မဆို ကွင်းကောက်များထဲတွင် ပိတ်ထားသော statements များစွာ ဖြစ်နိုင်သည်။) Conditions များထဲမှ တစ်ခုမှ ပြည့်မီခြင်း မရှိပါက နောက်ဆုံး else နောက်က statement ကို ၎င်းရှိပါက execute လုပ်သည်။ Word count ပရိုဂရမ်တွင်ကဲ့သို့ နောက်ဆုံး else နှင့် statement ကို ချန်လှပ်ပါက မည်သည့် လုပ်ဆောင်ချက်မျှ မဖြစ်ပေါ်ပါ။

```
else if(condition)
statement
```

groups များသည် ကနဦး if နှင့် နောက်ဆုံး else အကြား မည်မျှပင် ရှိနိုင်သည်။

Style ကိစ္စအဖြစ် ကျွန်ုပ်တို့ ပြသခဲ့သည့်အတိုင်း ဤတည်ဆောက်မှုကို format လုပ်ရန် အကြံပြုထိုက်သည်။ if တစ်ခုချင်းစီကို အရင် else ကျော်ပြီး indent လုပ်ပါက ဆုံးဖြတ်ချက်များ၏ ရှည်လျားသော အစီအစဉ်သည် စာမျက်နှာ၏ ညာဘက်ခြမ်းမှ ထွက်သွားလိမ့်မည်။

Chapter 4 တွင် ဆွေးနွေးမည့် switch statement သည် condition သည် integer သို့မဟုတ် character expression တစ်ခုခုသည် constants များ၏ set တစ်ခုထဲမှ တစ်ခုနှင့် ကိုက်ညီခြင်း ရှိ/မရှိ ဖြစ်သည့်အခါ အထူးသဖြင့် သင့်လျော်သော multi-way branch ကို ရေးရန် အခြားနည်းလမ်းတစ်ခု ပေးသည်။ ဆန့်ကျင်ဘက်အဖြစ် ကျွန်ုပ်တို့သည် ဤပရိုဂရမ်၏ switch ဗားရှင်းကို Section 3.4 တွင် တင်ပြမည်။
