# အခန်း ၁ - မိတ်ဆက်သင်ခန်းစာ

C ဘာသာစကားဖြင့် အမြန်မိတ်ဆက်ပေးခြင်းဖြင့် စတင်ကြပါစို့။ ကျွန်ုပ်တို့၏ ရည်ရွယ်ချက်မှာ အသေးစိတ်အချက်များ၊ စည်းမျဉ်းများနှင့် ခြွင်းချက်များတွင် မရှုပ်ထွေးစေဘဲ အစစ်အမှန် ပရိုဂရမ်များတွင် ဘာသာစကား၏ မရှိမဖြစ် အစိတ်အပိုင်းများကို ပြသရန်ဖြစ်ပါသည်။ ယခုအချိန်တွင် ကျွန်ုပ်တို့သည် အပြည့်အစုံ သို့မဟုတ် တိကျစွာပင် ဖော်ပြရန် မကြိုးစားပါ (ဥပမာများမှာမူ မှန်ကန်ရန် ရည်ရွယ်ပါသည်)။ သင့်အား အသုံးဝင်သော ပရိုဂရမ်များကို ရေးသားနိုင်သည့် အဆင့်သို့ အမြန်ဆုံး ရောက်ရှိစေလိုပါသည်။ ထို့ကြောင့် အခြေခံများဖြစ်သော variable များနှင့် constant များ၊ ဂဏန်းတွက်ချက်မှုများ၊ control flow၊ function များနှင့် input နှင့် output ၏ အခြေခံများကို အာရုံစိုက်ရပါမည်။ ကျွန်ုပ်တို့သည် ပိုကြီးသော ပရိုဂရမ်များ ရေးသားရန် အရေးကြီးသော C ဘာသာစကား၏ feature များကို ဤအခန်းတွင် တမင်ချန်ထားခဲ့ပါသည်။ ယင်းတို့တွင် pointer များ၊ structure များ၊ C ၏ ကြွယ်ဝသော operator အများစု၊ control-flow statement အများအပြားနှင့် standard library တို့ ပါဝင်ပါသည်။

ဤချဉ်းကပ်မှုနှင့် ၎င်း၏ အားနည်းချက်များရှိပါသည်။ အထင်ရှားဆုံးမှာ မည်သည့် feature အတွက်မဆို အပြည့်အစုံ ဖော်ပြချက်ကို ဤနေရာတွင် မတွေ့ရှိရပါ၊ ထို့ပြင် သင်ခန်းစာသည် တိုတောင်းသောကြောင့် လမ်းမှားပြနိုင်ပါသည်။ ဥပမာများတွင် C ၏ စွမ်းအားအပြည့်ကို အသုံးမပြုသောကြောင့် ၎င်းတို့သည် ဖြစ်နိုင်သမျှ တိုတောင်းပြီး လှပမှု မရှိပါ။ ကျွန်ုပ်တို့သည် ဤသက်ရောက်မှုများကို လျှော့ချရန် ကြိုးစားခဲ့သော်လည်း သတိပေးလိုပါသည်။ နောက်ထပ် အားနည်းချက်တစ်ခုမှာ နောက်ပိုင်း အခန်းများတွင် ဤအခန်း၏ အချို့ကို ထပ်ခါတလဲလဲ ရှင်းပြရမည် ဖြစ်ပါသည်။ ထပ်ခါတလဲလဲ ပြောခြင်းသည် စိတ်ညစ်ခြင်းထက် သင့်အတွက် ပိုမိုအထောက်အကူ ဖြစ်လိမ့်မည်စ ကျွန်ုပ်တို့ မျှော်လင့်ပါသည်။

မည်သို့ပင်ဆိုစေကာမူ၊ အတွေ့အကြုံရှိသော programmer များသည် ဤအခန်းမှ အကြောင်းအရာများမှ ၎င်းတို့၏ programming လိုအပ်ချက်များသို့ အပိုဆောင်း ခန့်မှန်းနိုင်ကြပါလိမ့်မည်။ အစပြုသူများသည် ၎င်းတို့ကိုယ်တိုင် သေးငယ်၍ ဆင်တူသော ပရိုဂရမ်များကို ရေးသားခြင်းဖြင့် ဖြည့်စွက်သင့်ပါသည်။ အဖွဲ့နှစ်ခုစလုံးသည် အခန်း ၂ တွင် စတင်သော ပိုမိုအသေးစိတ် ရှင်းလင်းချက်များကို ချိတ်ဆက်ရန် မူဘောင်တစ်ခုအဖြစ် အသုံးပြုနိုင်ပါသည်။

## ၁.၁ စတင်ခြင်း

Programming ဘာသာစကားအသစ်တစ်ခုကို သင်ယူရန် တစ်ခုတည်းသော နည်းလမ်းမှာ ၎င်းဖြင့် ပရိုဂရမ်များ ရေးသားခြင်းဖြင့်ဖြစ်ပါသည်။ ရေးသားရမည့် ပထမပရိုဂရမ်သည် ဘာသာစကားအားလုံးအတွက် တူညီပါသည်-

_စာလုံးများကို ပရင့်ထုတ်ပါ_

```
hello, world
```

ဤသည်မှာ ကြီးမားသော အတားအဆီးတစ်ခုဖြစ်ပါသည်။ ၎င်းကို ခုန်ကျော်ရန် သင်သည် ပရိုဂရမ် text ကို တစ်နေရာရာတွင် ဖန်တီးနိုင်ရမည်၊ အောင်မြင်စွာ compile လုပ်ရမည်၊ load လုပ်ရမည်၊ run လုပ်ရမည်၊ ထို့နောက် သင့် output ဘယ်မှာ ရောက်သွားသည်ကို ရှာရမည် ဖြစ်ပါသည်။ ဤစက်ပိုင်းဆိုင်ရာ အသေးစိတ်များကို ကျွမ်းကျင်ပြီးပါက အခြားအရာအားလုံးသည် နှိုင်းယှဉ်လျှင် လွယ်ကူပါသည်။

C တွင် "hello, world" ကို ပရင့်ထုတ်ရန် ပရိုဂရမ်မှာ-

```
#include <stdio.h>

main()
{
    printf("hello, world\n");
}
```

ဤပရိုဂရမ်ကို မည်သို့ run လုပ်ရမည်ကမူ သင်အသုံးပြုနေသော system ပေါ်တွင် မူတည်ပါသည်။ တိကျသော ဥပမာတစ်ခုအနေဖြင့်၊ UNIX operating system တွင် သင်သည် ပရိုဂရမ်ကို အမည်သည် ".c" ဖြင့်အဆုံးသတ်သော file တစ်ခုတွင် ဖန်တီးရမည်၊ ဥပမာ hello.c၊ ထို့နောက် အောက်ပါ command ဖြင့် compile လုပ်ရမည်-

```
cc hello.c
```

သင်သည် တစ်စုံတစ်ရာကို အမှားမလုပ်ခဲ့ပါက၊ ဥပမာ စာလုံးတစ်လုံး ချန်လှပ်ခြင်း သို့မဟုတ် စာလုံးပေါင်း မှားယွင်းခြင်း၊ compilation သည် တိတ်ဆိတ်စွာ ဆက်လုပ်ကာ a.out ဟုခေါ်သော executable file တစ်ခုကို ဖန်တီးပါလိမ့်မည်။ သင်သည် a.out ကို အောက်ပါ command ဖြင့် run လုပ်ပါက-

```
a.out
```

၎င်းသည် အောက်ပါကို ပရင့်ထုတ်ပါလိမ့်မည်-

```
hello, world
```

အခြား system များတွင် စည်းမျဉ်းများ ကွဲပြားပါလိမ့်မည်။ ဒေသခံ ကျွမ်းကျင်သူနှင့် စစ်ဆေးပါ။

ယခု၊ ပရိုဂရမ်ကိုယ်တိုင်နှင့် ပတ်သက်၍ အချို့ ရှင်းလင်းချက်များ ပြုလုပ်ပါမည်။ C ပရိုဂရမ်သည် ၎င်း၏အရွယ်အစား မည်မျှပင်ဖြစ်စေ function များနှင့် variable များဖြင့် ဖွဲ့စည်းထားပါသည်။ Function တစ်ခုတွင် လုပ်ဆောင်ရမည့် computing လုပ်ဆောင်ချက်များကို သတ်မှတ်သော statement များ ပါဝင်ပြီး variable များသည် တွက်ချက်မှုအတွင်း အသုံးပြုသော တန်ဖိုးများကို သိမ်းဆည်းပါသည်။ C function များသည် Fortran ရှိ subroutine များနှင့် function များ သို့မဟုတ် Pascal ရှိ procedure များနှင့် function များနှင့် တူညီပါသည်။ ကျွန်ုပ်တို့၏ ဥပမာသည် main ဟု အမည်ပေးထားသော function တစ်ခု ဖြစ်ပါသည်။ ပုံမှန်အားဖြင့် သင်သည် function များကို မည်သည့်အမည်မဆို သင်နှစ်သက်သလို ပေးနိုင်သော်လည်း "main" သည် အထူးဖြစ်ပါသည် - သင့်ပရိုဂရမ်သည် main ၏အစတွင် စတင်ကာ execute လုပ်ပါသည်။ ဆိုလိုသည်မှာ ပရိုဂရမ်တိုင်းတွင် တစ်နေရာရာတွင် main ရှိရမည် ဖြစ်ပါသည်။

main သည် ပုံမှန်အားဖြင့် ၎င်း၏ အလုပ်လုပ်ရန် အထောက်အကူပြုရန် အခြား function များကို ခေါ်ပါမည်၊ အချို့မှာ သင်ရေးသားထားသောအရာများ၊ အချို့မှာ သင့်အတွက် ပံ့ပိုးပေးထားသော library များမှ ဖြစ်ပါသည်။ ပရိုဂရမ်၏ ပထမစာကြောင်း-

```
#include <stdio.h>
```

သည် compiler အား standard input/output library နှင့်ပတ်သက်သော အချက်အလက်များကို ထည့်သွင်းရန် ပြောပါသည်။ ဤစာကြောင်းသည် C source file များစွာ၏ အစတွင် ပေါ်လာပါသည်။ Standard library ကို အခန်း ၇ နှင့် ပူးတွဲ B တွင် ဖော်ပြထားပါသည်။

Function များကြားတွင် ဒေတာ ဆက်သွယ်ရန် နည်းလမ်းတစ်ခုမှာ ခေါ်သော function သည် ၎င်းခေါ်သော function အတွက် argument များဟုခေါ်သော တန်ဖိုးများ စာရင်းကို ပံ့ပိုးပေးရန် ဖြစ်ပါသည်။ Function အမည်နောက်ရှိ ကွင်းစကွင်းများသည် argument list ကို ဝန်းရံထားပါသည်။ ဤဥပမာတွင် main သည် argument များ မမျှော်လင့်သော function တစ်ခုအဖြစ် သတ်မှတ်ထားပြီး ယင်းကို ကွင်းဗလာ ( ) ဖြင့် ညွှန်ပြထားပါသည်။

```
#include <stdio.h>              standard library အကြောင်း အချက်အလက် ထည့်သွင်းပါ

main()                          argument တန်ဖိုးများ မရရှိသော
                                main ဟုခေါ်သော function တစ်ခု သတ်မှတ်ပါ

{                               main ၏ statement များကို braces ဖြင့် ဝန်းရံထားပါသည်

printf("hello, world\n");       main သည် library function printf ကို ခေါ်ပြီး
                                ဤစာလုံးများ အစီအစဉ်ကို ပရင့်ထုတ်ပါသည်

}                               \n သည် newline character ကို ကိုယ်စားပြုပါသည်
```

ပထမ C ပရိုဂရမ်

Function ၏ statement များကို braces { } ဖြင့် ဝန်းရံထားပါသည်။ Function main တွင် statement တစ်ခုသာ ပါဝင်ပါသည်-

```
printf("hello, world\n");
```

Function ကို ၎င်း၏အမည်ဖြင့် ခေါ်ဆိုပြီး ကွင်းစကွင်းပါ argument list ကို နောက်တွင် ထားခြင်းဖြင့် ခေါ်ပါသည်၊ ထို့ကြောင့် ဤသည်မှာ printf function ကို "hello, world\n" argument ဖြင့် ခေါ်ခြင်းဖြစ်ပါသည်။ printf သည် output ပရင့်ထုတ်သော library function ဖြစ်ပြီး ဤကိစ္စတွင် quotation mark များကြားရှိ စာလုံးများ string ကို ပရင့်ထုတ်ပါသည်။

Double quote များတွင်ရှိသော စာလုံးများ အစီအစဉ်ကို "hello, world\n" ကဲ့သို့ character string သို့မဟုတ် string constant ဟုခေါ်ပါသည်။ လောလောဆယ် character string များ၏ ကျွန်ုပ်တို့၏ တစ်ခုတည်းသော အသုံးပြုမှုမှာ printf နှင့် အခြား function များအတွက် argument များအဖြစ် ဖြစ်ပါမည်။

String ထဲရှိ \n sequence သည် newline character အတွက် C notation ဖြစ်ပြီး ပရင့်ထုတ်သောအခါ output ကို နောက်တစ်ကြောင်းရှိ ဘယ်ဘက် margin သို့ ရွှေ့ပါသည်။ သင်သည် \n ကို ချန်လှပ်ထားပါက (စမ်းသပ်ရန် တန်ဖိုးရှိသောစမ်းသပ်မှု)၊ output ပရင့်ထုတ်ပြီးနောက် line advance မရှိသည်ကို တွေ့ရပါလိမ့်မည်။ printf argument တွင် newline character ထည့်သွင်းရန် သင်သည် \n ကို အသုံးပြုရမည်။ သင်သည် အောက်ပါကဲ့သို့ တစ်စုံတစ်ခုကို ကြိုးစားပါက-

```
printf("hello, world
");
```

C compiler သည် error message ထုတ်ပေးပါလိမ့်မည်။

printf သည် newline character ကို အလိုအလျောက် ဘယ်တော့မှ မပံ့ပိုးပါ၊ ထို့ကြောင့် call များစွာကို အဆင့်များဖြင့် output line တစ်ခု တည်ဆောက်ရန် အသုံးပြုနိုင်ပါသည်။ ကျွန်ုပ်တို့၏ ပထမပရိုဂရမ်ကို အောက်ပါကဲ့သို့ ရေးသားနိုင်ပါသည်-

```
#include <stdio.h>

main()
{
    printf("hello, ");
    printf("world");
    printf("\n");
}
```

အတူတူပင် output ထုတ်ပေးရန်။

\n သည် စာလုံးတစ်လုံးတည်းကိုသာ ကိုယ်စားပြုသည်ကို သတိပြုပါ။ \n ကဲ့သို့ escape sequence သည် ရိုက်ရခက်သော သို့မဟုတ် မမြင်ရသော စာလုံးများကို ကိုယ်စားပြုရန် ယေဘုယျနှင့် တိုးချဲ့နိုင်သော ယန္တရားကို ပံ့ပိုးပေးပါသည်။ C ပံ့ပိုးပေးသော အခြားအရာများတွင် tab အတွက် \t၊ backspace အတွက် \b၊ double quote အတွက် " နှင့် backslash ကိုယ်တိုင်အတွက် \ တို့ ပါဝင်ပါသည်။ Section ၂.၃ တွင် အပြည့်အစုံ စာရင်းရှိပါသည်။

**လေ့ကျင့်ခန်း ၁-၁။** သင့် system တွင် "hello, world" ပရိုဂရမ်ကို run လုပ်ပါ။ ပရိုဂရမ်၏ အစိတ်အပိုင်းများကို ချန်လှပ်ထားခြင်းဖြင့် သင်မည်သည့် error message များ ရရှိသည်ကို စမ်းသပ်ကြည့်ပါ။

**လေ့ကျင့်ခန်း ၁-၂။** printf ၏ argument string တွင် \c ပါဝင်သောအခါ မည်သို့ဖြစ်သည်ကို စမ်းသပ်ကြည့်ပါ၊ ဤနေရာတွင် c သည် အထက်တွင် ဖော်ပြမထားသော စာလုံးတစ်စုံတစ်ခု ဖြစ်ပါသည်။
