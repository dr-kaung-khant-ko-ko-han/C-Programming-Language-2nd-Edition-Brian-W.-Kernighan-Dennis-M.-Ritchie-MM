## 1.7 Functions

C တွင် function တစ်ခုသည် Fortran ရှိ subroutine သို့မဟုတ် function နှင့် ညီမျှသည်၊ သို့မဟုတ် Pascal ရှိ procedure သို့မဟုတ် function နှင့် ညီမျှသည်။ Function တစ်ခုသည် တွက်ချက်မှုတစ်ခုကို ထုပ်ပိုးရန် အဆင်ပြေသော နည်းလမ်းတစ်ခု ပေးပြီး ၎င်း၏ အကောင်အထည်ဖော်မှုကို စိုးရိမ်ရန် မလိုဘဲ အသုံးပြုနိုင်သည်။ သင့်လျော်စွာ ဒီဇိုင်းထားသော functions များဖြင့် အလုပ်ကို မည်သို့ လုပ်ဆောင်သည်ကို လျစ်လျူရှုနိုင်သည်။ မည်သည့်အရာ လုပ်ဆောင်သည်ကို သိခြင်းသည် လုံလောက်သည်။ C သည် functions များ အသုံးပြုမှုကို လွယ်ကူစေ၊ အဆင်ပြေစေပြီး ထိရောက်စေသည်။ ကုဒ်၏ အစိတ်အပိုင်းတစ်ခုကို ရှင်းလင်းစေသောကြောင့်သာ တစ်ကြိမ်တည်း သတ်မှတ်ပြီး ခေါ်သော function တိုတောင်းတစ်ခုကို သင် မကြာခဏ မြင်လိမ့်မည်။

ယခုထိ ကျွန်ုပ်တို့သည် ကျွန်ုပ်တို့အတွက် ပံ့ပိုးပေးထားသော printf၊ getchar နှင့် putchar ကဲ့သို့ functions များကိုသာ အသုံးပြုခဲ့သည်။ ယခု ကျွန်ုပ်တို့ကိုယ်တိုင် အချို့ကို ရေးရန် အချိန်ရောက်ပြီ။ C တွင် Fortran ၏ ** ကဲ့သို့ exponentiation operator မရှိသောကြောင့် integer m ကို positive integer power n သို့ မြှောက်တင်ရန် function power(m,n) ကို ရေးခြင်းဖြင့် function သတ်မှတ်ခြင်း၏ ယန္တရားကို သရုပ်ပြကြပါစို့။ ဆိုလိုသည်မှာ power(2,5) ၏ တန်ဖိုးသည် 32 ဖြစ်သည်။ ဤ function သည် သေးငယ်သော integers များ၏ positive powers များကိုသာ ကိုင်တွယ်သောကြောင့် လက်တွေ့ကျသော exponentiation routine မဟုတ်ပါ၊ သို့သော် သရုပ်ပြရန်အတွက် လုံလောက်သည်။ (Standard library တွင် x^y ကို တွက်ချက်သော function pow(x,y) ပါဝင်သည်။)

ဤနေရာတွင် function power နှင့် ၎င်းကို လေ့ကျင့်ရန် main ပရိုဂရမ်တစ်ခု ရှိသည်၊ ထို့ကြောင့် တည်ဆောက်ပုံ တစ်ခုလုံးကို တစ်ကြိမ်တည်း မြင်နိုင်သည်။

```
#include

int power(int m, int n);

/* test power function */

main()

{

int i;

for (i = 0; i < 10; ++i)

printf("%d %d %d\n", i, power(2,i), power(-3,i));

return 0;

}

/* power: raise base to n-th power; n >= 0 */ int power(int base, int n)

{

int i, p;

p = 1;

for (i = 1; i <= n; ++i)

p = p * base;

return p;

}
```

Function သတ်မှတ်ခြင်းတွင် ဤပုံစံ ရှိသည်:

```
return-type function-name(parameter declarations, if any) {

declarations

statements

}
```

Function သတ်မှတ်ချက်များသည် မည်သည့်အစဉ်တွင်မဆို ပေါ်လာနိုင်ပြီး source file တစ်ခု သို့မဟုတ် အများအပြားတွင် ပေါ်လာနိုင်သည်၊ function တစ်ခုကို files များကြား ခွဲခြား၍ မရသော်လည်း။ Source ပရိုဂရမ်သည် files များစွာတွင် ပေါ်လာပါက တစ်ခုတွင် အားလုံး ပေါ်လာခြင်းထက် compile နှင့် load လုပ်ရန် ပိုပြောရလိမ့်မည်၊ သို့သော် ၎င်းသည် operating system ကိစ္စ ဖြစ်သည်၊ ဘာသာစကား attribute မဟုတ်ပါ။ လောလောဆယ် functions နှစ်ခုလုံးသည် တူညီသော file တွင် ရှိသည်ဟု ယူဆမည်၊ ထို့ကြောင့် C ပရိုဂရမ်များ လုပ်ဆောင်ခြင်းအကြောင်း သင်သင်ယူခဲ့သည်မှန်သမျှ ဆက်လက် အလုပ်လုပ်လိမ့်မည်။

Function power ကို main မှ စာကြောင်းတွင် နှစ်ကြိမ် ခေါ်သည်

```
printf("%d %d %d\n", i, power(2,i), power(-3,i)); 
```

ခေါ်ဆိုမှုတစ်ခုချင်းစီသည် power သို့ arguments နှစ်ခု ပေးပို့ပြီး တစ်ခုချင်းစီတွင် format လုပ်ပြီး ပုံနှိပ်ရန် integer တစ်ခု ပြန်ပေးသည်။ Expression တစ်ခုတွင် power(2,i) သည် 2 နှင့် i ကဲ့သို့ integer တစ်ခု ဖြစ်သည်။ (Functions အားလုံးသည် integer တန်ဖိုး မထုတ်ပါ။ ကျွန်ုပ်တို့ ဤအကြောင်းကို Chapter 4 တွင် လုပ်ဆောင်မည်။)

power ကိုယ်တိုင်၏ ပထမစာကြောင်း

```
int power(int base, int n)
```

သည် parameter အမျိုးအစားများနှင့် အမည်များကို ကြေညာပြီး function ပြန်ပေးသော ရလဒ်၏ အမျိုးအစားကို ကြေညာသည်။ power က ၎င်း၏ parameters များအတွက် အသုံးပြုသော အမည်များသည် power ထံတွင် local ဖြစ်ပြီး အခြား function များမှ မမြင်နိုင်ပါ။ အခြား routines များသည် ပဋိပက္ခမရှိဘဲ တူညီသော အမည်များကို အသုံးပြုနိုင်သည်။ ဤသည်မှာ ကိန်းရှင်များ i နှင့် p အတွက်လည်း မှန်ကန်သည်။ power ရှိ i သည် main ရှိ i နှင့် မဆက်စပ်ပါ။

Function တစ်ခုရှိ ကွင်းစကွင်းပိတ်ထဲက စာရင်းတွင် အမည်ပေးထားသော ကိန်းရှင်အတွက် parameter ကို ကျွန်ုပ်တို့ ယေဘုယျအားဖြင့် အသုံးပြုမည်။ formal argument နှင့် actual argument ဟူသော အသုံးအနှုန်းများကို တူညီသော ခွဲခြားမှုအတွက် တစ်ခါတစ်ရံ အသုံးပြုသည်။

power က တွက်ချက်သော တန်ဖိုးကို return: statement ဖြင့် main သို့ ပြန်ပေးသည်။ return နောက်တွင် မည်သည့် expression မဆို လိုက်နိုင်သည်:

```
return expression;
```

Function တစ်ခုသည် တန်ဖိုးတစ်ခု ပြန်ပေးရန် မလိုအပ်ပါ။ Expression မပါသော return statement သည် ထိန်းချုပ်မှုကို ပြန်ပေးသော်လည်း အသုံးဝင်သော တန်ဖိုး မပေးပါ၊ function ၏ "အဆုံးသို့ ကျရောက်ခြင်း" ဖြင့် နောက်ဆုံး ညာဘက် ကွင်းကောက်သို့ ရောက်ရှိခြင်း ကဲ့သို့ပင်။ ခေါ်ဆိုသော function သည် function မှ ပြန်ပေးသော တန်ဖိုးကို လျစ်လျူရှုနိုင်သည်။

main ၏ အဆုံးတွင် return statement တစ်ခု ရှိကြောင်း သင် သတိပြုမိမည်။ main သည် အခြား function များကဲ့သို့ function တစ်ခု ဖြစ်သောကြောင့် ၎င်းသည် ၎င်း၏ caller သို့ တန်ဖိုးတစ်ခု ပြန်ပေးနိုင်ပြီး ၎င်းသည် ပရိုဂရမ် execute လုပ်ခဲ့သော environment ဖြစ်သည်။ ပုံမှန်အားဖြင့် သုည၏ return တန်ဖိုးသည် ပုံမှန် ပြီးဆုံးခြင်းကို ဆိုလိုသည်။ သုညမဟုတ်သော တန်ဖိုးများသည် ပုံမှန်မဟုတ်သော သို့မဟုတ် အမှားပါသော ပြီးဆုံးခြင်း အခြေအနေများကို အချက်ပြသည်။ ရိုးရှင်းမှု အကျိုးစီးပွားအတွက် ကျွန်ုပ်တို့သည် ယခုအချိန်အထိ ကျွန်ုပ်တို့၏ main functions များမှ return statements များကို ချန်လှပ်ခဲ့သော်လည်း ပရိုဂရမ်များသည် ၎င်းတို့၏ environment သို့ status ပြန်ပေးသင့်ကြောင်း အသိပေးချက်အဖြစ် ယခုနောက် ၎င်းတို့ကို ထည့်သွင်းမည်။

ကြေညာချက်

```
int power(int base, int n);
```

main မတိုင်မီက power သည် int arguments နှစ်ခု မျှော်လင့်ပြီး int ပြန်ပေးသော function တစ်ခု ဖြစ်ကြောင်း ပြောသည်။ Function prototype ဟု ခေါ်သော ဤကြေညာချက်သည် power ၏ သတ်မှတ်ချက်နှင့် အသုံးပြုမှုများနှင့် သဘောတူရမည်။ Function ၏ သတ်မှတ်ချက် သို့မဟုတ် ၎င်း၏ အသုံးပြုမှု မည်သည့်အရာမဆို ၎င်း၏ prototype နှင့် သဘောမတူပါက အမှား ဖြစ်သည်။

Parameter အမည်များသည် သဘောတူရန် မလိုအပ်ပါ။ ဧကန်မုချ function prototype တွင် parameter အမည်များသည် ရွေးချယ်ခွင့် ရှိသည်၊ ထို့ကြောင့် prototype အတွက် ကျွန်ုပ်တို့

```
int power(int, int);
```

ဟု ရေးနိုင်သည်။

သို့သော် ကောင်းစွာ ရွေးချယ်ထားသော အမည်များသည် ကောင်းသော documentation ဖြစ်သည်၊ ထို့ကြောင့် ကျွန်ုပ်တို့ မကြာခဏ ၎င်းတို့ကို အသုံးပြုမည်။

သမိုင်းဆိုင်ရာ မှတ်ချက်: ANSI C နှင့် အစောပိုင်း ဗားရှင်းများကြား အကြီးမားဆုံး အပြောင်းအလဲမှာ functions များကို မည်သို့ ကြေညာပြီး သတ်မှတ်သည် ဖြစ်သည်။ C ၏ မူလ သတ်မှတ်ချက်တွင် power function ကို ဤကဲ့သို့ ရေးမည်:

```
/* power: raise base to n-th power; n >= 0 */

/* (old-style version) */

power(base, n)

int base, n;

{

int i, p;

p = 1;

for (i = 1; i <= n; ++i)

p = p * base;

return p;

}
```

Parameters များကို ကွင်းစကွင်းပိတ်များကြား အမည်ပေးပြီး ဘယ်ဘက် ကွင်းကောက်မဖွင့်မီ ၎င်းတို့၏ အမျိုးအစားများကို ကြေညာသည်။ ကြေညာမထားသော parameters များကို int အဖြစ် ယူသည်။ (Function ၏ body သည် အရင်နှင့် တူညီသည်။)

ပရိုဂရမ်၏ အစတွင် power ၏ ကြေညာချက်က ဤကဲ့သို့ ဖြစ်မည်:

```
int power();
```

Parameter စာရင်းကို ခွင့်မပြုထားသောကြောင့် compiler သည် power ကို မှန်ကန်စွာ ခေါ်နေခြင်း ရှိမရှိ အလွယ်တကူ စစ်ဆေးလို့ မရခဲ့ပါ။ ဧကန်မုချ default အားဖြင့် power သည် int ပြန်ပေးသည်ဟု ယူဆခံရမည်ဖြစ်သောကြောင့် ကြေညာချက် တစ်ခုလုံးကို ချန်လှပ်ထားနိုင်သည်။

Function prototypes ၏ syntax အသစ်က compiler အတွက် arguments များ၏ အရေအတွက် သို့မဟုတ် ၎င်းတို့၏ အမျိုးအစားများရှိ အမှားများကို ရှာဖွေရန် များစွာ လွယ်ကူစေသည်။ ကြေညာချက်နှင့် သတ်မှတ်ချက်၏ ဟောင်း style သည် ANSI C တွင် ယခုထိ အလုပ်လုပ်ဆဲ ဖြစ်သည်၊ အနည်းဆုံး အသွင်ကူးပြောင်းရေး ကာလအတွက်၊ သို့သော် ၎င်းကို ပံ့ပိုးသော compiler ရှိလျှင် ပုံစံအသစ်ကို အသုံးပြုရန် ကျွန်ုပ်တို့ အပြင်းအထန် အကြံပြုပါသည်။
